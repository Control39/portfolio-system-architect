# Базовые Docker образы для проекта

Эта директория содержит базовые Docker образы, используемые всеми микросервисами проекта для обеспечения единообразия, оптимизации и безопасности.

## Структура

```
docker/base-images/
├── python/              # Базовый образ для Python микросервисов
│   ├── Dockerfile      # Многоступенчатый Dockerfile
│   └── README.md       # Документация Python образа
├── node/               # Базовый образ для Node.js микросервисов (планируется)
├── alpine/             # Базовый образ Alpine для минимальных образов (планируется)
└── build.sh            # Скрипт для сборки всех базовых образов
```

## Преимущества использования базовых образов

1. **Единообразие**: Все сервисы используют одинаковые версии зависимостей
2. **Безопасность**: Централизованное обновление зависимостей и исправление уязвимостей
3. **Оптимизация**: Кэширование слоёв и уменьшение размера финальных образов
4. **Скорость разработки**: Быстрая сборка дочерних образов

## Использование

### Для Python микросервисов

В Dockerfile микросервиса используйте базовый образ:

```dockerfile
# Использование базового Python образа
FROM portfolio-python-base:latest AS base

# Копирование зависимостей сервиса
COPY requirements-service.txt .

# Установка специфичных зависимостей поверх базовых
RUN pip install --no-cache-dir -r requirements-service.txt

# Копирование кода приложения
COPY src/ /app/src/

# Запуск приложения
CMD ["uvicorn", "src.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Сборка базовых образов

```bash
# Сборка всех базовых образов
./docker/base-images/build.sh

# Или сборка конкретного образа
docker build -t portfolio-python-base:latest -f docker/base-images/python/Dockerfile .
```

## Миграция существующих сервисов

Для миграции существующего сервиса на базовый образ:

1. **Анализ зависимостей**: Выделить общие зависимости в базовый образ
2. **Обновление Dockerfile**: Заменить FROM на базовый образ
3. **Тестирование**: Проверить работоспособность сервиса
4. **Деплой**: Постепенная миграция сервисов

## Политика версионирования

- **MAJOR**: Критические изменения, ломающие обратную совместимость
- **MINOR**: Добавление новых зависимостей или функций
- **PATCH**: Исправления безопасности и багов

## Мониторинг и обновление

- Регулярное сканирование на уязвимости (Trivy, Grype)
- Автоматическое обновление базовых зависимостей
- Тестирование совместимости перед обновлением

## Контрибуция

Для добавления новой зависимости в базовый образ:

1. Обновить Dockerfile и зависимости
2. Протестировать на нескольких сервисах
3. Обновить документацию
4. Создать pull request
