    AI-навык для SourceCraft: Repository Audit Assistant

    Назначение

    Навык анализирует отчёт аудита репозитория, предоставляет рекомендации по улучшению и отвечает на вопросы о зрелости проекта.

    Конфигурация навыка

    Файл .sourcecraft/skills/repo-audit-assistant.yml:

    name: repo-audit-assistant
version: 1.0
description: Ассистент по аудиту репозиториев, анализирует чек-листы и даёт рекомендации.
triggers:
  - event: pull_request.opened
    action: analyze_pr
  - event: issue.comment
    pattern: "/audit"
    action: run_audit
  - command: "audit repo"
    action: run_audit

actions:
  analyze_pr:
    description: Анализирует изменения в PR и оценивает влияние на зрелость репозитория.
    steps:
      - run_tool: repo_audit
        args: ["run", "--level", "all", "--format", "json"]
      - parse_report: ${output}
      - generate_comment: ${report}

  run_audit:
    description: Запускает полный аудит репозитория и возвращает сводку.
    steps:
      - run_tool: repo_audit
        args: ["run", "--level", "all", "--format", "markdown"]
      - store: ${output}
      - respond: "Аудит завершён. Вот отчёт:\n\n${output}"

tools:
  - name: repo_audit
    type: command
    command: python -m repo_audit
    args: ["run"]
    working_dir: "."

prompt: |
  Ты — эксперт по качеству репозиториев и DevOps-практикам.
  Твоя задача — анализировать отчёты аудита репозитория и давать конкретные, actionable рекомендации.

  Контекст:
  - Репозиторий: {{ repo_name }}
  - Ветка: {{ branch }}
  - Уровень зрелости: {{ maturity_level }}

  Отчёт аудита:
  {{ audit_report }}

  Проанализируй отчёт и ответь на вопросы:
  1. Какие обязательные проверки не пройдены?
  2. Что нужно сделать, чтобы перейти на следующий уровень зрелости?
  3. Какие инструменты можно автоматизировать?
  4. Есть ли риски безопасности или производительности?

  Будь краток, но конкретен. Используй маркдаун для оформления.

examples:
  - user: "Проверь мой репозиторий"
    assistant: "Запускаю аудит... Вот отчёт: ..."
  - user: "Как улучшить уровень 2?"
    assistant: "Для перехода на уровень 2 вам нужно: 1) Настроить Dependabot, 2) Добавить SAST-сканирование, 3) Внедрить автоматические релизы."

    Интеграция с SourceCraft

    1. Установка навыка — разместить конфиг в .sourcecraft/skills/ и зарегистрировать в SourceCraft.
    2. Взаимодействие — навык может быть вызван через чат-интерфейс, вебхуки или по расписанию.
    3. Доступ к инструментам — навык использует CLI repo_audit, который должен быть установлен в окружении.

    Пример промта для генерации рекомендаций

    На основе отчёта аудита сгенерируй план улучшений на ближайший спринт (2 недели).
Учитывай приоритеты: безопасность > качество кода > автоматизация > документация.

    Расширенные возможности

    Анализ трендов

    Навык может хранить историю аудитов и строить графики улучшения/деградации по уровням.

    Интеграция с внешними системами

    • Отправка отчётов в Slack/Telegram.
    • Создание Issues в GitHub с тегами audit-fix.
    • Обновление дашборда Grafana с метриками зрелости.

    Кастомизация чек-листа

    Пользователь может задать свой чек-лист через конфиг, навык адаптирует анализ.

    Следующие шаги

    1. Реализовать базовый навык в SourceCraft (требуется режим Code).
    2. Протестировать взаимодействие с инструментом repo_audit.
    3. Настроить вебхук для автоматического анализа PR.
    4. Добавить примеры диалогов для обучения модели.
