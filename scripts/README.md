## ⚠️ ВАЖНОЕ ПРЕДУПРЕЖДЕНИЕ

### Документация скриптов - критически важна!

**Если скрипт может изменять репозиторий, это ДОЛЖНО быть описано в его README.md или документации!**

Это наиважнейшее правило для AI-агента:
- Агент читает документацию, чтобы понять, что делает скрипт
- Если документация устарела или содержит ошибки, агент может **испортить репозиторий**
- Всё изменение репозитория должно быть **явно описано в документации**

### Что проверять перед запуском скрипта:

1. **Читает ли скрипт README.md или другие файлы документации?**
   - Если да → документация должна быть актуальной
   - Если нет → в скрипте могут быть "зашиты" статичные данные, которые устарели

2. **Может ли скрипт изменять файлы?**
   - Если да → должна быть документация, описывающая **какие именно файлы** и **каким образом** будут изменены
   - Если документации нет → скрипт считается **только для чтения**

3. **Динамические или статичные данные?**
   - Динамические данные → обновляются при каждом запуске
   - Статичные данные ("зашиты" в коде) → могут устареть!

### Пример опасной ситуации:

```python
# ❌ ПЛОХО: статичное значение, которое может устареть
def check_services():
    expected_count = 20  # Это число устарело! Теперь сервисов 21
    # Агент увидит ошибку и попытается "исправить" репозиторий
```

```python
# ✅ ХОРОШО: динамический анализ
from pathlib import Path

def count_services():
    return len([d for d in Path("apps").iterdir() if d.is_dir()])
# Агент всегда увидит актуальное состояние
```

## 📚 Документация и безопасность

### Почему документация скриптов критически важна:

1. **AI-агент читает документацию**
   - Без документации агент не может понять, что делает скрипт
   - Агент может ошибочно "исправить" репозиторий на основе устаревшей информации

2. **Документация = контракт**
   - Если скрипт изменяет файлы, это должно быть описано в README
   - Агент проверяет документацию перед изменениями

3. **Статичные данные = опасность**
   - "Зашитые" в коде значения (количество сервисов, версии и т.д.) устаревают!
   - Всегда используйте динамический анализ (Path().glob(), os.listdir())

### Правила для разработчиков:

- ✅ **Всегда** добавляйте README.md к новым скриптам
- ✅ Описывайте, **что именно** делает скрипт
- ✅ Если скрипт изменяет файлы - опишите, **какие** и **каким образом**
- ❌ **НЕ** используйте статичные значения, которые могут устареть
- ❌ **НЕ** полагайтесь на "очевидные" знания - они не очевидны для агента!

### Пример хорошей документации:

```markdown
# scripts/fix_service.py

## Что делает
Автоматически исправляет структуру сервиса, добавляя недостающие файлы.

## Изменения
- Создаёт `README.md` если отсутствует
- Создаёт `requirements.txt` если отсутствует
- Создаёт `Dockerfile` если отсутствует

## Безопасность
Скрипт ничего не удаляет - только создаёт недостающие файлы.
```
