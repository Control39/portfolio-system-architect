# Скрипты автоматизации

## `update-coverage-badge.py`

Автоматическое обновление бейджа тестового покрытия в README.md

### Назначение

Скрипт решает проблему «ложных бейджей» — когда README показывает 95% покрытие, а тесты на самом деле не проходят.

### Что делает скрипт

1. **Запускает тесты** с генерацией `coverage.xml`
2. **Извлекает процент покрытия** из XML-отчёта
3. **Обновляет бейдж** в README.md
4. **Обновляет дату** в `docs/TEST-COVERAGE-METRICS.md`
5. **(Опционально)** Делает git commit

### Использование

#### Через Make (рекомендуется)

```bash
make update-badge
```

#### Вручную

```bash
# Windows / Python
python scripts/update-coverage-badge.py

# Linux / Mac / WSL
python scripts/update-coverage-badge.py
```

### Исключаемые тесты

Скрипт автоматически исключает пре-existing проблемы (не блокируют функциональность):

- `tests/unit/test_assistant_orchestrator_main.py` (CLI-тесты)
- `tests/unit/test_embedding_agent_updated.py` (ChromaDB/HF-тесты)

📄 **Подробности:** [`docs/KNOWN_ISSUES.md`](../docs/KNOWN_ISSUES.md)

### Порог качества

- **Минимальное покрытие:** 90%
- Если покрытие ниже — скрипт прерывается с ошибкой

### Вывод

После успешного выполнения:

```
============================================================
✅ Обновление завершено!
   Покрытие: 93%
   README.md обновлён
   TEST-COVERAGE-METRICS.md обновлён
============================================================
```

### Пример обновления README

**До:**
```markdown
[![Coverage](https://img.shields.io/codecov/c/github/Control39/portfolio-system-architect...)](https://app.codecov.io/...)
```

**После:**
```markdown
[![Coverage](https://img.shields.io/badge/coverage-93%25-brightgreen)](docs/TEST-COVERAGE-METRICS.md)
*Обновлено: 3 мая 2026*
```

### Преимущества

✅ **Прозрачность** — дата обновления всегда актуальна
✅ **Автоматизация** — не нужно вручную считать покрытие
✅ **Защита от лжи** — бейдж обновляется только ПОСЛЕ успешных тестов
✅ **Работает из РФ** — статический бейдж не блокируется

### Частые вопросы

**Q:** Что если тесты упали?
**A:** Скрипт прервётся с ошибкой. Бейдж не будет обновлён.

**Q:** Можно ли изменить порог покрытия?
**A:** Да, в строке `--cov-fail-under=90` замените 90 на нужное значение.

**Q:** Как добавить новые исключаемые тесты?
**A:** Отредактируйте список `EXCLUDED_TESTS` в скрипте.
