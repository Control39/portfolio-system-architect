### 2. Что такое виртуальное окружение
**Вопрос:** Расскажи что такое виртуальное окружение

**Ответ:**
- Изолированное пространство для проекта с собственными зависимостями
- Решает конфликты версий между проектами
- Пример: `FastAPI 0.10.0` в Проекте A, `FastAPI 0.136.1` в Проекте B
- Команды: `python -m venv .venv`, `.venv\Scripts\Activate.ps1`, `deactivate`

### 9. Что происходит при git push
**Вопрос:** Что происходит при git push?

**Ответ:**
**Процесс:**
1. Git проверяет доступ к `origin` и актуальность ветки
2. Отправляет коммиты, деревья файлов, блобы, теги
3. На GitHub обновляется `refs/heads/main`, запускаются GitHub Actions
4. Все становятся видны команде

**В вашем случае:** 102 локальных коммита будут отправлены, запустятся CI/CD workflows, Dependabot может создать новые PR.

### 10. Проверка конфликтов перед пушем
**Вопрос:** Как именно проверяют на конфликты перед пушем?

**Ответ:**
**Чеклист:**
1. `git fetch origin`
2. `git status` (впереди или позади remote)
3. `git log --oneline origin/main..HEAD` (просмотр коммитов)
4. `git diff origin/main..HEAD` (различия)
5. `git rebase --dry-run origin/main` (проверка конфликтов)
6. `make test` (локальные тесты перед пушем)
7. `git push origin main`

### 14. Контейнеризация
**Вопрос:** Контейнеризированы это значит каждый можно установить независимо от другого и всё для установки готово?

**Ответ:**
**Нет.** Контейнеризация означает:
- Упаковка сервиса с зависимостями в Docker-контейнер
- Изоляция от других сервисов
- Переносимость (работает везде с Docker)

**НО:** Сервисы **зависят друг от друга** (API, БД, очереди), поэтому нужна **оркестрация** (Docker Compose, Kubernetes).

**Запуск:** `docker-compose up` (запускает ВСЕ сервисы сразу).
