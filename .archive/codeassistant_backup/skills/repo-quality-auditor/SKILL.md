# Skill: Repository Quality Auditor

## Проверяет:
✅ **Структура**: Соответствие заявленной архитектуре, отсутствие «матрешек» (docs/docs/)
✅ **Ссылки**: Все внутренние/внешние ссылки рабочие (lychee)
✅ **Бейджи**: 21 бейдж на месте, все показывают актуальный статус
✅ **Тесты**: Покрытие >85% в core, тесты запускаются без ошибок
✅ **Язык**: Документация на естественном русском/английском, без машинного перевода
✅ **Честность**: Код в README соответствует реальному коду в репозитории

## Детальные проверки

### 1. Структура репозитория
- Проверить соответствие структуры заявленной в `docs/architecture/`
- Отсутствие вложенных папок с одинаковыми именами (например, `docs/docs/`)
- Все компоненты имеют `README.md` с описанием назначения
- Конфигурационные файлы на месте: `pyproject.toml`, `docker-compose.yml`, `mkdocs.yml`

### 2. Проверка ссылок
- Запустить `lychee --no-progress --output json .` для всех `.md` файлов
- Внутренние ссылки (относительные) должны вести на существующие файлы
- Внешние ссылки не должны возвращать 404/410
- Ссылки на Issues/PRs должны быть актуальными

### 3. Бейджи (21 обязательных)
1. **CI/CD**: GitHub Actions status
2. **Тесты**: pytest coverage
3. **Лицензия**: MIT badge
4. **Версия Python**: 3.11+
5. **Docker**: Image size, pulls
6. **Code Quality**: CodeFactor, SonarCloud
7. **Документация**: ReadTheDocs, MkDocs
8. **Зависимости**: Dependabot, Snyk
9. **Безопасность**: Trivy, OWASP
10. **Форматирование**: Black, isort
11. **Линтинг**: Ruff, Pylint
12. **Типы**: MyPy strict
13. **Производительность**: Benchmarks
14. **Совместимость**: OS support
15. **Активность**: Last commit
16. **Звёзды**: GitHub stars
17. **Форки**: GitHub forks
18. **Issues**: Open/closed
19. **PRs**: Open/merged
20. **Релизы**: Latest version
21. **Скачивания**: PyPI/downloads

### 4. Тестирование
- Запустить `pytest --cov=src --cov=apps --cov-report=html`
- Проверить, что покрытие core-модулей >85%
- Все тесты должны проходить без ошибок
- Интеграционные тесты должны работать с `docker-compose up`

### 5. Язык документации
- Проверить отсутствие машинного перевода (неестественные конструкции)
- Единый стиль: либо русский, либо английский, либо оба с чётким разделением
- Технические термины используются корректно
- Примеры кода работают (можно запустить `python -m doctest`)

### 6. Честность (Honesty Check)
- Сравнить код в `README.md` с реальными файлами
- Проверить, что скриншоты соответствуют текущему интерфейсу
- Убедиться, что метрики в бейджах актуальны
- Проверить, что примеры использования работают

## Формат отчёта:
```yaml
repo_quality:
  structure: 
    status: "✅/❌"
    issues: ["найдена матрешка docs/docs/", "отсутствует README в apps/cloud-reason"]
    
  links:
    total: 156
    broken: ["https://example.com/old", "docs/missing.md"]
    
  badges:
    total: 21
    working: 19
    broken: ["codecov", "docker-pulls"]
    
  tests:
    coverage: "87%"
    core_coverage: "92%"
    failing: ["test_edge_case.py::test_failure"]
    
  language:
    issues: ["машинный перевод в CONTRIBUTING.md стр. 45"]
    
  honesty:
    mismatches: ["README показывает старый API", "скриншот устарел"]
```

## Автоматизация
- Ежедневный запуск через GitHub Actions
- Отчёт публикуется в `badges/quality-report.json`
- Уведомления в Slack при деградации качества
- Авто-создание Issues для критических проблем
