# Руководство по исправлению проблем с кодировкой

Добро пожаловать в набор инструментов для исправления проблем с кодировкой в проекте **cloud-reason**.

Этот комплект инструментов предназначен для решения распространенных проблем с кодировками,
особенно в средах Windows с поддержкой кириллицы. Все инструменты оптимизированы для работы
с Windows 11 и современными инструментами разработки.

## Обзор системы

Набор инструментов включает в себя:

- **Скрипты Python** для анализа, конвертации и валидации кодировок
- **Конфигурационные файлы** для VS Code, Git и Windows
- **Утилиты командной строки** для быстрой настройки и восстановления
- **Документация** для пользователей и разработчиков

Все компоненты работают вместе, обеспечивая безопасное и надежное решение проблем с кодировкой.

## Требования

### Минимальные требования

- **Операционная система**: Windows 10 или 11
- **Python**: 3.8 или выше
- **Git**: 2.30 или выше (рекомендуется)
- **Дисковое пространство**: минимум 100 МБ свободного места

### Рекомендуемые компоненты

- **Терминал**: Windows Terminal (из Microsoft Store)
- **Редактор кода**: Visual Studio Code
- **Права администратора**: для полной настройки системы

## Подробное руководство

### Анализ кодировок (`analyze_all.py`)

Скрипт анализирует все текстовые файлы в проекте и определяет их кодировку.

**Возможности**:
- Автоматическое определение кодировки (UTF-8, CP1251, ISO-8859-1 и др.)
- Поддержка кириллицы в путях и именах файлов
- Подробная статистика по кодировкам
- Логирование в формате JSON и текстовом виде

**Запуск**:
```cmd
python scripts\analyze_all.py
```

**Выходные файлы**:
- `logs/encoding_analysis.json` - результаты анализа в JSON
- `logs/encoding_analysis.log` - подробный лог операции

### Конвертация в UTF-8 (`convert_to_utf8.py`)

Основной скрипт для массовой конвертации файлов в UTF-8.

**Особенности**:
- Автоматическое резервное копирование перед конвертацией
- Поддержка всех распространенных кодировок
- Безопасная обработка ошибок с восстановлением
- Поддержка кириллических путей
- Подробное логирование

**Запуск**:
```cmd
python scripts\convert_to_utf8.py
```

**Процесс работы**:
1. Определение текущей кодировки каждого файла
2. Создание резервной копии в директории `backups/`
3. Конвертация в UTF-8
4. Проверка успешности операции
5. Обновление статистики и логов

**Выходные файлы**:
- `backups/pre_utf8_conversion_YYYYMMDD_HHMMSS/` - резервные копии
- `logs/conversion_results.json` - результаты конвертации
- `logs/conversion.log` - подробный лог операции

### Валидация результатов (`validate_results.py`)

Проверка корректности выполненной конвертации.

**Проверяемые аспекты**:
- Читаемость файлов в кодировке UTF-8
- Отсутствие символов-заменителей (�)
- Корректность специальных символов и диакритики
- Отсутствие бинарных данных в текстовых файлах

**Запуск**:
```cmd
python scripts\validate_results.py
```

### Подготовка коммитов Git (`git_fix_history.py`)

Создание корректных коммитов после конвертации кодировок.

**Функции**:
- Проверка состояния Git репозитория
- Создание коммита с резервной копией
- Создание коммита с изменениями кодировок
- Поддержка сообщений коммитов на UTF-8
- Интеграция с системой логирования

**Запуск**:
```cmd
python scripts\git_fix_history.py
```

## Безопасность и резервное копирование

### Принципы безопасности

1. **Автоматическое резервное копирование**
   - Все скрипты создают резервные копии перед изменениями
   - Резервные копии хранятся в отдельной директории `backups/`
   - Имена директорий включают временную метку

2. **Пошаговая обработка**
   - Операции выполняются поэтапно
   - Каждый этап можно проверить перед продолжением
   - Возможность отката на любом этапе

3. **Проверка целостности**
   - Все изменения проверяются на корректность
   - При ошибках выполняется восстановление
   - Подробное логирование всех операций

### Восстановление из резервной копии

Если что-то пошло не так, вы можете восстановить оригинальные файлы:

```cmd
# Запустите скрипт восстановления
call tools\restore_backups.bat
```

Скрипт найдет последнюю резервную копию и предложит восстановить файлы.

## Логирование

Все операции подробно логируются для отслеживания и отладки.

### Форматы логов

- **Текстовые логи** (`*.log`): Человекочитаемые сообщения о ходе выполнения
- **JSON логи** (`*.json`): Структурированные данные для программной обработки

### Директория логов

Все логи сохраняются в директории `logs/`, которая создается автоматически.

### Уровни логирования

- **INFO**: Основные события и прогресс операций
- **WARNING**: Предупреждения о потенциальных проблемах
- **ERROR**: Ошибки, требующие внимания
- **DEBUG**: Подробная отладочная информация (при необходимости)
