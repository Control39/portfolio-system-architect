## 🔍 Суть подхода

Вместо субъективных оценок («знаю Python на 4/5») — объективные, верифицируемые маркеры:

❌ «Я разбираюсь в Docker»
✅ «Создал Dockerfile для Python-приложения и запустил контейнер»

Маркер — это конкретное, проверяемое действие, подтверждённое артефактом:
- код на GitHub
- скриншот работающего сервиса
- лог выполнения
- ссылка на деплой
