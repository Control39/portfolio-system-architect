# Архитектура компонента IT Compass

## Обзор

IT Compass - это модульная система для отслеживания профессиональных компетенций в IT, состоящая из нескольких слабосвязанных компонентов, обеспечивающих гибкость и расширяемость.

## Архитектурные принципы

### 1. Модульность
- Четкое разделение на функциональные модули
- Минимальные зависимости между модулями
- Возможность независимой разработки и тестирования

### 2. Расширяемость
- Поддержка плагинов для интеграции с внешними API
- Гибкая система маркеров технологий
- Возможность добавления новых типов ресурсов

### 3. Наблюдаемость
- Встроенная система логирования
- Метрики производительности
- Мониторинг состояния системы

### 4. Безопасность
- Защита конфиденциальных данных
- Аутентификация и авторизация
- Шифрование при хранении и передаче данных

## Компоненты системы

### 1. Core Tracker
**Ответственность**: Основная логика отслеживания компетенций

**Компоненты**:
- `TechnologyMarker` - модель технологии
- `UserProgress` - модель прогресса пользователя
- `CompetencyTracker` - основной трекер
- `Recommendation` - модель рекомендаций
- `LearningPath` - модель пути обучения

**Интерфейсы**:
```python
class CompetencyTracker:
    def add_marker(self, marker: TechnologyMarker) -> None
    def record_progress(self, progress: UserProgress) -> None
    def generate_recommendations(self) -> List[Recommendation]
    def get_competency_summary(self) -> Dict[str, Any]
```

### 2. API Integration
**Ответственность**: Интеграция с внешними платформами

**Компоненты**:
- `APIIntegration` - базовый класс интеграции
- `GitHubIntegration` - интеграция с GitHub
- `LearningPlatformIntegration` - интеграция с платформами обучения
- `LinkedInIntegration` - интеграция с LinkedIn
- `IntegrationManager` - менеджер интеграций

**Интерфейсы**:
```python
class APIIntegration:
    def get_user_activities(self, user_id: str) -> List[UserActivity]
    def get_user_resources(self, user_id: str) -> List[ExternalResource]

class IntegrationManager:
    def sync_user_data(self, user_id: str) -> Dict[str, Any]
```

### 3. Mental Support
**Ответственность**: Психологическая поддержка пользователя

**Компоненты**:
- `MoodRecord` - запись психологического состояния
- `MentalSupport` - основная система поддержки
- `SelfHelpPractices` - практики самопомощи
- `ITCompassMentalSupport` - интеграция с основной системой

**Интерфейсы**:
```python
class MentalSupport:
    def record_mood(self, mood_record: MoodRecord) -> None
    def get_current_recommendations(self) -> List[SupportRecommendation]
    def get_mood_summary(self) -> Dict[str, Any]
```

### 4. Portfolio Generator
**Ответственность**: Генерация профессионального портфолио

**Компоненты**:
- `PortfolioSection` - секция портфолио
- `PortfolioGenerator` - генератор портфолио

**Интерфейсы**:
```python
class PortfolioGenerator:
    def add_section(self, section: PortfolioSection) -> None
    def generate_portfolio(self) -> str
```

## Слои архитектуры

### 1. Слой данных (Data Layer)
- Хранение маркеров технологий
- Хранение прогресса пользователя
- Хранение настроек
- Кэширование данных

**Технологии**:
- JSON файлы для простоты развертывания
- Возможность подключения внешних БД

### 2. Слой бизнес-логики (Business Logic Layer)
- Основные алгоритмы отслеживания
- Генерация рекомендаций
- Анализ прогресса
- Психологическая поддержка

**Компоненты**:
- CompetencyTracker
- MentalSupport
- PortfolioGenerator

### 3. Слой интеграции (Integration Layer)
- Внешние API
- Системы мониторинга
- Системы уведомлений

**Компоненты**:
- IntegrationManager
- API интеграции

### 4. Слой представления (Presentation Layer)
- Веб-интерфейс
- CLI интерфейс
- API для внешних систем

**Компоненты**:
- Web UI (упрощенный)
- REST API (в разработке)

## Потоки данных

### 1. Запись прогресса
```
User -> UI/API -> CompetencyTracker -> Data Layer
```

### 2. Генерация рекомендаций
```
CompetencyTracker -> Analysis Engine -> Recommendation Engine -> User
```

### 3. Синхронизация с внешними API
```
Scheduler -> IntegrationManager -> External APIs -> Data Layer -> CompetencyTracker
```

### 4. Психологическая поддержка
```
User -> MentalSupport -> Analysis -> Recommendations -> User
```
