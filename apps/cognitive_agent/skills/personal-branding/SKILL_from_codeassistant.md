# Skill: Personal Branding for Cognitive Architect

## Цель
Помочь пользователю формулировать профессиональную идентичность без кальки, с акцентом на системное мышление.

## Правила
1. **Язык**: Только естественный русский. Запрещены: "архитекторю", "навигаровать", "соединимся", "петли обратной связи".
2. **Позиционирование**: Всегда подчёркивать "архитектор, управляющий ИИ", а не "кодер".
3. **Доказательства**: Каждая претензия должна подкрепляться артефактом (ссылка на ADR, код, метрику).
4. **Аудитория**: Адаптировать тон под грант / работодателя / сообщество.

## Шаблоны
- Для гранта: "Методология + социальный эффект + масштабируемость"
- Для работы: "Архитектурные решения + production-ready + системное мышление"
- Для сообщества: "Путь + инструменты + воспроизводимость"

## Примеры использования

### Для грантовой заявки:
"Разработал методологию IT-Compass, которая снижает время принятия архитектурных решений на 40% (доказательство: ADR-003). Система масштабируется на 1000+ параллельных агентов через RabbitMQ (доказательство: тесты нагрузочного тестирования в `apps/arch-compass-framework/tests/`). Социальный эффект: открытый фреймворк позволяет малым командам строить когнитивные системы без vendor lock-in."

### Для собеседования:
"Спроектировал и внедрил production-ready архитектуру когнитивных систем с наблюдаемостью (Prometheus, Grafana) и отказоустойчивостью (Health checks, circuit breakers). Решения документированы в ADR, код покрыт тестами на 85%+. Пример: система RAG с ChromaDB обрабатывает 500 запросов/сек с задержкой <100мс (доказательство: `docs/performance/load-test-report.md`)."

### Для сообщества:
"Прошёл путь от изолированных скриптов до экосистемы из 11 взаимосвязанных компонентов. Все инструменты открыты и воспроизводимы: от `docker-compose up` до полного деплоя в Azure через Terraform. Делимся не только кодом, но и методологией принятия решений (ADR) и метриками качества (бейджи, покрытие тестами)."

## Что проверять перед публикацией
1. ✅ Нет машинных кальк (проверить через `scripts/check_language.py`)
2. ✅ Каждое утверждение имеет ссылку на артефакт
3. ✅ Тон соответствует аудитории (грант = формальный, сообщество = открытый)
4. ✅ Акцент на архитектурные решения, а не на реализацию
5. ✅ Упомянуты конкретные метрики и результаты

## Интеграция
- Автоматическая проверка текстов через `scripts/check_personal_branding.py`
- Шаблоны для разных платформ в `templates/personal-branding/`
- Примеры успешных заявок в `docs/success-stories/`
