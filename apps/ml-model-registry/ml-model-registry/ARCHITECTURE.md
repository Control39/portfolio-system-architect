# Архитектура ML Model Registry

## Обзор

ML Model Registry - это система для управления жизненным циклом моделей машинного обучения. Она предоставляет API для регистрации, версионирования, хранения метаданных и развертывания ML-моделей.

## Архитектурные компоненты

### 1. API Layer
- RESTful API для взаимодействия с внешними системами
- GraphQL API для сложных запросов к метаданным
- Аутентификация и авторизация

### 2. Model Storage
- Хранилище моделей (локальное или облачное)
- Поддержка различных форматов (TensorFlow, PyTorch, ONNX)
- Система версионирования моделей

### 3. Metadata Store
- Хранение метаданных моделей (метрики, параметры, теги)
- История обучения и развертывания
- Реестр экспериментов

### 4. Model Serving
- Интеграция с платформами serving'а (Seldon, KServe, MLflow)
- Мониторинг производительности моделей
- A/B тестирование

## Технологический стек

- **Backend**: Python/FastAPI
- **База данных**: PostgreSQL для метаданных
- **Хранилище**: MinIO/S3 для артефактов моделей
- **Контейнеризация**: Docker, Kubernetes
- **Мониторинг**: Prometheus, Grafana

## Потоки данных

1. Регистрация модели → Сохранение артефактов → Сохранение метаданных
2. Запрос модели → Поиск метаданных → Извлечение артефактов → Serving
3. Мониторинг → Сбор метрик → Обновление метаданных

## Безопасность

- Аутентификация через JWT
- RBAC для управления доступом
- Шифрование данных в покое и в передаче
