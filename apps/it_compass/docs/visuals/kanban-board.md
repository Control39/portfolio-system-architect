## Процесс работы

### 1. Добавление новых задач
1. Создайте issue в GitHub с тегом `it-compass`
2. Добавьте задачу в соответствующий раздел канбана
3. Укажите приоритет, сложность и оценку времени

### 2. Перемещение задач
- **To Do → In Progress**: Когда начинаете работу над задачей
- **In Progress → Review**: Когда задача готова для проверки
- **Review → Done**: После успешной проверки
- **Done → Released**: После выпуска в production

### 3. Приоритеты
- **Высокий**: Критически важные для работы системы
- **Средний**: Важные улучшения и новые функции
- **Низкий**: "Nice to have" и долгосрочные улучшения

### 4. Сложность
- **Низкая**: 1-2 часа работы
- **Средняя**: 3-8 часов работы
- **Высокая**: 8+ часов работы, возможно разделение на подзадачи
