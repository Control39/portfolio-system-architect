```
┌─────────────────────────────────────────────────────────────┐
│                    МЕТОДОЛОГИЯ                              │
│           ОБЪЕКТИВНЫЕ МАРКЕРЫ КОМПЕТЕНЦИЙ                   │

                            │
                            ▼

```

## Структура маркера

```
┌─────────────────────────────────────────────────────────────┐
│                      МАРКЕР КОМПЕТЕНЦИИ                     │
├─────────────────────────────────────────────────────────────┤
│  ID: python_1_1                                            │
│  Маркер: "Написал скрипт для автоматизации рутинной задачи" │
│  Валидация: "Код выложен на GitHub с описанием"            │
│  Приоритет: high                                           │
├─────────────────────────────────────────────────────────────┤
│                    SMART КРИТЕРИИ                           │
├─────────────────────────────────────────────────────────────┤
│  • Specific: Написать скрипт для автоматизации             │
│  • Measurable: Скрипт выполняет конкретную задачу          │
│  • Achievable: Уровень начинающего                         │
│  • Relevant: Требуется для автоматизации процессов         │
│  • Time-bound: 2-3 часа                                    │
└─────────────────────────────────────────────────────────────┘
```

## Уровни компетенций

```
УРОВЕНЬ 1: НАЧИНАЮЩИЙ (FOUNDATION)
┌─────────────────────────────────────────────────────────────┐
│  • Базовые задачи                                           │
│  • Четкие инструкции                                        │
│  • Простые скрипты                                          │
│  • Пример: "Написать скрипт для автоматизации"              │
└─────────────────────────────────────────────────────────────┘

УРОВЕНЬ 2: РАЗВИВАЮЩИЙСЯ (INTERMEDIATE)
┌─────────────────────────────────────────────────────────────┐
│  • Комплексные задачи                                       │
│  • Самостоятельное решение                                  │
│  • Интеграции                                               │
│  • Пример: "Создать микросервис с API"                      │
└─────────────────────────────────────────────────────────────┘

УРОВЕНЬ 3: ПРОДВИНУТЫЙ (ADVANCED)
┌─────────────────────────────────────────────────────────────┐
│  • Архитектурные решения                                    │
│  • Оптимизация производительности                           │
│  • Менторство                                               │
│  • Пример: "Спроектировать систему высокой доступности"     │
└─────────────────────────────────────────────────────────────┘

УРОВЕНЬ 4: ЭКСПЕРТ (EXPERT)
┌─────────────────────────────────────────────────────────────┐
│  • Инновационные решения                                    │
│  • Стратегическое планирование                              │
│  • Создание методологий                                     │
│  • Пример: "Разработать новую архитектурную парадигму"      │
└─────────────────────────────────────────────────────────────┘

УРОВЕНЬ 5: ЛИДЕР (LEADER)
┌─────────────────────────────────────────────────────────────┐
│  • Трансформация индустрии                                  │
│  • Создание экосистем                                       │
│  • Влияние на сообщество                                    │
│  • Пример: "Основать успешный open-source проект"           │
└─────────────────────────────────────────────────────────────┘
```

## Процесс валидации

```
┌─────────────────────────────────────────────────────────────┐
│                ПРОЦЕСС ВАЛИДАЦИИ МАРКЕРА                    │
├─────────────────────────────────────────────────────────────┤
│  1. ВЫПОЛНЕНИЕ: Пользователь выполняет задачу              │
│  2. ДОКАЗАТЕЛЬСТВО: Создает артефакт (код, документ)       │
│  3. РАЗМЕЩЕНИЕ: Выкладывает на GitHub/GitLab с описанием    │
│  4. ВАЛИДАЦИЯ: Система проверяет соответствие критериям    │
│  5. ЗАЧЕТ: Маркер отмечается как выполненный               │
│  6. УРОВЕНЬ: Проверяется достижение нового уровня          │
└─────────────────────────────────────────────────────────────┘
```

## Примеры маркеров по категориям

### Python разработка
- **Уровень 1**: Написать скрипт для автоматизации рутинной задачи
- **Уровень 2**: Создать REST API с использованием FastAPI
- **Уровень 3**: Оптимизировать производительность приложения
- **Уровень 4**: Разработать библиотеку для решения сложной проблемы
- **Уровень 5**: Создать фреймворк, используемый сообществом

### DevOps практики
- **Уровень 1**: Настроить простой CI/CD пайплайн
- **Уровень 2**: Реализовать инфраструктуру как код (Terraform)
- **Уровень 3**: Спроектировать систему мониторинга и алертинга
- **Уровень 4**: Оптимизировать стоимость и производительность cloud
- **Уровень 5**: Создать DevOps платформу для компании

### Системное мышление
- **Уровень 1**: Проанализировать простую систему и выявить зависимости
- **Уровень 2**: Спроектировать архитектуру микросервиса
- **Уровень 3**: Оптимизировать взаимодействие между компонентами системы
- **Уровень 4**: Разработать стратегию масштабирования сложной системы
- **Уровень 5**: Создать архитектурную парадигму для новой области

## Как использовать маркеры

1. **Выберите категорию**, соответствующую вашим целям
2. **Начните с уровня 1**, даже если у вас есть опыт
3. **Выполняйте маркеры последовательно**, создавая доказательную базу
4. **Используйте артефакты** для создания портфолио
5. **Переходите на следующий уровень** после выполнения всех маркеров текущего уровня
