## 📐 Детальная архитектура компонентов

### 1. **DATA LAYER (Слой данных)**

#### 1.1 Маркеры компетенций
- **Расположение:** `src/data/markers/*.json`
- **Количество направлений:** 17
- **Формат:** JSON с иерархической структурой (skill → levels → markers)
- **Структура маркера:**
  ```json
  {
    "id": "уникальный_идентификатор",
    "marker": "конкретное действие",
    "validation": "критерий проверки",
    "priority": "high/medium/low",
    "resources": ["ссылки"],
    "smart_criteria": {
      "specific": "...",
      "measurable": "...",
      "achievable": "...",
      "relevant": "...",
      "time_bound": "..."
    }
  }
  ```

**Направления (17 файлов):**
1. Python (3 уровня)
2. Docker (2 уровня)
3. Data Analysis (2 уровня)
4. Frontend (2 уровня)
5. System Design (2 уровня)
6. Database (2 уровня)
7. Product Management (2 уровня)
8. Cloud Computing (2 уровня)
9. Cybersecurity (2 уровня)
10. Mobile Development (2 уровня)
11. AI Applications (2 уровня)
12. Git (2 уровня)
13. Linux (2 уровня)
14. Business Analysis (2 уровня)
15. DevOps (3 уровня)
16. QA (2 уровня)
17. Communication (2 уровня)

**Оценка:** ⭐⭐⭐⭐⭐
- Структурированный, масштабируемый формат
- SMART-критерии интегрированы в каждый маркер
- Чёткая иерархия (skill → level → marker)
- Поддерживает метаданные (автор, лицензия)

#### 1.2 Прогресс пользователя
- **Файл:** `src/data/user_progress.json`
- **Структура:**
  ```json
  {
    "completed_markers": ["marker_id_1", "marker_id_2"],
    "in_progress_markers": ["marker_id_3"]
  }
  ```
- **Текущее состояние:** пустой (новый пользователь)

**Оценка:** ⭐⭐⭐⭐
- Простая, эффективная структура
- Минималистичный подход (только необходимое)
- Легко расширяется

### 2. **BUSINESS LOGIC LAYER (Бизнес-логика)**

#### 2.1 Core Tracker (`src/core/tracker.py`)
**Класс:** `CareerTracker`

**Основные функции:**
- `_load_all_markers()` — загрузка маркеров из JSON
- `_load_progress()` — загрузка прогресса пользователя
- `_save_progress()` — сохранение прогресса
- `show_progress()` — отображение прогресса с прогресс-барами
- `mark_completed(marker_id)` — отметка маркера как выполненного
- `show_recommendations()` — рекомендации high-priority маркеров
- `get_skill_progress(skill_name)` — детальная статистика по навыку

**Архитектурные решения:**
- Использует `dataclass` для типизации (`Marker`, `SkillData`)
- Логирование через `logging`
- Обработка ошибок на всех уровнях
- Кэширование маркеров в памяти

**Оценка:** ⭐⭐⭐⭐⭐
- Чистая архитектура
- Хорошая обработка ошибок
- Типизация через dataclass
- Разделение ответственности

#### 2.2 Portfolio Generator (`src/utils/portfolio_gen.py`)
**Класс:** `PortfolioGenerator`

**Функции:**
- `generate_portfolio()` — основная функция генерации
- `_load_progress()` — загрузка прогресса
- `_load_all_markers()` — загрузка маркеров с кэшированием
- `_create_portfolio_content()` — формирование Markdown-контента
- `_group_markers_by_skill()` — группировка по навыкам
- `_save_portfolio()` — сохранение в `docs/my_portfolio.md`

**Формат вывода:** Markdown с:
- Заголовками по навыкам
- Списками выполненных маркеров
- Валидационными критериями
- Лицензионной информацией

**Оценка:** ⭐⭐⭐⭐
- Автоматическая генерация портфолио
- Группировка по навыкам
- Сохранение лицензионной информации
- Можно улучшить: шаблонизация, экспорт в другие форматы

### 4. **INTEGRATION LAYER (Интеграции)**

#### 4.1 Reasoning Integration (`scripts/reasoning_integration.py`)
**Класс:** `ReasoningIntegrator`

**Функционал:**
- Анализ заметок через LLM API (OpenAI-совместимое)
- Автоматическое сопоставление действий с маркерами
- Обработка файлов и директорий
- Применение найденных совпадений к трекеру

**Архитектура:**
```
Заметки/диалоги → Reasoning API → JSON совпадений → CareerTracker
```

**Особенности:**
- Поддержка симуляции (без API-ключа)
- Извлечение JSON из ответов LLM
- Фильтрация по уверенности (confidence)
- Автоматическое обновление портфолио

**Оценка:** ⭐⭐⭐⭐
- Мощная интеграция с AI
- Автоматизация анализа
- Fallback на симуляцию
- Можно улучшить: батч-обработка, кэширование API-запросов
