# Security Policy

## Supported Versions

Only the latest major version of Cloud-Reason receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately via the repository's security advisory feature (GitHub) or contact the maintainers directly.

Do not disclose the vulnerability publicly until it has been addressed.

## Security Measures

- All dependencies are regularly scanned for known vulnerabilities.
- Secrets are managed via Yandex Cloud Lockbox.
- API keys are never hardcoded in the source code.
- Access control is enforced via IAM roles.
