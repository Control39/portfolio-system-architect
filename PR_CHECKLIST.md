# Чек-лист для Pull Request

Используйте этот чек-лист перед созданием PR для саморевью или проверки чужого PR.

---

## Основные проверки

### Работоспособность
- [ ] Код работает как ожидается
- [ ] Все тесты проходят локально (`python -m pytest tests/`)
- [ ] Тесты добавлены/обновлены для нового функционала

### Стандарты кода
- [ ] Соблюдён PEP 8 (для Python)
- [ ] Использованы type hints
- [ ] Написаны docstrings для новых функций
- [ ] Нет «магических чисел» — используйте константы
- [ ] Код обрабатывает ошибки (try/except где нужно)
- [ ] Используется логирование для важных операций

### Безопасность
- [ ] Нет уязвимостей (SQL injection, XSS и т.д.)
- [ ] Секреты не захардкожены (используйте .env)
- [ ] Проверено на уязвимости (`gitleaks` или аналог)

### Документация
- [ ] Обновлён README.md компонента (если изменён компонент)
- [ ] Добавлены комментарии для сложной логики
- [ ] Обновлена документация в `docs/` при необходимости

### Чистота кода
- [ ] Удалён отладочный код (print, console.log)
- [ ] Удалены закомментированные участки кода
- [ ] Нет временных файлов в коммите

### Git
- [ ] Коммиты имеют понятные сообщения
- [ ] Ветка обновлена до latest main
- [ ] Нет конфликтов с main

---

## Тип изменений: Документация

- [ ] Проверены ссылки (внутренние и внешние)
- [ ] Использован правильный формат Markdown
- [ ] Нет орфографических ошибок
- [ ] Структура соответствует шаблону

---

## Тип изменений: Новый компонент

- [ ] Компонент имеет структуру `src/`, `tests/`, `docs/`, `README.md`
- [ ] Добавлен в основную документацию
- [ ] Зависимости указаны в requirements.txt
- [ ] Компонент самодостаточен (не зависит от внешних файлов)

---

## Тип изменений: Рефакторинг

- [ ] Функциональность сохранена после рефакторинга
- [ ] Все тесты проходят после изменений
- [ ] Изменения минимальны и целенаправленны
- [ ] Причины рефакторинга задокументированы

---

## Тип изменений: Исправление бага

- [ ] Воспроизведён баг локально
- [ ] Найдена корневая причина
- [ ] Исправление решает проблему
- [ ] Добавлен тест, предотвращающий регресс

---

## Сложные изменения (требуют дополнительного внимания)

Если изменения затрагивают несколько компонентов или архитектуру:

- [ ] Обновлена диаграмма зависимостей (если есть)
- [ ] Изменения согласованы с другими компонентами
- [ ] Проведено интеграционное тестирование
- [ ] Обновлена документация API (если есть)

---

## После создания PR

- [ ] Заполнен шаблон описания PR
- [ ] Указаны затронутые компоненты
- [ ] Прикреплены скриншоты/логи (если визуальные изменения)
- [ ] Связаны соответствующие issue

---

## Для ревьюера

- [ ] Проверен код на безопасность
- [ ] Проверена документация
- [ ] Запущены тесты
- [ ] Даны конструктивные комментарии
- [ ] Проверено соответствие стандартам проекта
