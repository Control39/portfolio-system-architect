# Следующие шаги

*Этот файл будет заполнен в рамках развития проекта. Здесь будет план развития системы управления знаниями.*

## Краткосрочные задачи (1‑2 недели)
1. **Рефакторинг core‑модулей** — объединить `mental_support.py` и `low_energy_mode.py` в единый модуль психологической поддержки.
2. **Тестирование** — достичь покрытия кода не менее 70% для модулей психологической поддержки и CLI.
3. **Интеграция с IT‑Compass** — подключить систему к портфолио‑организатору.
4. **Документация** — перенести методологическую документацию в `02_METHODOLOGY/`, оставив в `components/` только техническую.

## Среднесрочные задачи (1‑2 месяца)
1. **Веб‑интерфейс** — разработать простой UI на Streamlit для визуализации прогресса.
2. **Расширение базы маркеров** — добавить маркеры для новых технологий (Kubernetes, GraphQL, etc.).
3. **Персонализация рекомендаций** — внедрить алгоритм машинного обучения для подбора материалов.
4. **Интеграция с внешними сервисами** — подключение к GitHub, LinkedIn для автоматического сбора достижений.

## Долгосрочное видение (3‑6 месяцев)
1. **Мобильное приложение** — версия для iOS/Android с уведомлениями и мотивационными push‑сообщениями.
2. **Комьюнити‑функции** — возможность делиться прогрессом, создавать группы обучения.
3. **Монетизация** — премиум‑функции для корпоративных клиентов.
4. **Научная публикация** — описание методологии и результатов в рецензируемом журнале.

## Зависимости
- Успешное прохождение гранта SourceCraft.
- Обратная связь от ранних пользователей.
- Технические ресурсы (хостинг, CI/CD).
