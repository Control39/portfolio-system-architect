# Архитектура системы управления знаниями

*Этот файл будет заполнен в рамках развития проекта. Здесь будет описана архитектура, компоненты, взаимодействия и технологический стек.*

## Обзор архитектуры
Система строится по модульному принципу с чётким разделением ответственности.

### Компоненты
1. **Core‑модуль** — управление маркерами, прогрессом, рекомендациями.
2. **Data‑слой** — хранение JSON‑данных (маркеры, прогресс пользователя).
3. **UI‑слой** — CLI и веб‑интерфейс.
4. **Интеграционный слой** — подключение к внешним API (например, психологической поддержки).

### Технологический стек
- **Backend**: Python 3.9+
- **Фреймворки**: FastAPI (для будущего REST API)
- **Данные**: JSON (переход на SQLite при росте)
- **Интерфейс**: Click (CLI), Streamlit (веб‑прототип)
- **Тестирование**: pytest, coverage

*Будет добавлена диаграмма компонентов.*

## Принципы проектирования
- **Расширяемость**: возможность добавления новых маркеров без изменения кода.
- **Модульность**: каждый компонент независим и заменяем.
- **Простота**: минимум зависимостей, понятная логика.

## Связь с IT‑Compass
