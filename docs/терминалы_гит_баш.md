Разберу, как настроить и использовать Git Bash в Visual Studio Code (VS Code), а также дам нужные команды.

## Как настроить Git Bash в VS Code


**Способ 1. Через интерфейс VS Code (простой)**

1. Откройте VS Code.
2. Нажмите `Ctrl + `` (обратный апостроф)` — откроется терминал.
3. Нажмите `Ctrl + Shift + P`, чтобы открыть **Command Palette**.
4. Введите `Terminal: Select Default Profile`.
5. Выберите **Git Bash** из списка.
6. Нажмите на значок `+` в панели терминала — откроется новая вкладка с Git Bash.


**Способ 2. Через настройки JSON (ручной)**


1. Откройте настройки VS Code: `File → Preferences → Settings` или `Ctrl + ,`.
2. Нажмите на иконку `{ }` (Open Settings JSON) в верхнем правом углу.
3. Добавьте в `settings.json` следующие строки:
```json
{
  "terminal.integrated.profiles.windows": {
    "Git Bash": {
      "path": "C:\\Program Files\\Git\\bin\\bash.exe"
    }
  },
  "terminal.integrated.defaultProfile.windows": "Git Bash"
}
```
4. Сохраните файл (`Ctrl + S`) и перезапустите VS Code.

**Важно:** если Git установлен в другую папку, измените путь в `path`. Например:
* для 32‑битной версии: `"C:\\Program Files (x86)\\Git\\bin\\bash.exe"`;
* если вы не уверены в пути, найдите `bash.exe` в папке установки Git.

## Базовые команды для Git Bash в VS Code

После настройки используйте эти команды прямо в терминале VS Code:

### Работа с Git

* `git status` — показать статус репозитория (изменённые, новые, удалённые файлы);
* `git add .` — добавить все изменённые файлы в индекс;
* `git commit -m "Ваше сообщение"` — зафиксировать изменения с комментарием;
* `git log` — посмотреть историю коммитов;
* `git branch` — показать существующие ветки;
* `git checkout <ветка>` — переключиться на указанную ветку;
* `git pull` — получить последние изменения из удалённого репозитория;
* `git push` — отправить локальные коммиты в удалённый репозиторий.


### Навигация и работа с файлами
* `ls` — показать содержимое текущей папки;
* `cd <папка>` — перейти в указанную папку;
* `pwd` — показать полный путь к текущей папке;
* `mkdir <имя_папки>` — создать новую папку;
* `touch <файл>` — создать новый пустой файл;
* `cp <исходный> <целевой>` — скопировать файл;
* `mv <старое_имя> <новое_имя>` — переименовать или переместить файл;
* `rm <файл>` — удалить файл.

### Управление Python‑окружением
* `python --version` — проверить версию Python;
* `pip list` — показать установленные пакеты;
* `pip install <пакет>` — установить пакет;
* `pip freeze > requirements.txt` — сохранить список зависимостей в файл;
* `.venv/Scripts/activate` — активировать виртуальное окружение (Windows);
* `deactivate` — деактивировать виртуальное окружение.

### Проверка утилит
* `git --version` — убедиться, что Git доступен;
* `uname -a` — показать информацию о системе;
* `sed --version` — проверить доступность `sed`.

## Что делать, если команды не работают

Если вы видите ошибку `command not found`, выполните следующие шаги:

1. **Проверьте установку Git:**
   * откройте **CMD** или **PowerShell**;
   * введите `git --version`;
   * если команда не выполняется, переустановите Git с официального сайта (https://git-scm.com/download/win), выбрав опцию **«Use Git from the Windows Command Prompt»**.

2. **Убедитесь, что путь к Git Bash указан верно** в `settings.json`.
3. **Перезапустите VS Code** после внесения изменений.
4. **Откройте новый терминал** (`Ctrl + ``), чтобы применить настройки.

## Проверка работоспособности после настройки

Выполните в терминале VS Code:
```bash
git status
uname -s
echo "test" | sed 's/test/hello/'
```

**Ожидаемые результаты:**
* `git status` — вывод статуса репозитория или сообщение об отсутствии репозитория;
* `uname -s` — `MINGW64_NT‑10.0` (или аналогичное);
* `sed` — вывод `hello`.

## Полезные советы

* **Горячие клавиши:**
   * `Ctrl + `` — открыть/закрыть терминал;
   * `Ctrl + Shift + `` — создать новую вкладку терминала;
   * `Ctrl + C` — прервать выполнение команды.
* **Автодополнение:** используйте `Tab` для автодополнения имён файлов и команд.
* **История команд:** нажимайте `↑`/`↓` для просмотра предыдущих команд.
* **Копирование/вставка:**
   * в Git Bash: `Shift + Insert` для вставки;
   * в Windows Terminal: `Ctrl + V`.

Если хотите, могу подробнее разобрать какую‑то конкретную команду или помочь с настройкой определённого инструмента!
