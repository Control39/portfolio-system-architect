# Анализ стратегии Cognitive Agent: Вынос в отдельный репозиторий

**Дата анализа:** 14 мая 2026 г.  
**Тип документа:** Стратегическое решение  
**Статус:** Завершено  

---

## 📋 Резюме

Обсуждение целесообразности выноса компонента `cognitive-agent` из monorepo `portfolio-system-architect` в отдельный публичный репозиторий для привлечения внимания сообщества.

---

## 🎯 Контекст

### Текущее состояние
- **Расположение:** `apps/cognitive-agent/` в monorepo с 14 микросервисами
- **Проблема:** Компонент «затерялся» среди других сервисов, не получает должного внимания
- **Цель:** Дать сообществу доступ к автономному ИИ-агенту с функциями самообучения

### Архитектурная роль
`cognitive-agent` — автономный оркестратор, который:
1. Сканирует проекты и определяет технологический стек
2. Планирует задачи автоматически
3. Выполняет рутинные операции без участия человека
4. Самообучается на основе метрик эффективности

---

## ⚖️ Анализ «ЗА» и «ПРОТИВ»

### ✅ Аргументы ЗА вынос

| Критерий | Обоснование |
|----------|-------------|
| **Видимость** | В monorepo с 14 сервисами агент не заметен HR и разработчикам |
| **Фокус** | Отдельное репо = чёткое позиционирование как «автономный ИИ-агент» |
| **Сообщество** | Потенциал для коллаборации: новые скиллы, интеграции, перенос на другие языки |
| **Репутация** | Позиционирование автора как создателя когнитивной архитектуры, а не просто микросервисов |
| **Упрощение входа** | `pip install cognitive-agent` вместо клонирования 14-сервисного monorepo |

### ⚠️ Риски и проблемы

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| **Низкое качество кода** | Высокая | Агент — skeleton, нет реальных интеграций, тесты валидационные |
| **Отсутствие поддержки** | Высокая | Кто будет поддерживать issues/PR после выноса? |
| **Репутационный ущерб** | Средняя | «Обещали автономность, а код — заглушки» |
| **Фрагментация** | Средняя | Дублирование кода с `codeassistant/` и корневым `src/` |

## 🎯 Рекомендация: Гибридный подход

### Вариант А: Mirror-репозиторий (быстро, безопасно)
```bash
git remote add mirror git@github.com:yourname/cognitive-agent.git
git push mirror main:main
```
**Плюсы:** Нет дублирования, можно откатить  
**Минусы:** Те же проблемы с качеством кода

### Вариант Б: Выделенный репо с MVP (рекомендуется)

#### Этап 1: Подготовка (1-2 недели)
1. Вынести общие зависимости в `packages/portfolio-common/`
2. Реализовать минимум рабочего кода:
   - `project-scanner`: реальный анализ `requirements.txt`, `package.json`
   - `task-planner`: простой планировщик на YAML
   - 3 интеграционных теста с mock-сервисами
3. Создать `README.md` для сообщества с демо-сценарием

#### Этап 2: Создание репозитория
```
cognitive-agent/
├── README.md (фокус на сообщество, быстрый старт за 5 минут)
├── docs/
│   ├── quickstart.md
│   ├── architecture.md
│   └── contributing.md
├── src/
│   ├── scanner.py
│   ├── planner.py
│   └── agent.py
├── tests/
│   ├── test_scanner.py
│   └── test_planner.py
├── examples/
│   └── project-setup-demo/
├── requirements.txt
└── LICENSE (MIT)
```

#### Этап 3: Продвижение
- **Reddit:** r/MachineLearning, r/Python, r/devops
- **Hacker News:** «I built an autonomous project setup agent»
- **LinkedIn/Twitter:** Демонстрация результата («агент настроил проект за 2 минуты»)
