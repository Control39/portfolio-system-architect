﻿# ADR-003: Выбор формата диаграмм (Mermaid)

## Status

Принято

## Context

Для визуализации архитектурных решений, компонентов системы и потоков данных необходимо выбрать
формат диаграмм, который будет:

1. Легко читаем и поддерживаем
1. Интегрируется с системой документации (Markdown)
1. Поддерживает интерактивность
1. Имеет широкую поддержку инструментами
1. Позволяет автоматизировать генерацию

## Considered Options

1. **PlantUML** - мощный инструмент для создания диаграмм UML
1. **Graphviz/DOT** - язык описания графов
1. **Mermaid** - диаграммы в формате Markdown
1. **Draw.io** - визуальный редактор диаграмм
1. **Visio** - Microsoft Visio

## Decision

Выбран Mermaid как основной формат для создания диаграмм по следующим причинам:

1. **Интеграция с Markdown** - диаграммы хранятся прямо в Markdown файлах
1. **Простота использования** - синтаксис близок к естественному описанию диаграмм
1. **Поддержка GitHub** - нативная поддержка на GitHub
1. **Интерактивность** - возможность создания интерактивных диаграмм
1. **Широкая экосистема** - поддержка множеством инструментов и редакторов

## Consequences

### Положительные:

- Единый формат для всех диаграмм в репозитории
- Простота поддержки и обновления диаграмм
- Интеграция с GitHub Pages для интерактивного отображения
- Возможность автоматической генерации диаграмм из кода

### Отрицательные:

- Ограниченная сложность диаграмм по сравнению с PlantUML
- Необходимость изучения синтаксиса Mermaid
- Ограниченная поддержка сложных диаграмм
