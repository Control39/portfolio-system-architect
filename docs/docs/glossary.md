# Глоссарий терминов

> Словарь ключевых терминов проекта «Portfolio System Architect»

## 🔹 Категория 1: Роли и идентичность

### Когнитивный архитектор
Специалист, проектирующий системы мышления и автоматизации с использованием ИИ, сочетающий технические и метакогнитивные навыки.

**См. также:** [about-me.md](../about-me.md), [ARCHITECTURE.md](../ARCHITECTURE.md)

### Системный архитектор
Профессионал, отвечающий за высокоуровневую структуру программных систем и их компонентов.

### Архитектор когнитивных систем
Расширенная роль архитектора, фокусирующаяся на интеграции ИИ в процессы принятия решений.

## 🔹 Категория 2: Методология

### IT-Compass
Система отслеживания компетенций на основе объективных маркеров. Позволяет оценить уровень навыков по 18 категориям маркеров.

**Расположение:** `components/it-compass/`

### Arch-Compass Framework
Архитектурный фреймворк для проектирования систем с ИИ. Предоставляет набор принципов и паттернов для проектирования когнитивных систем.

**Расположение:** `components/arch-compass-framework/`

### Объективный маркер
Измеримый индикатор компетенции. Включает: сертификаты, проекты, тесты, публикации, репозитории.

**См. также:** `docs/methodology/markers/`

### Цикл рассуждений (Reasoning Loop)
Итеративный процесс анализа → синтеза → верификации решений. Используется в RAG-системах для улучшения качества ответов.

### Когнитивный паттерн
Повторяемая структура мышления для решения типовых задач.

**См. также:** `docs/patterns.md`

## 🔹 Категория 3: Технические компоненты

### RAG (Retrieval-Augmented Generation)
Архитектура ИИ, сочетающая поиск в базе знаний с генерацией ответов. Реализована в компоненте Cloud Reason.

**Расположение:** `components/cloud_reason/`

### Эмбеддинг (Embedding)
Векторное представление текста/данных для семантического поиска.

### ML Model Registry
Реестр версий ML-моделей с тестами и метаданными. Включает 11 видов тестов.

**Расположение:** `components/ml-model-registry/`

### Компонент системы
Независимый модуль с определённой функцией. Основные компоненты:
- IT-Compass
- Arch-Compass Framework
- Cloud Reason (RAG)
- ML Model Registry
- Career Development System
- Portfolio Organizer
- System Proof
- Thought Architecture

### Интеграционный слой
Механизм связи между компонентами системы. Обеспечивает единую работу экосистемы.

**См. также:** `docs/integration/`, `docs/architecture/components.md`

## 🔹 Категория 4: Документация и артефакты

### ADR (Architecture Decision Record)
Документ, фиксирующий архитектурное решение и его обоснование. В проекте 7 ADR-документов.

**Расположение:** `docs/adr/`

### Кейс (Case Study)
Описание решённой проблемы с контекстом, инструментами и результатами.

**Расположение:** `cases/`
**Шаблон:** `docs/templates/case-template.md`

### Артефакт
Готовый материал, доказывающий экспертизу: код, документ, презентация.

### Манифест
Декларация принципов и видения профессии/методологии.

## 🔹 Категория 5: Инфраструктура

### SourceCraft
Платформа Yandex для хостинга open-source проектов. Основная платформа для гранта.

**См. также:** [Гранты](grants/)

### GitHub
Платформа для версионирования и коллаборации. Зеркало основного репозитория.

### CI/CD
Автоматизация сборки, тестирования и развёртывания кода. Настроена через GitHub Actions.

**См. также:** `.github/`

### Векторная база
Хранилище эмбеддингов для семантического поиска. Используется в RAG-системе.
