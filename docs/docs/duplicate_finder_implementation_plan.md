## Алгоритм работы

### Этап 1: Сканирование
1. Рекурсивный обход всех директорий от корня проекта
2. Фильтрация по игнорируемым паттернам
3. Сбор списка всех файлов (исключая файлы нулевой длины)

### Этап 2: Хеширование
1. Для каждого файла вычисляется SHA256 хеш содержимого
2. Хеши сохраняются в словаре {file_path: hash}

### Этап 3: Группировка
1. Файлы группируются по хешу
2. Идентифицируются группы с более чем одним файлом (дубликаты)

### Этап 4: Вывод результатов
1. Для каждой группы дубликатов:
   - Название файла (basename)
   - Первые 100 символов содержимого
   - Полный путь к файлу
2. Форматирование в виде таблицы
