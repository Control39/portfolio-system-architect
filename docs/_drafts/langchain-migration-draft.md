# 🔧 Кейс: Миграция langchain 0.3 → 1.x (черновик)

- **Дата создания:** 3 мая 2026 г.
- **Статус:** 🟡 Отложено (техдолг)
- **Контекст:** dev-зависимости, уязвимости CVE-2024-XXXX, брейкинг-чейнджи в API.

## Проблема

При аудите безопасности (`pip-audit`) выявлены 2 известные уязвимости в dev-зависимостях:

| Пакет | Версия | ID уязвимости | Fix версия |
|-------|--------|---------------|------------|
| `langchain-text-splitters` | 0.3.11 | GHSA-fv5p-p927-qmxr | 1.1.2 |
| `langchain-openai` | 0.3.35 | GHSA-r7w7-9xr2-qq2r | 1.1.14 |

**Технический конфликт:**
- Новые версии `langchain-openai>=1.1.14` требуют `langchain-core>=1.3.2`
- Текущий стек: `langchain-core<1.0.0` (осознанный выбор для стабильности)
- Миграция на 1.x требует пересмотра всего кода API

## Решение

**Осознанная отсрочка миграции.**

### Обоснование

1. **Риск принят** — пакеты находятся в `dev-зависимостях`, не используются в production
2. **Стоимость миграции** — требуется рефакторинг всех модулей, использующих langchain
3. **Время на решение** — оценка: 2-3 дня на миграцию + 1 день на регрессионное тестирование
4. **Бизнес-приоритет** — фокус на функциональности, не на обновлении dev-инструментов

### Инфраструктурные шаги

- ✅ Зафиксировано в `pip-audit --ignore` для CI/CD
- ✅ Подготовлен чек-лист регрессии для будущей миграции
- ✅ Скрипт аудита pre-push блокирует опасные изменения в production-коде

## План миграции

1. **Вынести в feature-ветку** — `feature/langchain-1x-migration`
2. **Запустить тесты на совместимость** — проверить breaking changes
3. **Заменить deprecated-вызовы** — обновить импорты и API-вызовы
4. **Обновить документацию** — зафиксировать новые версии и зависимости
5. **Снять ignore** — удалить исключения из `pip-audit` конфигурации

## Ценность для портфолио

Демонстрация:
- Управления техническим долгом (осознанное принятие рисков)
- Системного подхода к безопасности (разделение dev/prod)
- Подготовки миграции без остановки разработки
- Документации решений (ADR-подход)
