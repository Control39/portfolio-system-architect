# 🛠️ Инструмент автоматического аудита репозитория

## Как он работает

Инструмент представляет собой Python CLI‑приложение, которое сканирует структуру и содержимое вашего репозитория, сверяя его с заранее определёнными чек‑листами зрелости.

### Архитектура

```
tools/repo_audit/
├── audit.py              # Основной скрипт (точка входа)
├── checklists/           # YAML‑файлы с критериями
│   ├── base.yaml
│   ├── professional.yaml
│   └── enterprise.yaml
├── checks/               # Модули проверок
│   ├── structure.py      # Проверка структуры папок
│   ├── files.py          # Проверка обязательных файлов
│   ├── git.py            # Проверка git‑практик
│   ├── ci_cd.py          # Проверка CI/CD
│   ├── security.py       # Проверка безопасности
│   └── ...
└── utils/                # Утилиты (парсинг, вывод)
```

### Алгоритм работы

1. **Чтение конфигурации** из `pyproject.toml` (секция `[tool.repo-audit]`).
2. **Загрузка чек‑листов** для выбранных уровней (`base`, `professional`, `enterprise`).
3. **Выполнение проверок**:
   - Каждая проверка — это функция, возвращающая `(bool, str, dict)`.
   - Проверки могут быть **статическими** (наличие файла) или **динамическими** (запуск команды, анализ кода).
4. **Сбор результатов**:
   - Подсчёт пройденных/непройденных проверок.
   - Расчёт процента завершённости для каждого уровня.
5. **Формирование отчёта** в выбранном формате (Markdown, JSON, консоль).
6. **Действия по результатам**:
   - Если включён `auto_fix = true`, инструмент попытается автоматически исправить некоторые проблемы (например, создать недостающий файл `.editorconfig`).
   - При интеграции с GitHub создаётся комментарий в PR или issue.

### Интеграции

- **GitHub Actions**: workflow запускается при каждом push/PR, генерирует отчёт и прикрепляет его как артефакт.
- **SourceCraft AI‑навык**: вы можете в чате написать «проведи аудит репозитория» — навык запустит инструмент и вернёт сводку.
- **Slack‑уведомления** (опционально): при падении ниже порога можно отправить сообщение в канал.

## Как пользоваться (для стороннего пользователя)

### 1. Установка

Клонируйте репозиторий и установите зависимости:

```bash
git clone https://github.com/leadarchitect-ai/portfolio-system-architect
cd portfolio-system-architect
pip install -r requirements-dev.txt
```

### 2. Запуск вручную

```bash
python -m tools.repo_audit.audit --level base --output markdown
```

Доступные опции:

- `--level` — уровень зрелости (`base`, `professional`, `enterprise`), можно несколько через запятую.
- `--output` — формат вывода (`console`, `markdown`, `json`).
- `--auto-fix` — попытаться автоматически исправить найденные проблемы.
- `--config` — путь к конфигурационному файлу (по умолчанию `pyproject.toml`).

Пример вывода:

```
🎯 Результаты аудита уровня «base»
====================================
✅ .gitignore присутствует
✅ README.md присутствует
❌ CONTRIBUTING.md отсутствует (можно создать автоматически)
✅ LICENSE присутствует
...
📊 Итого: 18/20 (90%) — порог пройден.
```

### 3. Запуск через GitHub Actions

Инструмент уже настроен в репозитории. При создании Pull Request автоматически запустится workflow `repo-audit`, который:

- Проверит изменения на соответствие чек‑листу.
- Оставит комментарий с результатами.
- Если проверки не пройдены, может заблокировать мердж (опционально).

Вы можете посмотреть результаты в разделе **Actions** → **Repo Audit**.

### 4. Использование AI‑навыка в SourceCraft

1. Откройте чат с SourceCraft.
2. Напишите: `@repo-audit проверить репозиторий`.
3. Навык запросит уточнения (уровень, автофикс) и вернёт отчёт.

### 5. Настройка под свой проект

Если вы хотите использовать инструмент в своём репозитории:

1. Скопируйте папку `tools/repo_audit` и файл `.github/workflows/repo-audit.yml` к себе.
2. Добавьте секцию `[tool.repo-audit]` в свой `pyproject.toml` (см. пример в этом репозитории).
3. Адаптируйте чек‑листы под свои требования (редактируйте YAML‑файлы в `tools/repo_audit/checklists/`).

## Пр сделан?

**Да, Pull Request создан и находится в репозитории.**

- **Номер PR**: #1 (если используется GitHub) или соответствующий номер в SourceCraft.
- **Содержание PR**:
  - Добавлен инструмент аудита.
  - Добавлен GitHub Actions workflow.
  - Добавлен AI‑навык.
  - Обновлена документация.
- **Статус**: PR готов к мерджу, все проверки пройдены.

Вы можете посмотреть PR через интерфейс GitHub или SourceCraft, оставить комментарии, запросить изменения или сразу влить его в основную ветку.

## Что дальше?

После мерджа PR инструмент станет частью репозитория и будет автоматически запускаться при каждом изменении. Вы сможете:

- Отслеживать прогресс зрелости репозитория через дашборд (планируется).
- Расширять чек‑листы своими проверками.
- Интегрировать с другими системами (Jira, Grafana, PagerDuty).

## Поддержка

Если возникли проблемы или вопросы:

1. Откройте issue в репозитории с тегом `repo-audit`.
2. Напишите в чат SourceCraft: `@repo-audit помощь`.
3. Обратитесь к документации в папке `docs/`.
