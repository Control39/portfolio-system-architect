## 🇷🇺 Российский технологический стек

Проект разработан с использованием **отечественных ИИ-решений** для соответствия требованиям импортозамещения и 152-ФЗ:

| Технология | Производитель | Назначение |
|------------|---------------|------------|
| **GigaChat** | Сбер (Россия) | Основная LLM для генерации кода и анализа |
| **Azure AI** | Microsoft (через локальные узлы) | Резервные модели (опционально) |
| **OpenAI** | OpenAI (резерв) | Для совместимости с международными библиотеками |

**Принцип:** Приоритет — российским решениям (GigaChat), международные — как fallback.

### 2. **GigaChat Agent** (LLM-движок)

**Назначение:** Генерация кода, рефакторинг, анализ архитектуры.

**Библиотека:** `gigachat==0.2.1`
**Конфигурация:** `.sourcecraft/` (skills, CI/CD)

**Интеграция:**
```python
from gigachat import GigaChat

chat = GigaChat(credentials="YOUR_CREDENTIALS")
response = chat.chat("Сгенерируй FastAPI endpoint для ...")
```

**Использованные навыки (skills):**
- Код-генерация (Python/FastAPI)
- Рефакторинг (PowerShell → Python)
- Анализ безопасности (Bandit, Trivy)
- Генерация тестов (pytest)

**Локальные конфиги:**
```
.sourcecraft/
├── skills/              # Навыки для конкретных задач
├── ci.yaml              # CI/CD пайплайны
└── sites.yaml           # Конфигурация деплоя
```

### 3. **Sourcecraft** (оркестратор агентов)

**Назначение:** Координация нескольких ИИ-агентов, управление workflow.

**Инструменты:**
- `.sourcecraft/skills/` — набор навыков
- `legacy/python_server_sourcecraft/` — серверная часть (архив)

**Функции:**

## 🔐 Безопасность и конфиденциальность

### Соответствие 152-ФЗ

- **Хранение данных:** Все данные хранятся на серверах в РФ (Yandex Cloud, локальные серверы)
- **Обработка:** GigaChat обрабатывает данные внутри юрисдикции РФ
- **Шифрование:** Используется `cryptography==48.0.0` (AES-256)

### Секреты и креденшиалы

```bash
# Управление токенами GigaChat
make giga-token          # Получить новый токен
make giga-vscode         # Обновить в VS Code
```

**Никогда не коммитить секреты!**
Используйте `.secrets.baseline` и `trivy-secret.yaml`.

## 🔄 Интеграция с CI/CD

```yaml

- name: Run GigaChat code analysis
  uses: sber-cloud/gigachat-action@v1
  with:
    credentials: ${{ secrets.GIGACHAT_CREDENTIALS }}
    scan-type: 'security'
```

# Перед коммитом
make pre-commit

## 🌐 Локальные vs Облачные модели

| Модель | Расположение | Преимущества | Ограничения |
|--------|--------------|--------------|-------------|
| **GigaChat (локальный)** | РФ (Yandex Cloud) | 152-ФЗ, низкая задержка | Требует ресурсов |
| **GigaChat (облачный)** | SberCloud | Масштабируемость | Зависит от интернета |
| **OpenAI (резерв)** | International | Глобальная доступность | Не 152-ФЗ, задержка |

**Рекомендация:** Использовать локальные/российские модели по умолчанию, международные — только как fallback.
