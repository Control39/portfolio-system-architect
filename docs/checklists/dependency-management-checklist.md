# 🔍 Чек-лист: Управление зависимостями в Python-проекте

> **Внимание!** Этот чек-лист предназначен для обеспечения безопасности, стабильности и воспроизводимости зависимостей в проекте. Перед коммитом и релизом убедитесь, что все пункты выполнены.

**Автор:** Архитектор системы
**Версия:** 1.0
**Для:** Разработчиков, DevOps, архитекторов

## 🎯 Цель

Обеспечить:
* безопасность от уязвимостей в зависимостях;
* воспроизводимость установки на всех окружениях;
* стабильность системы при обновлениях;
* прозрачность управления зависимостями;
* соответствие архитектурным принципам проекта.

## 🧩 Структура управления зависимостями
portfolio-system-architect/
├── requirements.in               # Основные зависимости ("что нужно")
├── requirements.txt             # Полный снимок ("что установлено")
├── requirements-dev.txt         # Зависимости для разработки
├── apps/cognitive-agent/                     # Автономные агенты
│   └── requirements.txt         # Специфические зависимости агентов
└── apps/                        # Модульные приложения
└── */requirements.txt       # Зависимости отдельных приложений


## ✅ Итоговый чек-лист «Готово к коммиту»

Перед коммитом убедитесь, что:

1. [ ] `requirements.txt` **актуален** и сгенерирован из `requirements.in`
2. [ ] **Нет уязвимостей** в зависимостях (проверено `safety`)
3. [ ] `requirements-dev.txt` **содержит все нужные инструменты**
4. [ ] **Нет конфликтов версий** между глобальными и модульными зависимостями
5. [ ] **pre-commit хуки ��астроены** и работают
6. [ ] `Makefile` **содержит команды** для управления зависимостями
7. [ ] Зависимости **устанавливаются и работают** в чистом окружении
8. [ ] **Тесты и линтинг проходят** после установки зависимостей

Если все пункты отмечены — можно коммитить!
