# ADR 1: Использовать Git подмодули для модульности
## 📅 Дата
12 февраля 2026
## 🎯 Статус
Принято
## 🧩 Контекст
Нужно управлять несколькими связанными проектами:
- cloud-reason
- portfolio-organizer
- system-proof
Требуется:
- Независимое развитие
- Централизованное управление
- Чёткая версионность
## 💡 Решение
Использовать Git подмодули в `my-ecosystem` как основной репозиторий-органайзер.
## ✅ Последствия
**Плюсы:**
- Чёткая привязка версий
- Автоматизация через скрипты
- Возможность клониров
