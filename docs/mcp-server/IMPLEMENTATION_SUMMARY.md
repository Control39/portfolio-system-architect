## Архитектурные решения

### MCP-сервер как микросервис
```
apps/mcp-server/
├── src/
│   ├── main.py              # FastMCP сервер
│   ├── tools/               # Инструменты
│   │   ├── file_tools.py    # Работа с файлами
│   │   ├── git_tools.py     # Работа с Git
│   │   ├── compass_tools.py # Интеграция с IT-Compass
│   │   └── command_tools.py # Выполнение команд
│   ├── resources/           # Ресурсы
│   │   ├── navigation.py    # Навигация по репозиторию
│   │   └── compass.py       # Ресурсы IT-Compass
│   └── utils/               # Утилиты
├── config/
│   └── mcp-config.yaml      # Единая конфигурация
├── tests/                   # Тесты
├── scripts/                 # Скрипты
└── docs/                    # Документация
```

## Решенные проблемы

### Проблема 1: Устаревшие конфигурации
**Решение:** Создан скрипт валидации, который:
- Обнаруживает упоминания GPT-5 и других несуществующих моделей
- Автоматически заменяет на актуальные модели (GPT-4, Claude-3)
- Создает резервные копии перед изменениями

### Проблема 2: Избыточная вложенность
**Решение:** Упрощена структура MCP-сервера:
- Максимальная глубина вложенности: 4 уровня
- Логическая группировка файлов
- Четкое разделение ответственности

### Проблема 3: Разбросанные конфигурации
**Решение:** Создана единая конфигурация:
- `apps/mcp-server/config/mcp-config.yaml`
- Централизованное управление настройками
- Поддержка разных окружений (development, production)

### Проблема 4: Блокировки доступа
**Решение:** Документирована проблема и предложены пути решения:
- Настройка `.codeassistantignore`
- Альтернативные методы доступа
- Ручное вмешательство владельца репозитория

## Технические детали

### Технологический стек
- **Язык**: Python 3.11+
- **Фреймворк**: FastMCP (Model Context Protocol)
- **Конфигурация**: YAML + JSON
- **Тестирование**: unittest + pytest
- **Документация**: Markdown + OpenAPI

### Требования к развертыванию
```bash
# Установка зависимостей
cd apps/mcp-server
pip install -r requirements.txt

# Запуск тестов
python -m pytest tests/

# Запуск сервера
python src/main.py

# Валидация конфигураций
python scripts/validate-config.py
```

### Конфигурация для Claude Desktop
```json
{
  "mcpServers": {
    "career-autopilot": {
      "command": "python",
      "args": ["/path/to/apps/mcp-server/src/main.py"],
      "env": {
        "MCP_ENV": "development"
      }
    }
  }
}
```

## Риски и ограничения

### Технические риски
1. **Блокировки доступа**: Некоторые файлы недоступны из-за `.codeassistantignore`
2. **Зависимости**: Требуются специфические версии Python и библиотек
3. **Безопасность**: Выполнение команд требует careful sandboxing

### Организационные риски
1. **Принятие**: Нужно обучать пользователей работе с MCP-сервером
2. **Поддержка**: Требуется документация и примеры использования
3. **Эволюция**: IT-Compass маркеры могут меняться, нужна адаптация

### Ограничения
- Только локальный запуск на первом этапе
- Ограниченная поддержка Windows (ориентация на Linux/macOS)
- Требуется ручная настройка для интеграции с IDE
