# Аудит конфигураций агентов и AI-инструментов
**Дата:** 2026-04-07
**Автор:** SourceCraft Code Assistant Agent
**Цель:** Выявление и устранение устаревших, конфликтующих и избыточных конфигураций агентов

## Обзор проблемы

В репозитории обнаружены следующие проблемы:
1. **Устаревшие конфигурации**: Упоминание GPT-5 (который еще не существует) в конфигурационных файлах
2. **Избыточная вложенность**: Сложная структура директорий с дублирующими настройками
3. **Разбросанные конфигурации**: Настройки агентов находятся в разных местах (`.continue/`, `.codeassistant/`, `settings/`, `apps/ai-config-manager/`)
4. **Конфликты версий**: Разные форматы и версии конфигурационных файлов
5. **Блокировки доступа**: Файлы заблокированы `.codeassistantignore`, что мешает автоматическому аудиту

## Анализ конкретных проблем

### Проблема 1: Устаревшие модели AI
**Факт:** В конфигурации Continue.dev упоминается GPT-5
**Влияние:** Может вызывать ошибки при попытке использования несуществующей модели
**Решение:** Заменить на актуальные модели (GPT-4, Claude-3, локальные модели)

### Проблема 2: Избыточная вложенность
**Факт:** Множество уровней вложенности в директориях
**Пример:** `apps/ai-config-manager/src/core/config/loaders/yaml/`
**Влияние:** Усложняет навигацию и понимание структуры
**Решение:** Упростить структуру, объединить связанные файлы

### Проблема 3: Разбросанные настройки
**Факт:** Конфигурации находятся в 5+ разных местах
**Влияние:** Трудно поддерживать согласованность
**Решение:** Создать единую точку управления конфигурациями

### Проблема 4: Блокировки доступа
**Факт:** Ключевые файлы заблокированы `.codeassistantignore`
**Влияние:** Невозможно автоматически анализировать и исправлять
**Решение:** Настроить `.codeassistantignore` для разрешения доступа к конфигурациям

## План исправлений

### Фаза 1: Настройка доступа (Срочно)
1. Создать `.codeassistantignore` с разрешающими правилами для конфигураций
2. Проверить настройки SourceCraft Code Assistant
3. Получить доступ к заблокированным файлам

### Фаза 2: Очистка конфигураций
1. Прочитать и проанализировать `.continue/agents/new-config.yaml`
2. Удалить упоминания GPT-5 и других несуществующих моделей
3. Обновить конфигурацию на актуальные модели
4. Проверить и обновить другие конфигурационные файлы

### Фаза 3: Упрощение структуры
1. Проанализировать избыточную вложенность
2. Предложить новую структуру директорий
3. Объединить дублирующие конфигурации

### Фаза 4: Создание единой конфигурации
1. Разработать схему единой конфигурации для всех AI-агентов
2. Создать инструмент для синхронизации конфигураций
3. Документировать процесс управления конфигурациями

## Рекомендации по единой конфигурации

Предлагаемая структура:
```
config/
├── ai-agents/
│   ├── continue.yaml          # Continue.dev конфигурация
│   ├── codeassistant.yaml     # Code Assistant конфигурация
│   ├── mcp-servers.yaml       # MCP серверы
│   └── models.yaml           # Доступные AI модели
├── project/
│   ├── project-config.yaml    # Общая конфигурация проекта
│   └── devcontainer.json      # Dev Container настройки
└── tools/
    ├── sync-configs.py        # Инструмент синхронизации
    └── validate-configs.py    # Валидатор конфигураций
```

- **Зависимости**: Конфигурации могут использоваться другими инструментами

5. Документирован процесс обновления конфигураций
