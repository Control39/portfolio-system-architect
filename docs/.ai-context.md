## 🏗️ Архитектурные правила (Обязательно для ИИ-разработчиков)

### 📐 Трёхуровневая архитектура (strict top-down dependencies)

```
УРОВЕНЬ 3: АГЕНТЫ (agents/cognitive_agent/) →
    ↓ использует
УРОВЕНЬ 2: СЕРВИСЫ (apps/*/) →
    ↓ используют
УРОВЕНЬ 1: АТОМЫ (src/)
```

### ✅ Правила импортов (КРИТИЧНО ВАЖНО):

1. **Агенты могут использовать сервисы и атомы**
   ```python
   # ✅ Правильно
   from apps.decision_engine.src.decision_engine import DecisionEngine
   from src.security.rate_limiter import RateLimiter
   ```

2. **Сервисы могут использовать только атомы**
   ```python
   # ✅ Правильно
   from src.ai.config import ConfigManager

   # ❌ НЕЛЬЗЯ - сервис не должен зависеть от другого сервиса
   from apps.job_automation_agent.job_agent import JobAgent
   ```

3. **Атомы не должны зависеть от других уровней**
   ```python
   # ✅ Атомы используют только стандартные библиотеки
   import os
   from typing import Optional
   ```

### 📂 Структура проекта:

```
src/                    # АТОМЫ - переиспользуемые компоненты
├── ai/                 # AI интеграции
├── security/           # Безопасность (rate_limiter, secret_manager, sandbox_executor)
├── common/             # Общие утилиты
├── config/             # Конфигурации
├── core/               # Базовые компоненты
├── interfaces/         # Интерфейсы и API
├── shared/             # Общие модели
└── vector_store/       # Векторное хранилище

apps/                   # СЕРВИСЫ - микросервисы
├── decision_engine/
├── job_automation_agent/
├── knowledge_graph/
├── it_compass/
├── auth_service/
└── ... (20+ сервисов)

agents/cognitive_agent/ # АГЕНТЫ - когнитивные системы
```

### 🚫 Запрещённые зависимости:

- **Services → Services** (например, `apps/decision_engine` → `apps/job_automation_agent`)
- **Atoms → Services** (например, `src/security` → `apps/decision_engine`)
- **Atoms → Agents** (например, `src/ai/config` → `agents/cognitive_agent`)

### 🔒 Безопасность:

- Все секреты должны проходить через `src/security/`
- Все path operations должны использовать `src/security/secure_path.py`
- Все rate limiting через `src/security/rate_limiter.py`
- Все sandbox execution через `src/security/sandbox_executor.py`

### 📝 Документация:

- Каждый сервис имеет `README.md` в корне
- Каждый модуль имеет docstrings
- Архитектурные решения описываются в `docs/architecture/`

---
*Помни: эти правила не рекомендации, а обязательные условия. Нарушение архитектуры приведёт к техническому долгу и сломанной масштабируемости.*
