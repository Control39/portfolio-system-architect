# ADR-005: Выбор технологии для пользовательского интерфейса

## Статус
Принято

## Контекст
Необходимо выбрать технологию для реализации пользовательского интерфейса в IT Compass. Интерфейс должен обеспечивать отображение прогресса, рекомендаций, генерацию портфолио и визуализацию данных. Выбор влияет на скорость разработки, кастомизацию и пользовательский опыт.

## Рассмотренные варианты
1. **Streamlit** - текущая реализация
2. **React + Flask/FastAPI** - полноценное веб-приложение
3. **Vue.js + Express** - альтернативный фреймворк
4. **Angular + Node.js** - корпоративное решение
5. **Desktop приложение** (Electron/PyQt)

## Решение
Выбран Streamlit для реализации пользовательского интерфейса по следующим причинам:

### Преимущества Streamlit:
- **Скорость прототипирования**: Быстрая разработка интерфейсов
- **Интеграция с Python**: Прямая интеграция с бизнес-логикой
- **Data Science ориентированность**: Отлично подходит для визуализации данных
- **Минимализм**: Не требует отдельного фронтенд-разработчика
- **Демонстрационность**: Идеален для показа на собеседованиях

### Компромиссы:
- **Кастомизация**: Ограниченные возможности кастомизации
- **Производительность**: Может быть медленнее полноценных веб-приложений
- **Масштабируемость**: Не подходит для высоконагруженных приложений

## Последствия
### Положительные:
- Быстрая итерация при разработке интерфейса
- Простая интеграция с существующей Python-бизнес логикой
- Отличное визуальное представление данных для демонстрации
- Минимальные требования к инфраструктуре

### Отрицательные:
- Ограниченные возможности кастомизации внешнего вида
- Зависимость от Streamlit runtime
- Не подходит для сложных интерактивных приложений

## Альтернативы для будущего:
- При необходимости более сложного интерфейса рассмотреть React + FastAPI
- Для desktop-версии рассмотреть Electron или PyQt
