## 🔍 Как проверить результат

1. **Запушить изменения:**
   ```powershell
   git add .trivyignore .trivyignorefs .github/workflows/security-scan.yml
   git commit -F commit_msg.txt  # (создай файл с сообщением)
   git push origin main
   ```

2. **Проверить GitHub Actions:**
   - Открой: `https://github.com/Control39/portfolio-system-architect/actions/workflows/security-scan.yml`
   - Дождись завершения `trivy-fs-scan` (5-10 минут)

3. **Проверить Security tab:**
   - Открой: `https://github.com/Control39/portfolio-system-architect/security/advisories`
   - Сравните количество alerts до и после

4. **Если осталось много alerts:**
   - Открой каждый alert → проверь, действительно ли это production-уязвимость
   - Если dev-only → отметь как "Won't Fix" с комментарием
   - Если false positive → добавь в `.trivyignore`

## 📞 Если что-то пойдёт не так

**Проблема:** Trivy всё равно сканирует игнорируемые папки  
**Решение:** Проверь, что `.trivyignorefs` в корне и формат правильный (по одному пути на строку)

**Проблема:** Alerts не уменьшились после пуша  
**Решение:**  
1. Проверь, что workflow запустился (Actions tab)
2. Проверь логи Trivy в workflow (выбери job → "Run Trivy vulnerability scanner")
3. Убедись, что `skip-dirs` правильно указан (через запятую, без пробелов)

**Проблема:** Остались Critical уязвимости в production коде  
**Решение:** Срочно обновить зависимости или изолировать уязвимый код
