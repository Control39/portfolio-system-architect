# docs/

Документация проекта.

## Назначение
- Централизованное хранение всей документации проекта
- Обеспечение доступности информации для всех участников
- Поддержка знаний в актуальном состоянии

## Структура
- `adr/` - Архитектурные Решения (Architecture Decision Records)
- `api/` - Документация API
- `architecture/` - Архитектурные диаграммы и описания
- `cases/` - Кейсы использования, примеры и исследования
- `methodology/` - Методология разработки и подходы
- `reports/` - Отчеты о состоянии проекта, аудитах и анализах
- `security/` - Документация по безопасности

Этот каталог является основным источником истины для всей документации проекта, обеспечивая прозрачность решений и упрощая онбординг новых участников.
