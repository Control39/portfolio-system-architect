# План: Вынос 5 продуктов из репозитория

## Цель
Выделить 5 ключевых компонентов из монорепозитория `portfolio-system-architect` в отдельные, независимые open-source проекты для максимального раскрытия их коммерческого и сообщественного потенциала.

## Продукты
1.  **`ai-config-manager`** (Менеджер конфигураций ИИ) - Главный кандидат на коммерциализацию.
2.  **`cognitive-agent`** (Когнитивный агент) - Ядро автономной разработки.
3.  **`decision-engine`** (Движок решений) - Система принятия решений на основе ИИ.
4.  **`it-compass`** (IT-Компас) - Методология и фреймворк для карьерного роста.
5.  **`mcp-server`** (Model Context Protocol Server) - Инфраструктурный компонент для ИИ-агентов.

## Фаза 1: Подготовка и Анализ (1 день)

### Задачи
1.  **Анализ зависимостей:** Для каждого из 5 компонентов провести анализ, какие части кода из `src/` или других сервисов они используют.
    -   Инструмент: `search_in_files`
    -   Пример: `search_in_files(pattern="from src.shared import", path="C:/Projects/portfolio-system-architect/apps/ai-config-manager")`
2.  **Оценка готовности:** Проверить, есть ли у каждого сервиса необходимые артефакты для автономной работы (Dockerfile, requirements.txt, тесты).
    -   Инструмент: `find_files`
    -   Пример: `find_files(pattern="**/Dockerfile", path="C:/Projects/portfolio-system-architect/apps/ai-config-manager")`
3.  **Создание шаблона:** Разработать универсальный шаблон для нового репозитория (структура, README, лицензия, CI/CD).
    -   Инструмент: `write_to_file` (в директории `.gigacode/templates/`)

### Критические файлы
- `apps/ai-config-manager/README.md`
- `apps/cognitive-agent/README.md`
- `apps/decision-engine/README.md`
- `apps/it_compass/README.md`
- `apps/mcp-server/README.md`

## Фаза 2: Создание и Инициализация Новых Репозиториев (2 дня)

### Задачи
1.  **Создание репозиториев:** Создать 5 публичных репозиториев на GitHub.
    -   Имена: `ai-config-manager`, `cognitive-agent`, `decision-engine`, `it-compass`, `mcp-server`.
    -   Инструмент: `execute_command` (используя GitHub CLI `gh repo create`)
2.  **Инициализация:** Инициализировать каждый репозиторий с помощью созданного шаблона.
    -   Инструмент: `write_to_file`, `execute_command` (git init, add, commit, push)
3.  **Перенос кода:** Скопировать исходный код каждого компонента в соответствующий новый репозиторий.

## Фаза 3: Адаптация и Автономизация (3 дня)

### Задачи
1.  **Разрешение зависимостей:** Для кода, использующего общие модули из `src/`, создать отдельные пакеты (например, `src-shared`) или включить код напрямую.
    -   Инструмент: `edit_file`, `write_to_file`
2.  **Настройка CI/CD:** Настроить GitHub Actions для каждого нового репозитория (запуск тестов, проверка линтером).
    -   Инструмент: `write_to_file` (в `.github/workflows`)
3.  **Обновление документации:** Переписать README.md каждого сервиса, чтобы он описывал продукт как самостоятельный проект, а не часть монорепозитория.
    -   Инструмент: `edit_file`
4.  **Создание PyPI пакета (для ai-config-manager):** Настроить `pyproject.toml` и подготовить первый релиз на PyPI.
    -   Инструмент: `write_to_file`, `execute_command` (twine upload)

### Критические файлы
- `pyproject.toml` (для `ai-config-manager`)
- `.github/workflows/ci.yml` (для каждого репозитория)
- Все `README.md` в новых репозиториях
