# ADR-003: Выбор формата диаграмм (Mermaid)

## Статус
Принято

## Контекст
Для визуализации архитектурных решений, компонентов системы и потоков данных необходимо выбрать формат диаграмм, который будет:

1. Легко читаем и поддерживаем
2. Интегрируется с системой документации (Markdown)
3. Поддерживает интерактивность
4. Имеет широкую поддержку инструментами
5. Позволяет автоматизировать генерацию

## Рассмотренные варианты
1. **PlantUML** - мощный инструмент для создания диаграмм UML
2. **Graphviz/DOT** - язык описания графов
3. **Mermaid** - диаграммы в формате Markdown
4. **Draw.io** - визуальный редактор диаграмм
5. **Visio** - Microsoft Visio

## Решение
Выбран Mermaid как основной формат для создания диаграмм по следующим причинам:

1. **Интеграция с Markdown** - диаграммы хранятся прямо в Markdown файлах
2. **Простота использования** - синтаксис близок к естественному описанию диаграмм
3. **Поддержка GitHub** - нативная поддержка на GitHub
4. **Интерактивность** - возможность создания интерактивных диаграмм
5. **Широкая экосистема** - поддержка множеством инструментов и редакторов

## Последствия
### Положительные:
- Единый формат для всех диаграмм в репозитории
- Простота поддержки и обновления диаграмм
- Интеграция с GitHub Pages для интерактивного отображения
- Возможность автоматической генерации диаграмм из кода

### Отрицательные:
- Ограниченная сложность диаграмм по сравнению с PlantUML
- Необходимость изучения синтаксиса Mermaid
- Ограниченная поддержка сложных диаграмм
