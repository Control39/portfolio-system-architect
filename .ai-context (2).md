# Контекст для ИИ-ассистентов

**Екатерина Куделя** - архитектор когнитивных систем и специалист по системному мышлению.

## Профессиональная идентичность
- **Не традиционный кoder**, а **архитектор систем мышления**
- **2-летний переход** от нулевых знаний в IT до архитектуры предприятий
- **Создатель новых профессиональных категорий** вместо вписывания в существующие

## Ключевые файлы для понимания контекста

## Основные принципы работы с автором
1. **Фокус на системном мышлении**, а не только на реализации кода
2. **Каждый компонент системы** решал конкретную личную проблему из её пути
3. **Методология важнее кода** - IT-Compass, маркеры, SMART-критерии
4. **Интеграция и архитектура** приоритетнее отдельных технологий

## Системные компоненты как артефакты пути
- **IT-Compass**: родился из таблицы Excel для отслеживания навыков
- **RAG система**: ответ на информационный хаос и потерянные диалоги с ИИ
- **System Thinking маркеры**: решение проблемы "как измерить системное мышление?"
- **Portfolio-Organizer**: автоматический сбор доказательств для нетрадиционного пути

## Текущий контекст (Март 2026)
- Завершён рефакторинг системы маркеров IT-Compass
- Стандартизированы 18 областей навыков с трёхуровневой структурой
- Создан `system_thinking.json` с 9 маркерами системного мышления
- Подготовка к поиску работы и подаче на грант SourceCraft Open Source
