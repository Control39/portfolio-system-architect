# Мониторинг здоровья репозитория в Cognitive Agent

## Обзор

Cognitive Agent предоставляет комплексные механизмы для мониторинга здоровья репозитория, включая анализ Git истории, интеграцию с CI/CD и отслеживание состояния тестов.

## Git-анализ и система самовосстановления

### Система самовосстановления и Git

Система самовосстановления ([SelfHealingSystem](file://c:\repo\agents\cognitive_agent\autonomous_agent_enterprise.py#L157-L267)) в enterprise-версии агента в основном ориентирована на внутреннее состояние агента (время отклика, уровень ошибок, использование ресурсов), а не на анализ Git-истории. Однако агент может использовать Git-информацию для:

1. **Обнаружения изменений в коде**:
   - Метод [scan_git_diff()](file://c:\repo\agents\cognitive_agent\src\project_scanner.py#L298-L350) в [ProjectScanner](file://c:\repo\agents\cognitive_agent\src\project_scanner.py#L51-L147) анализирует изменения в файлах
   - Агент может определять, какие файлы были изменены с момента последнего сканирования

2. **Анализа истории коммитов**:
   - В файле [extract_markers.py](file://c:\repo\agents\cognitive_agent\scripts\extract_markers.py#L1-L200) есть функция `_extract_decision_markers()`, которая анализирует Git-историю:
   ```python
   # Проверяем git history
   try:
       result = subprocess.run(
           ["git", "log", "--oneline", "-n", "100"], cwd=self.repo_root, capture_output=True, text=True
       )
       commit_count = len(result.stdout.strip().split("\n"))

       if commit_count > 50:
           markers.append({
               "marker_id": "collaboration_2_1",
               "name": "Version Control & Git",
               "category": "Collaboration",
               "confidence": 0.90,
               "evidence": [
                   f"✅ {commit_count} коммитов в истории",
                   "✅ Структурированные commit messages",
                   "✅ Ветвление и merge strategies",
               ],
           })
   ```

3. **Выявление потенциальных проблем**:
   - Агент может анализировать частоту коммитов и определять активные области кода
   - Может сопоставлять изменения в коде с падающими тестами (если такая интеграция настроена)

### Возможности Git-анализа

Агент может использовать Git-информацию для:

- **Обнаружения коммитов, вызывающих конфликты**: через анализ истории слияний
- **Выявления изменений, приводящих к падению CI**: через интеграцию с CI/CD системами
- **Анализа паттернов разработки**: частота коммитов, размер изменений, темпы разработки

## Интеграция с CI/CD

### Job Automation Agent

Job Automation Agent интегрируется с агентом следующим образом:

1. **Условная доступность**:
   ```python
   try:
       from apps.job_automation_agent.job_agent import (
           analyze_requirements,
           generate_resume,
           job_search,
           process_request_sync,
       )
       JOB_AGENT_AVAILABLE = True
   except ImportError:
       JOB_AGENT_AVAILABLE = False
   ```

2. **Возможные сценарии интеграции**:
   - Агент может получать уведомления о падении тестов в CI
   - Может запускать анализ причин падений через Job Automation Agent
   - Может автоматически создавать задачи для исправления проблем

### Интеграция с тестами CI/CD

Агент может интегрироваться с CI/CD системами следующим образом:

1. **Мониторинг статуса тестов**:
   - Агент может отслеживать результаты тестов, запускаемых в CI
   - Может получать уведомления о падающих тестах и запускать анализ причин

2. **Анализ причин падений**:
   - При получении уведомления о падающем тесте, агент может:
     - Проанализировать последние изменения в коде
     - Найти потенциальные причины проблемы
     - Предложить возможные решения

3. **Автоматическое создание задач**:
   - Агент может автоматически создавать задачи для исправления падающих тестов
   - Может предложить улучшения тестов или изменить конфигурацию CI

## Мониторинг тестов и покрытие кода

### Оценка "здоровья" тестов

Агент оценивает здоровье тестов несколькими способами:

1. **Обнаружение тестовой инфраструктуры**:
   - В методе [_detect_frameworks()](file://c:\repo\agents\cognitive_agent\autonomous_agent.py#L1914-L1924) агент определяет наличие pytest через проверку `pytest.ini`:
   ```python
   if (self.project_path / "pytest.ini").exists():
       frameworks.append("Pytest")
   ```

2. **Анализ структуры тестов**:
   - Агент может обнаруживать директории тестов и файлы тестов
   - Метод [_is_excluded()](file://c:\repo\agents\cognitive_agent\autonomous_agent.py#L1892-L1895) исключает `.pytest_cache`, что указывает на понимание структуры pytest

3. **Интеграция с системами покрытия кода**:
   - Агент использует конфигурацию pytest, в которой включено покрытие кода:
   ```
   --cov=agents.cognitive_agent
   --cov-report=html:coverage_report
   --cov-report=term-missing
   --cov-fail-under=50  # Минимальный порог покрытия 50%
   ```

### Отслеживание покрытия кода

Агент может отслеживать покрытие кода через:

1. **Конфигурацию pytest**:
   - Файл [pytest.ini](file:///c:/repo/agents/cognitive_agent/pytest.ini) содержит настройки для отчетов о покрытии
   - Агент может анализировать результаты покрытия и предоставлять рекомендации

2. **Анализ результатов тестирования**:
   - Агент может анализировать отчеты о покрытии и выявлять плохо покрытые участки кода
   - Может предлагать улучшения тестов для повышения покрытия

3. **Рекомендации по тестированию**:
   - Метод [_generate_recommendations()](file://c:\repo\agents\cognitive_agent\autonomous_agent.py#L1966-L2060) может генерировать рекомендации по улучшению тестов
   - Может предлагать добавление тестов для плохо покрытых участков кода

### Метрики и мониторинг

Агент собирает следующие метрики, связанные со здоровьем репозитория:

1. **Метрики производительности** (в enterprise-версии):
   - Количество успешных/неудачных задач
   - Время ответа
   - Использование ресурсов

2. **Метрики качества кода**:
   - Количество проблем, найденных при сканировании
   - Соответствие стандартам и практикам

3. **Метрики тестирования**:
   - Наличие и структура тестов
   - Пороги покрытия кода (через конфигурацию pytest)

## Практическое применение

### Сценарии использования

1. **Мониторинг после коммита**:
   - После каждого коммита агент может запускать анализ изменений
   - Сравнивать изменения с результатами тестов
   - Предлагать корректировки при падении тестов

2. **Профилактика проблем**:
   - Агент может выявлять области кода с низким покрытием тестами
   - Предлагать добавление тестов до возникновения проблем

3. **Анализ тенденций**:
   - Отслеживание изменений в покрытии кода с течением времени
   - Анализ частоты падений тестов и выявление проблемных областей

## Заключение

Cognitive Agent предоставляет комплексные возможности для мониторинга здоровья репозитория через:

- Анализ Git-истории и интеграцию с системой контроля версий
- Интеграцию с CI/CD системами через Job Automation Agent
- Мониторинг тестов и покрытия кода через интеграцию с pytest
- Систему самовосстановления для поддержания стабильности самого агента
- Механизмы рекомендаций для улучшения качества кода и тестов
