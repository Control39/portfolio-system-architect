# Анализ файлов документации в Cognitive Agent

## Обзор

Cognitive Agent может анализировать различные типы файлов документации в проекте, включая README.md, файлы конфигурации (YAML, TOML), и другие текстовые файлы документации. Анализ осуществляется через несколько компонентов:

## Компоненты анализа документации

### 1. ProjectScanner

[ProjectScanner](file://c:\repo\agents\cognitive_agent\src\project_scanner.py#L51-L147) сканирует все файлы проекта, включая файлы документации:

- Обнаруживает файлы с расширениями `.md`, `.yaml`, `.yml`, `.json`, `.toml`
- Проверяет изменения в этих файлах через git
- Использует .gitignore для исключения ненужных файлов

### 2. IT Compass Integration

IT Compass сканирует проект на наличие артефактов документации:

- Обнаруживает README.md файлы
- Проверяет наличие документации (папка `docs/`, файлы `.md`)
- Анализирует структуру документации

### 3. RAG (Retrieval-Augmented Generation) через ChromaDB

Агент индексирует документацию в векторной базе данных для семантического поиска:

- Метод `index_project_documents()` индексирует все текстовые файлы проекта
- Метод `search_similar_documents()` позволяет искать релевантные документы

## Поддерживаемые типы файлов документации

### Markdown файлы

- `README.md` - основное описание проекта
- `CONTRIBUTING.md` - руководство по внесению вклада
- `CHANGELOG.md` - история изменений
- `ARCHITECTURE.md` - архитектурные решения
- `docs/*.md` - дополнительная документация

### Файлы конфигурации

- `*.yaml`, `*.yml` - конфигурации проекта, CI/CD, инфраструктуры
- `*.toml` - файлы конфигурации (например, `pyproject.toml`)
- `*.json` - структурированные данные документации

### Другие текстовые файлы

- `.txt` файлы с документацией
- `.rst` файлы (reStructuredText)
- Другие текстовые форматы, содержащие документацию

## Механизмы анализа

### 1. Сканирование файлов

Агент использует следующие подходы для анализа файлов документации:

- **Хеширование**: файлы проверяются на изменения через вычисление хэшей
- **Параллельное сканирование**: для ускорения обработки больших проектов
- **Фильтрация**: исключение файлов по правилам .gitignore и конфигурации

### 2. Семантический анализ

Через RAG систему агент может:

- Понимать содержимое документации на естественном языке
- Находить релевантные документы для конкретных запросов
- Сопоставлять документацию с кодом проекта

### 3. Проверка соответствия

Агент может обнаруживать:

- Устаревшую информацию в документации
- Несоответствие между описанием и реализацией
- Отсутствие документации для важных компонентов

## Безопасность при анализе документации

Все операции с файлами документации проходят через систему guardrails:

- Проверка доступа к файлам через enterprise guardrails
- Валидация содержимого на опасные паттерны
- Ограничение на размер файлов для анализа

## Примеры анализа

### Обнаружение README файлов

Агент может обнаруживать наличие и содержимое README файлов:

```python
# IT Compass сканирует на наличие артефактов, включая README
artifacts = {
    "readme_exists": True,
    "docs_count": 5,  # количество файлов .md в проекте
    "docs": ["README.md", "docs/setup.md", "docs/api.md", ...]
}
```

### Анализ конфигурационных файлов

Агент может анализировать файлы конфигурации:

```python
# Обнаружение конфигурационных файлов
frameworks = agent._detect_frameworks()
# Может включать:
# - "Docker" (если есть docker-compose.yml)
# - "Pytest" (если есть pytest.ini)
# - "Node.js" (если есть package.json)
```

## Интеграция с AI

При анализе документации агент может использовать AI для:

- Генерации рекомендаций по улучшению документации
- Сравнения описания в документации с фактической реализацией
- Поиска пробелов в документации

## Заключение

Cognitive Agent предоставляет комплексный подход к анализу файлов документации, позволяя:

- Автоматически обнаруживать и индексировать документацию
- Сравнивать документацию с кодом проекта
- Предлагать улучшения и обновления документации
- Обеспечивать безопасность при работе с файлами документации
