# Общие компоненты Cognitive Agent

## Обзор

В рамках приоритетной задачи №1 по устранению дублирования кода, создан модуль `common` с общими компонентами для Standard и Enterprise версий агента. Это позволяет избежать дублирования функциональности и упрощает сопровождение кода.

В рамках приоритетной задачи №2 реализована унифицированная система обработки ошибок.

В рамках приоритетной задачи №3 реализована оптимизация управления памятью и кэшированием.

В рамках следующей задачи реализована система адаптивного обучения и анализа паттернов.

В рамках следующей задачи реализован улучшенный планировщик задач с поддержкой сложных зависимостей.

В рамках следующей задачи реализована улучшенная интеграция с ChromaDB и внедрение полноценного Knowledge Graph.

В рамках следующей задачи реализована система метакогнитивного контроля для оценки качества рассуждений агента.

В рамках следующей задачи реализована расширенная система безопасности и надежности (Guardrails).

## Компоненты

### BaseLogger

Класс [BaseLogger](file:///c:/repo/agents/cognitive_agent/common/base_logger.py) предоставляет унифицированный интерфейс для структурированного логирования с использованием `structlog`. Поддерживает JSON и текстовый форматы логов, а также привязку контекста к логгеру.

**Функциональность:**
- Поддержка различных уровней логирования (info, debug, warning, error, critical)
- Структурированные логи в формате JSON
- Возможность привязки контекста к логгеру
- Настройка формата и уровня логирования
- Обработка ошибок при настройке логирования через исключения из модуля exceptions

### BaseSecurityChecker

Класс [BaseSecurityChecker](file:///c:/repo/agents/cognitive_agent/common/base_security.py) реализует базовые проверки безопасности, включая валидацию команд, путей к файлам и кода. Также содержит проверки на изменение критических файлов.

**Функциональность:**
- Проверка команд на опасные паттерны
- Валидация путей к файлам на безопасность
- Проверка кода на потенциально опасные конструкции
- Определение критических файлов
- Валидация разрешений на изменение файлов
- Санитизация входных данных

### BaseProjectScanner

Класс [BaseProjectScanner](file:///c:/repo/agents/cognitive_agent/common/base_scanner.py) реализует функциональность сканирования проекта, включая определение файлов, директорий, технологического стека и проблем в проекте.

**Функциональность:**
- Сканирование файлов и директорий с учетом .gitignore
- Определение технологического стека проекта
- Поиск проблем (большие файлы, TODO комментарии)
- Подсчет статистики проекта
- Получение списка измененных файлов
- Вычисление хэша проекта
- Обработка ошибок сканирования через иерархию исключений
- Использование генераторов для обработки больших файлов
- Эффективное сканирование больших файлов с помощью генераторов

**Функциональность:**

- Проверка контекста задач на безопасность
- Кэширование результатов
- Выполнение функций с таймаутом
- Подготовка начального состояния агента
- Обработка ошибок через централизованный обработчик
- Интеграция с менеджером памяти
- Контроль использования памяти при выполнении операций

### Exceptions

Модуль [exceptions.py](file:///c:/repo/agents/cognitive_agent/common/exceptions.py) содержит иерархию исключений и централизованный обработчик ошибок.

**Иерархия исключений:**
- `CognitiveAgentError` - базовое исключение для агента
- `ConfigurationError` - ошибка конфигурации
- `SecurityViolationError` - нарушение безопасности
- `ResourceExhaustionError` - исчерпание ресурсов
- `AIServiceError` - ошибка сервиса ИИ
- `FileOperationError` - ошибка операции с файлами
- `NetworkError` - ошибка сети
- `ValidationError` - ошибка валидации данных
- `TaskExecutionError` - ошибка выполнения задачи
- `IntegrationError` - ошибка интеграции
- `AgentStateError` - ошибка состояния агента
- `DataProcessingError` - ошибка обработки данных
- `CacheError` - ошибка кэширования
- `AuditLogError` - ошибка аудит-логирования

**Обработчик ошибок:**
- `ErrorHandler` - централизованный обработчик ошибок
- `handle_errors` - декоратор для обработки ошибок в функциях
- `register_error_handler` - регистрация глобального обработчика

## Преимущества

1. **Устранение дублирования кода**: Общая функциональность реализована в одном месте
2. **Упрощение сопровождения**: Изменения в общих компонентах автоматически применяются к обеим версиям
3. **Повышение согласованности**: Обе версии агента используют одинаковую реализацию базовой функциональности
4. **Ускорение разработки**: Новые функции можно добавлять в общие компоненты и использовать в обеих версиях
5. **Унифицированная обработка ошибок**: Единая стратегия обработки ошибок по всему агенту
6. **Улучшенная диагностика**: Централизованное логирование ошибок с контекстом
7. **Предсказуемое поведение**: Единая стратегия обработки ошибок
8. **Облегчение отладки**: Простое отслеживание ошибок через всю систему
9. **Оптимизация использования памяти**: Эффективное кэширование с учетом ограничений памяти
10. **Контроль ресурсов**: Мониторинг и управление использованием памяти
11. **Ленивая загрузка**: Экономия памяти за счет отложенной загрузки данных
12. **Генераторы для больших файлов**: Экономия памяти при обработке больших файлов
13. **Самообучение**: Агент улучшает свои решения на основе исторических данных
14. **Адаптивность**: Агент адаптирует подход к решению задач на основе прошлого опыта
15. **Прогнозирование**: Агент может предсказать успешность подхода до его реализации
16. **Улучшенное планирование**: Поддержка сложных зависимостей и условных ветвлений
17. **Параллельное выполнение**: Повышение эффективности за счет параллелизма
18. **Откат изменений**: Повышение надежности за счет возможности отката
19. **Глубокое понимание архитектуры**: Knowledge Graph для представления архитектурных связей
20. **Анализ зависимостей**: Более точный анализ зависимостей между компонентами
21. **Обнаружение паттернов**: Автоматическое обнаружение архитектурных паттернов и анти-паттернов
22. **Метакогнитивный контроль**: Самооценка качества рассуждений и принятие решений
23. **Самокоррекция**: Возможность пересмотра решений при обнаружении неуверенности
24. **Оценка уверенности**: Контроль уровня уверенности в принятых решениях
25. **Расширенная безопасность**: Защита от потенциально вредоносных действий
26. **Соответствие нормам**: Обеспечение соответствия внутренним политикам и стандартам
27. **Аудит и трассировка**: Подробное логирование действий агента для аудита
