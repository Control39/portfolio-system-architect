# Обучение Cognitive Agent и запрос архитектурных улучшений

## Как обучать агента

Cognitive Agent реализует систему самообучения и адаптации через несколько механизмов:

### 1. Система памяти решений

Агент использует метод `_remember_decision()` для сохранения решений и их результатов:

```python
def _remember_decision(self, context: dict, decision: str, outcome: str):
    """Запомнить решение и его результат для будущего обучения"""
    self.memory["decisions"].append(
        {"context": context, "decision": decision, "outcome": outcome, "timestamp": datetime.now().isoformat()}
    )

    # Ограничиваем историю 100 записями
    if len(self.memory["decisions"]) > 100:
        self.memory["decisions"] = self.memory["decisions"][-100:]

    # Обновляем success rate
    success_count = sum(1 for d in self.memory["decisions"] if d["outcome"] == "success")
    self.memory["success_rate"] = success_count / len(self.memory["decisions"]) if self.memory["decisions"] else 0.0
```

Функциональность обучения теперь также доступна через [BaseAgentExtensions](file:///c:/repo/agents/cognitive_agent/common/base_agent_extensions.py), что позволяет использовать общую логику обучения в обеих версиях агента.

### 2. Система метрик

Агент собирает метрики производительности и эффективности:

- **MetricsCollector** в enterprise-версии отслеживает:
  - Количество выполненных/неудачных задач
  - Вызовы AI и их результаты
  - Обработанные файлы
  - Время отклика
  - Использование ресурсов (CPU, память)

### 3. Как сообщить об ошибках агента

Чтобы "обучить" агента, когда он делает ошибки:

1. **Используйте отрицательную обратную связь**: При выполнении задач через агент, если он принимает неправильное решение, система фиксирует результат как `"outcome": "cancelled"` или `"outcome": "failed"`

2. **Добавьте контекст ошибки**: В методе `_remember_decision()` сохраняется контекст, решение и результат, что позволяет агенту учиться на ошибках

3. **Мониторьте метрики**: Система отслеживает success rate и другие показатели, которые помогают агенту адаптировать свое поведение

4. **Корректировка через конфигурацию**: В файле [config/learning.yaml](file:///c:/repo/agents/cognitive_agent/config/learning.yaml) можно настроить параметры обучения:
   - Режим обучения (continuous, batch, hybrid)
   - Скорость обучения
   - Типы моделей для машинного обучения
   - Критерии оценки эффективности

### 4. Механизмы обратной связи

Агент реализует несколько каналов обратной связи:

- **Явная обратная связь**: Результаты выполнения задач (успешно/неудачно)
- **Неявная обратная связь**: Время выполнения, использование ресурсов, частота ошибок
- **Контекстная обратная связь**: Условия, в которых было принято решение
- **Пользовательская обратная связь**: (планируется) оценки эффективности решений

## Как запросить архитектурные улучшения у агента

Агент может анализировать зависимости между сервисами и предлагать архитектурные улучшения через несколько подходов:

### 1. Использование сканера проекта

Агент анализирует зависимости через [BaseProjectScanner](file:///c:/repo/agents/cognitive_agent/common/base_scanner.py):

```python
# Сканирование проекта для анализа архитектуры
result = agent.scan_project()
# Результаты содержат информацию о зависимостях между файлами/модулями
```

### 2. Запрос анализа архитектуры

Для запроса анализа зависимостей между сервисами X, Y, Z, вы можете использовать следующий подход:

```python
# Пример вызова анализа архитектуры
task = """
Проанализируй зависимости между сервисами X, Y, Z.
Я вижу, что они стали слишком сильно связаны.
Предложи, как можно снизить эту связанность, возможно, разделив один из сервисов или введя брокер сообщений.
"""

result = agent.execute_task(task)
```

### 3. Использование RAG для архитектурного анализа

Агент использует ChromaDB для семантического поиска по проекту:

```python
# Поиск похожих архитектурных решений
similar_architectures = agent.search_similar_documents("service decoupling patterns", top_k=5)
```

### 4. Система рекомендаций

Агент генерирует рекомендации на основе анализа:

- Обнаружение тесно связанных компонентов
- Выявление потенциальных точек разделения
- Предложения по внедрению шины сообщений
- Рекомендации по рефакторингу

### 5. Пример архитектурного запроса

Для получения архитектурных улучшений от агента, используйте структурированный запрос:

```
"Пожалуйста, проанализируй текущую архитектуру сервисов [перечислите конкретные сервисы].
Выполни анализ связности и_coupling_ между ними.
Определи области с высокой связанностью и предложи стратегии декомпозиции:
1. Возможные границы сервисов
2. Паттерны декомпозиции (например, AsyncAPI, gRPC, REST)
3. Рекомендации по внедрению брокера сообщений
4. Потенциальные точки отказа и риски
5. План миграции с минимальным воздействием на систему
"
```

Агент будет использовать:
- Результаты сканирования проекта
- Информацию из ChromaDB (если доступна)
- Маркеры архитектурных решений из IT-Compass
- Исторические данные о принятых решениях
- Внешние знания из AI-моделей

Для реализации этих возможностей агент интегрируется с:
- [IT-Compass](file:///c:/repo/apps/it_compass/src/it_compass_scanner.py#L1-L33) для архитектурных маркеров
- [ChromaDB](file:///c:/repo/apps/embedding_agent/chroma_indexer.py#L1-L115) для RAG
- Системой самообучения для улучшения качества рекомендаций
- Системой метрик для оценки эффективности предложений
