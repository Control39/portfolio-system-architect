## Общее описание

Когнитивный агент - это автономный AI-агент, который запускается при открытии проекта и работает в фоновом режиме. Его основные функции:
- Сканирование кода и архитектуры проекта
- Анализ зависимостей и проблем
- Предложение улучшений
- Автоматическое выполнение задач (с подтверждением)

Агент интегрирован с AI Provider Manager, который использует GigaChat (облако) в качестве основного провайдера и Ollama (локально) в качестве fallback.

## Архитектура

Агент построен по принципу "атомов и молекул", где общие компоненты вынесены в базовый класс [BaseCognitiveAgent](file:///c:/repo/agents/cognitive_agent/src/base_agent.py), а специфичные функции реализованы в наследниках.

### Общие компоненты
- Система логирования (структурированное и аудит)
- Валидация входных данных
- Проверка безопасности (guardrails)
- Управление состоянием
- Интеграция с AI
- Сканирование проекта
- Обработка задач

## Стандартная версия агента

Файл: [autonomous_agent.py](file:///c:/repo/agents/cognitive_agent/autonomous_agent.py)

Стандартная версия агента предоставляет основные функции когнитивного агента:
- Сканирование проекта
- Генерация рекомендаций
- Выполнение задач
- Интеграция с AI
- Базовая система безопасности

### Основные методы
- `start()` - запуск агента
- `stop()` - остановка агента
- `scan_project()` - сканирование проекта
- `execute_task()` - выполнение задачи
- `get_status()` - получение статуса агента

## Enterprise-версия агента

Файл: [autonomous_agent_enterprise.py](file:///c:/repo/agents/cognitive_agent/autonomous_agent_enterprise.py)

Enterprise-версия агента включает расширенные возможности:
- Полная поддержка async/await для неблокирующих операций
- Расширенное планирование задач с графами зависимостей
- Система самовосстановления с обнаружением аномалий
- Комплексный мониторинг и наблюдаемость
- Корпоративная безопасность с ролевой моделью доступа (RBAC)
- Состояние с сохранением и версионированием снимков
- Обработка ошибок с градуированной стратегией восстановления
- Метрики производительности и отслеживание использования ресурсов

### Дополнительные компоненты
- `MetricsCollector` - система сбора метрик
- `SelfHealingSystem` - система самовосстановления
- `TaskPlanner` - планировщик задач с зависимостями
- `StateManager` - менеджер состояния с сохранением

## Классы и методы

### BaseCognitiveAgent (базовый класс)
Расположение: [src/base_agent.py](file:///c:/repo/agents/cognitive_agent/src/base_agent.py)

Абстрактный базовый класс, определяющий общую функциональность для обеих версий агента.

#### Атрибуты
- `agent_id` - уникальный идентификатор агента
- `project_path` - путь к проекту
- `running` - статус работы агента
- `scan_interval` - интервал между сканированиями
- `ai_call_counter` - счетчик вызовов AI
- `OPERATION_LIMITS` - лимиты операций
- `DANGEROUS_PATTERNS` - запрещенные паттерны
- `AI_RESPONSE_DANGEROUS_PATTERNS` - паттерны для проверки AI-ответов

#### Методы
- `_initialize_base_components()` - инициализация общих компонентов
- `_load_guardrails()` - загрузка правил безопасности
- `_validate_task()` - валидация задачи
- `_validate_ai_response()` - валидация AI-ответа
- `_check_file_access()` - проверка доступа к файлу
- `_log_action()` - логирование действий
- `_log_security_event()` - логирование событий безопасности
- `_remember_decision()` - запоминание решений
- `_run_compass_scan()` - сканирование IT Compass
- `_generate_recommendations()` - генерация рекомендаций
- `_save_scan_results()` - сохранение результатов сканирования
- `get_status()` - получение статуса агента

## Система логирования

Расположение: [src/logging_config.py](file:///c:/repo/agents/cognitive_agent/src/logging_config.py)

Система логирования включает:
- Стандартное логирование через `logging`
- Структурированное логирование через `structlog`
- Аудит-логирование действий
- Логирование безопасности

### StructuredLogger
Класс для JSON-логирования, совместимого с ELK/Grafana.

### AuditLogger
Класс для трассировки всех действий агента с детализацией.

## Безопасность

Система безопасности включает:
- Проверку guardrails для всех операций
- Валидацию входных данных
- Проверку AI-ответов на опасные паттерны
- Контроль доступа к файлам
- Rate limiting для AI-вызовов
- Enterprise-уровень безопасности через RBAC

### EnterpriseGuardrails
Расположение: [src/enterprise_guardrails.py](file:///c:/repo/agents/cognitive_agent/src/enterprise_guardrails.py)

Система enterprise-уровня безопасности с ролевой моделью доступа.

## Мониторинг и метрики

Enterprise-версия включает расширенную систему мониторинга:
- Сбор метрик производительности
- Отслеживание использования ресурсов
- Обнаружение аномалий
- Система самовосстановления
- Мониторинг состояния систем

### MetricsCollector
Сборщик метрик для enterprise-мониторинга, отслеживающий:
- Завершенные/неудачные задачи
- Вызовы AI
- Обработанные файлы
- Длительность сканирования
- Время ответа
- Использование CPU и памяти

### SelfHealingSystem
Система самовосстановления, которая:
- Обнаруживает аномалии в работе агента
- Применяет стратегии восстановления
- Мониторит показатели производительности

## Интеграции
