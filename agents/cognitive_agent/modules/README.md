# Модули Cognitive Agent

Этот модуль предоставляет расширяемую архитектуру для Cognitive Agent с четким разделением ответственности.

## Архитектура

```
cognitive_agent/
├── core/                    # Ядро агента (базовые классы, оркестраторы)
├── modules/                 # Модули функциональности
│   ├── planning/           # Планирование задач
│   ├── execution/          # Выполнение задач
│   ├── monitoring/         # Мониторинг состояния
│   └── reporting/          # Отчетность и логирование
├── common/                  # Общие компоненты
│   ├── cache/             # Кэширование
│   ├── memory/            # Управление памятью
│   ├── utils/             # Утилиты
│   └── exceptions/        # Исключения
└── agents/                  # Конкретные агенты
```

## Архитектурный паттерн: Core + Modules + Orchestrator

### Core (Ядро)
- Базовые классы агента
- Оркестраторы
- Центральные компоненты, необходимые для работы
- Не зависит от модулей

### Modules (Модули)
- Расширяемая функциональность
- Не зависит друг от друга
- Может быть подключен/отключен независимо
- Реализует конкретные аспекты поведения агента

### Orchestrator (Оркестратор)
- Координирует работу модулей
- Управляет жизненным циклом агента
- Обрабатывает события и переключения состояний
- Связывает core и modules

## Модули

### planning/ — Планирование задач

Планирование и управление задачами агента.

**Ключевые компоненты:**
- `TaskPlanner` — Планировщик задач
- `TaskGraph` — Граф зависимостей задач
- `Plan` — План выполнения
- `PlannerConfig` — Конфигурация планировщика

**Функциональность:**
- Разбиение целей на подзадачи
- Построение графа зависимостей
- Оптимизация порядка выполнения
- Управление приоритетами

**Зависимости:**
- `common.cache` — Кэширование планов
- `common.memory` — Хранение истории планов
- `common.utils` — Утилиты обработки данных

**Пример использования:**
```python
from modules.planning import TaskPlanner, Plan

planner = TaskPlanner()
plan = planner.create_plan(
    goal="Создать отчет",
    subtasks=["Собрать данные", "Анализ", "Форматирование"]
)
```

### execution/ — Выполнение задач

Выполнение запланированных задач и управление агентами.

**Ключевые компоненты:**
- `TaskExecutor` — Исполнитель задач
- `AgentExecutor` — Исполнитель агентов
- `ExecutionEngine` — Движок выполнения
- `ExecutorConfig` — Конфигурация исполнителя

**Функциональность:**
- Выполнение задач по плану
- Управление агентами
- Обработка ошибок выполнения
- Поддержка асинхронного выполнения

**Зависимости:**
- `modules.planning` — Для получения задач
- `common.memory` — Хранение результатов
- `common.exceptions` — Обработка ошибок

**Пример использования:**
```python
from modules.execution import TaskExecutor

executor = TaskExecutor()
results = executor.execute_plan(plan)
```

### monitoring/ — Мониторинг состояния

Мониторинг состояния агента и его компонентов.

**Ключевые компоненты:**
- `AgentMonitor` — Монитор агента
- `PerformanceMonitor` — Монитор производительности
- `StateMonitor` — Монитор состояния
- `MonitorConfig` — Конфигурация монитора

**Функциональность:**
- Отслеживание состояния агента
- Мониторинг производительности
- Сбор метрик
- Генерация алертов

**Зависимости:**
- `common.cache` — Кэширование метрик
- `common.memory` — Хранение истории
- `common.exceptions` — Обработка ошибок

**Пример использования:**
```python
from modules.monitoring import AgentMonitor

monitor = AgentMonitor()
metrics = monitor.get_metrics()
```

### reporting/ — Отчетность и логирование

Формирование отчетов и ведение логов.

**Ключевые компоненты:**
- `ReportGenerator` — Генератор отчетов
- `Logger` — Логгер
- `AuditLogger` — Аудит-логгер
- `ReportConfig` — Конфигурация отчета

**Функциональность:**
- Генерация отчетов о выполнении
- Ведение логов событий
- Аудит действий
- Экспорт в различные форматы

**Зависимости:**
- `modules.execution` — Для получения результатов
- `common.utils` — Утилиты форматирования
- `common.exceptions` — Обработка ошибок

**Пример использования:**
```python
from modules.reporting import ReportGenerator

reporter = ReportGenerator()
report = reporter.generate_report(execution_results)
```

## Добавление нового модуля

1. Создать подкаталог в `modules/`
2. Создать `__init__.py` с экспортами
3. Создать `README.md` с описанием
4. Добавить зависимости в `requirements.txt`
5. Обновить документацию

**Структура модуля:**
```
modules/
└── new_module/
    ├── __init__.py
    ├── README.md
    ├── core.py          # Основная логика
    ├── config.py        # Конфигурация
    └── tests/
        └── test_core.py
```

## Принципы проектирования

### 1. Single Responsibility
Каждый модуль отвечает за одну функциональность.

### 2. Open/Closed
Модули открыты для расширения, закрыты для модификации.

### 3. Dependency Inversion
Модули зависят от абстракций, не от конкретных реализаций.

### 4. Separation of Concerns
Четкое разделение ответственности между модулями.
