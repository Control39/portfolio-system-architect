# Директория agents

Эта директория содержит все агенты системы Portfolio System Architect. Агенты представляют собой автономные компоненты, предназначенные для выполнения различных задач автоматизации, анализа и управления в рамках проекта.

## Структура директории

- [cognitive_agent/](./cognitive_agent) - Когнитивный агент, основной интеллектуальный компонент системы
- [chroma_db/](./chroma_db) - Хранилище векторной базы данных для агентов
- [logs/](./logs) - Журналы работы агентов

## Основные компоненты

### Когнитивный агент (cognitive_agent)

Это центральный компонент системы, обладающий следующими возможностями:

- **Анализ кода**: Интеграция с инструментами статического анализа (MyPy, Ruff, Bandit, Pyright, Coverage)
- **Анализ документации**: Проверка полноты и согласованности документации с кодом
- **Анализ тестов**: Оценка качества и покрытия тестами
- **Планирование задач**: Управление сложными задачами с учетом зависимостей
- **Самообслуживание**: Система самовосстановления и мониторинга
- **Интеграция с AI**: Поддержка различных провайдеров искусственного интеллекта
- **Enterprise-безопасность**: Система контроля доступа и безопасного выполнения операций

#### Архитектура cognitive_agent

- **[autonomous_agent_enterprise.py](./cognitive_agent/autonomous_agent_enterprise.py)** - Основной класс enterprise-версии агента
- **[src/](./cognitive_agent/src)** - Исходный код базовых компонентов
- **[common/](./cognitive_agent/common)** - Общие утилиты и базовые классы
- **[skills/](./cognitive_agent/skills)** - Наборы навыков агента
- **[config/](./cognitive_agent/config)** - Конфигурационные файлы
- **[tests/](./cognitive_agent/tests)** - Тесты для компонентов агента
- **[docs/](./cognitive_agent/docs)** - Документация по агенту

#### Функциональные возможности

1. **Анализ качества кода**:
   - Проверка типизации с помощью MyPy
   - Проверка стиля кода с помощью Ruff
   - Анализ безопасности с помощью Bandit
   - Проверка типов с помощью Pyright
   - Оценка покрытия тестами

2. **Анализ документации**:
   - Проверка наличия docstring'ов
   - Анализ структуры Markdown файлов
   - Оценка согласованности между кодом и документацией

3. **Анализ тестов**:
   - Поиск файлов тестов
   - Определение используемого фреймворка
   - Оценка качества и покрытия тестами

### Хранилище векторной базы данных (chroma_db)

Используется для хранения и поиска семантических данных, что позволяет агентам:

- Выполнять поиск по схожести
- Понимать контекст кода и документации
- Реализовывать RAG (Retrieval-Augmented Generation) функциональность

### Журналы работы агентов (logs)

Содержит логи работы всех агентов, включая:

- Информационные сообщения
- Предупреждения
- Ошибки
- Статистику выполнения задач
- Аудит безопасности

## Использование

Для запуска когнитивного агента используйте:

```bash
cd agents/cognitive_agent
python main.py --start
```

## Безопасность

Все агенты работают в соответствии с системой enterprise-контроля доступа, которая обеспечивает:

- Проверку разрешений на выполнение операций
- Защиту от небезопасных команд
- Аудит выполненных действий
- Контроль использования ресурсов

Для получения подробной информации о системе безопасности см. [enterprise_guardrails.py](./cognitive_agent/enterprise_guardrails.py).
