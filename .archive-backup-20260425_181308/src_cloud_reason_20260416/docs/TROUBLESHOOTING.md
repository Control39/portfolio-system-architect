## Проблемы с установкой и настройкой

### Ошибка: "Python не найден"

**Симптомы**:
- Скрипты не запускаются
- Сообщение "'python' is not recognized as an internal or external command"

**Решение**:
1. Убедитесь, что Python установлен:
   - Скачайте Python с [официального сайта](https://www.python.org/downloads/)
   - Установите Python 3.8 или выше
   - В установщике обязательно отметьте "Add Python to PATH"

2. Проверьте установку:
```cmd
python --version
```

3. Если команда не работает, добавьте Python в PATH вручную:
   - Найдите путь установки Python (обычно `C:\Python39` или `C:\Users\<user>\AppData\Local\Programs\Python\Python39`)
   - Добавьте этот путь в переменную окружения PATH

### Ошибка: "Git не найден"

**Симптомы**:
- Команды git не распознаются
- Сообщение "'git' is not recognized as an internal or external command"

**Решение**:
1. Установите Git с [официального сайта](https://git-scm.com/download/win)
2. В установщике выберите опцию "Use Git from Windows Command Prompt"
3. Перезапустите терминал
4. Проверьте установку:
```cmd
git --version
```

### Ошибка при запуске setup_workspace.bat

**Симптомы**:
- Скрипт завершается с ошибками
- Некоторые компоненты не устанавливаются

**Решение**:
1. Запустите скрипт от имени администратора
2. Убедитесь, что у вас есть доступ к интернету
3. Проверьте, что антивирус не блокирует установку пакетов
4. Попробуйте установить зависимости вручную:
```cmd
pip install chardet
```

## Проблемы с кодировкой текста

### Проблема: "Кракозябры" в тексте (mojibake)

**Симптомы**:
- Текст отображается как набор странных символов
- Русский текст выглядит как "РџСЂРёРјРµСЂ С‚РµРєСЃС‚Р°"


**Решение**:
1. Определите исходную кодировку:
```cmd
python scripts\analyze_all.py
```

2. Выполните конвертацию в UTF-8:
```cmd
python scripts\convert_to_utf8.py
```

3. Проверьте результат:
```cmd
python scripts\validate_results.py
```

4. Если проблема сохраняется, попробуйте открыть файл в редакторе с явным указанием кодировки (CP1251) и сохранить как UTF-8.

### Проблема: Символы-заменители (�)

**Симптомы**:
- Вместо некоторых символов отображается знак вопроса в ромбе (�)
- Часть текста отсутствует

**Решение**:
1. Это означает, что файл содержит символы, которые не могут быть корректно отображены
2. Проверьте исходную кодировку файла
3. Убедитесь, что шрифт поддерживает необходимые символы
4. Если возможно, замените проблемные символы на стандартные

### Проблема: Смешанные кодировки в одном файле

**Симптомы**:
- Часть текста отображается нормально, часть - как кракозябры
- Разные фрагменты используют разные кодировки

**Решение**:
1. Разделите файл на части по кодировкам
2. Конвертируйте каждую часть отдельно
3. Объедините файлы, сохраняя все в UTF-8
4. Используйте редактор с поддержкой нескольких кодировок (например, Notepad++)


## Проблемы с путями и именами файлов

### Проблема: Ошибки с кириллическими путями

**Симптомы**:
- Ошибки при доступе к файлам с русскими названиями
- Сообщения о несуществующих файлах
- Скрипты не видят файлы в папках с кириллицей

**Решение**:
1. Убедитесь, что системная кодировка установлена в UTF-8:
```cmd
chcp 65001
```

2. Запустите конфигурацию Windows:
```cmd
call configs\windows_env.bat
```

3. Перезагрузите компьютер
4. Используйте короткие пути без кириллицы при возможности

### Проблема: Слишком длинные пути

**Симптомы**:
- Ошибки "Path too long"
- Невозможность создать или скопировать файлы

**Решение**:
1. Включите поддержку длинных путей в Windows:
   - Откройте редактор реестра (regedit)
   - Перейдите в `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\FileSystem`
   - Установите `LongPathsEnabled` в 1

2. Переместите проект ближе к корню диска:
```cmd
# Например, в C:\projects\ вместо вложенной структуры
```

3. Используйте символические ссылки:
```cmd
mklink /D "C:\short" "C:\very\long\path\to\project"
```

## Проблемы с Git

### Проблема: Неправильное отображение кириллицы в Git

**Симптомы**:
- Русские буквы в именах файлов отображаются некорректно
- Ошибки при коммите файлов с кириллицей

**Решение**:
1. Настройте Git для работы с UTF-8:
```cmd
git config --global core.quotepath false
git config --global core.autocrlf false
git config --global commit.encoding utf-8
```

2. Создайте или обновите файл `.gitattributes` в корне проекта:
```
* text=auto
*.txt text eol=lf
*.md text eol=lf
```

3. Перезапустите терминал

### Проблема: Ошибки при коммите после конвертации

**Симптомы**:
- Git не видит изменения в файлах
- Ошибки при попытке добавить файлы в индекс

**Решение**:
1. Убедитесь, что файлы действительно изменились:
```cmd
git status
```

2. Попробуйте явно добавить файлы:
```cmd
git add --all
git commit -m "[encoding] Convert to UTF-8"
```

3. Если проблема сохраняется, проверьте права доступа к файлам

## Проблемы с Python

### Проблема: Ошибки кодировки при запуске скриптов

**Симптомы**:
- Ошибки `UnicodeDecodeError` или `UnicodeEncodeError`
- Проблемы с чтением/записью файлов

**Решение**:
1. Установите переменные окружения:
```cmd
set PYTHONIOENCODING=utf-8
set PYTHONLEGACYWINDOWSSTDIO=1
```

2. Или добавьте в начало скрипта:
```python
import sys
import io
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8')
```

3. Убедитесь, что файлы сохранены в UTF-8

### Проблема: chardet не может определить кодировку

**Симптомы**:
- chardet возвращает `None` или низкий confidence
- Невозможно определить кодировку файла

**Решение**:
1. Попробуйте определить кодировку вручную:
   - CP1251 (Windows Cyrillic)
   - KOI8-R (Unix Cyrillic)
   - ISO-8859-1 (Latin-1)

2. Проверьте файл в разных редакторах
3. Используйте специализированные инструменты (например, `file` в Linux или Total Commander)

## Проблемы с Windows

### Проблема: CMD не поддерживает UTF-8

**Симптомы**:
- Русский текст не отображается в командной строке
- Символы заменяются на кракозябры

**Решение**:
1. Установите кодировку UTF-8 в CMD:
```cmd
chcp 65001
```

2. Или добавьте в автозапуск:
```cmd
# В реестре: HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor\Autorun
echo chcp 65001 >nul
```

3. Используйте современные терминалы:
   - Windows Terminal (из Microsoft Store)
   - PowerShell
   - Git Bash

### Проблема: PowerShell не отображает кириллицу

**Симптомы**:
- Русский текст не читаем в PowerShell
- Проблемы с выводом скриптов

**Решение**:
1. Установите кодировку UTF-8:
```powershell
[Console]::OutputEncoding = [Text.Encoding]::UTF8
```

2. Добавьте в профиль PowerShell:
```powershell
# Файл: $PROFILE (обычно в Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1)
[Console]::OutputEncoding = [Text.Encoding]::UTF8
$OutputEncoding = [Console]::OutputEncoding
```

3. Перезапустите PowerShell

## Общие рекомендации

### Профилактика проблем с кодировкой

1. **Используйте UTF-8 повсеместно**:
   - Сохраняйте все текстовые файлы в UTF-8
   - Настройте редакторы на сохранение в UTF-8
   - Убедитесь, что консоль поддерживает UTF-8

2. **Стандартизируйте окружение**:
   - Используйте одинаковые настройки на всех машинах
   - Документируйте конфигурацию
   - Автоматизируйте настройку

3. **Регулярно проверяйте кодировку**:
   - Запускайте анализ кодировок перед коммитами
   - Включайте проверку в CI/CD pipeline
   - Обучайте команду основам работы с кодировками

### Когда обращаться за помощью

Обратитесь к специалисту если:
- Проблема затрагивает множество файлов
- Стандартные методы не помогают
- Есть риск потери данных
- Требуется анализ сложных случаев смешанных кодировок

При обращении предоставьте:
- Описание проблемы
- Примеры файлов (если возможно)
- Содержимое логов
- Версии используемых программ
