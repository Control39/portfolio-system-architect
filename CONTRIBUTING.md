# Руководство для контрибьюторов

Спасибо за интерес к проекту `portfolio-system-architect`!

---

## Принципы проекта

1. **Системность** — все компоненты должны быть частью единой экосистемы
2. **Практичность** — каждый компонент должен решать реальную задачу
3. **Масштабируемость** — архитектура должна поддерживать расширение
4. **Документированность** — каждый компонент должен иметь полную документацию

## Как внести вклад

### 1. Форкните репозиторий

Создайте форк на GitHub.

### 2. Клонируйте

```bash
git clone https://github.com/ВАШ_ЛОГИН/portfolio-system-architect.git
cd portfolio-system-architect
```

### 3. Создайте ветку

```bash
git checkout -b feature/название-функции
```

### 4. Внесите изменения

- Исправьте ошибку в коде или документации
- Добавьте новый компонент в `components/`
- Создайте кейс в `cases/`
- Добавьте пример использования в `examples/`

### 5. Закоммитьте

```bash
git add .
git commit -m "feat: краткое описание изменения"
```

### 6. Отправьте и создайте PR

```bash
git push origin feature/название-функции
```

Создайте Pull Request с описанием изменений.

## Правила именования файлов и папок

### Основные принципы
1. **Лаконичность и ясность** — имена должны отражать содержимое
2. **Единый стиль** — используйте kebab-case для папок и файлов
3. **Английский язык** — все имена файлов и папок на английском

### Специфичные правила
- **Папки компонентов**: `component-name/` (например: `it-compass/`, `portfolio-organizer/`)
- **Папки кейсов**: `case-category/case-name/` (например: `thinking-cases/01-ai-communication-breakthrough/`)
- **Файлы документации**: `topic-name.md` (например: `architecture-tradeoffs.md`)
- **Скрипты**: `action-purpose.extension` (например: `duplicate_finder.py`, `generate_docs.py`)

### Запрещённые имена
- Общие имена: `temp/`, `test/`, `new/`
- Пробелы в именах файлов и папок
- Специальные символы кроме `-`, `_`, `.`

## Предотвращение дублирования контента

### Принципы
1. **Единственный источник истины** — каждый фрагмент информации должен храниться в одном месте
2. **Ссылки вместо копирования** — используйте относительные ссылки между документами
3. **Регулярный аудит** — ежемесячная проверка на дубликаты

### Практические рекомендации
1. **Перед созданием нового файла**:
   - Проверьте существующие файлы через поиск по содержимому
   - Используйте скрипт `scripts/duplicate_finder.py` для проверки
   
2. **При обнаружении дубликата**:
   - Определите каноническую версию
   - Удалите дубликат
   - Обновите все ссылки на каноническую версию
   
3. **Автоматическая проверка**:
   - CI/CD пайплайн запускает проверку дубликатов при каждом пуше
   - Критические дубликаты блокируют мерж

## Стандарты организации структуры репозитория

### Основная структура
```
portfolio-system-architect/
├── components/          # Реализации компонентов
├── cases/              # Практические кейсы
├── docs/               # Документация

├── scripts/            # Скрипты автоматизации
```

### Правила организации
1. **Компоненты** — каждый компонент должен быть самодостаточным:
   
```
   component-name/

   ├── tests/          # Тесты компонента

   └── README.md       # Описание компонента
   
```

2. **Документация** — структурирована по темам:
   
```
   docs/
   ├── architecture/   # Архитектурные решения
   ├── methodology/    # Методологические материалы
   ├── guides/         # Руководства
   └── templates/      # Шаблоны документов
   
```

3. **Кейсы** — разделены по типам:
   
```
   cases/
   ├── evolution-cases/    # Кейсы эволюции систем
   ├── thinking-cases/     # Кейсы системного мышления
   └── presentation-cases/ # Кейсы для презентаций
   
```

## Автоматическая проверка дубликатов (CI/CD)

### Настройка проверки
1. **Скрипт для поиска дубликатов**: `scripts/duplicate_finder.py`
2. **Частота проверки**: при каждом пуше в main
3. **Действия при обнаружении**:
   - Предупреждение для незначительных дубликатов
   - Блокировка пуша для критических дубликатов

### Интеграция с GitHub Actions
```yaml
name: Duplicate Check
on: [push, pull_request]
jobs:
  check-duplicates:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Check for duplicates
        run: python scripts/duplicate_finder.py --ci-mode
```
