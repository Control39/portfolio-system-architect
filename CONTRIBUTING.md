# Руководство по вкладу в проект

Спасибо, что хотите внести вклад в `portfolio-system-architect`! Этот проект- **документальный фильм о рождении новой профессии** и каждый вклад становится частью этого повествования.

---

## 🌱 Философия вклада

Здесь мы не просто пишем код. Мы **создаём будущее профессий**. Ваш вклад- это не строка в резюме, а часть истории о том, как меняется роль человека в эпоху ИИ.

**Главное правило:** пишите так, как чувствуете. Делитесь не только успехами, но и сомнениями. Именно в этом рождается новая экспертиза.

---

## 🚀 Как внести вклад

### 1. Форкните репозиторий
Создайте свой форк на SourceCraft.

### 2. Клонируйте
```bash
git clone https://sourcecraft.com/ваш-логин/portfolio-system-architect.git
cd portfolio-system-architect
```

### 3. Создайте ветку
```bash
git checkout -b feature/ваша-идея
```

### 4. Внесите изменения
Вы можете:
- 🐛 **Исправить ошибку** — в коде или документации
- ✨ **Добавить функцию** — новый скрипт, интеграцию, компонент
- 📖 **Улучшить документацию** — сделать её понятнее для других
- 🎨 **Создать артефакт** — новый кейс, диаграмму, демо
- 💭 **Дополнить манифест** — своей историей, инсайтом, вопросом

### 5. Зафиксируйте
```bash
git add .
git commit -m "Добавить: краткое описание вашего вклада"
```

### 6. Отправьте в форк
```bash
git push origin feature/ваша-идея
```

### 7. Создайте Pull Request
В оригинальном репозитории создайте PR из вашей ветки. Опишите, что вы сделали и почему это важно для повествования.

## 🎯 Для первого вклада

Если вы впервые участвуете в opensource-проектах, начните с задач с меткой `good first issue`.

**Что можно сделать прямо сейчас:**
- Найти и исправить опечатку в документации
- Добавить пример использования в README
- Дополнить описание одного из артефактов в `04_ARTIFACTS/`
- Написать, как этот проект изменил ваше мышление

**Как найти задачу:** перейдите в [Issues](https://sourcecraft.com/leadarchitect-ai/portfolio-system-architect/issues) и отфильтруйте по метке `good first issue`.

## 📝 Стиль сообщений о коммитах

Мы используем **повелительное наклонение**, но главное — искренность.

**Хорошие примеры:**
- `Добавить: моя история о том, как я перестал писать код и начал мыслить системами`
- `Поделиться: как манифест помог мне принять свою роль архитектора`
- `Исправить: опечатка в методологии, которая мешала пониманию`

**Не очень:**
- `fix bug`
- `update readme`
