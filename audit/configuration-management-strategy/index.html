    Стратегия управления конфигурациями AI-агентов

    Дата: 2026-04-07 Автор: SourceCraft Code Assistant Agent Статус: Предложено

    1. Обзор проблемы

    В репозитории portfolio-system-architect обнаружены конфигурации AI-агентов, которые исключены из Git через .gitignore, что затрудняет воспроизводимость и совместную работу. При этом некоторые конфигурации содержат устаревшие настройки (например, GPT-5) и не соответствуют используемому российскому стеку AI (GigaChat, YandexGPT, локальные модели).

    2. Результаты аудита

    2.1 Заблокированные директории

    Следующие директории исключены из Git (.gitignore):

    Директория Содержимое Должна ли быть в Git?
    codeassistant/ Навыки (skills) для Code Assistant (9 директорий) Да (навыки — это переиспользуемые инструкции)
    .continue/ Конфигурации Continue.dev (агенты) Да (шаблоны конфигураций без секретов)
    .sourcecraft/ Конфигурации SourceCraft Sites (ci.yaml, sites.yaml) Да (конфигурации CI/CD)
    .gigaide/ Конфигурации GigaIDE (gigaide.properties) Нет (локальные настройки IDE)
    .ai-config/ Конфигурации AI Config (пусто) Нет (локальные настройки)
    .ai-config-gui/ Конфигурации AI Config GUI (пусто) Нет (локальные настройки)

    2.2 Обнаруженные проблемы

    1. Устаревшая модель GPT-5 в .continue/agents/new-config.yaml — исправлено на GigaChat Pro Light.
    2. Отсутствие конфигураций для российского стека — в примерах используются OpenAI и Anthropic, хотя пользователь использует GigaChat, YandexGPT, Ollama.
    3. Полное исключение навыков из Git — навыки codeassistant/skills/ должны быть версионированы.
    4. Смешение секретов и конфигураций — API ключи хранятся в виде плейсхолдеров, что допустимо, но нужно обеспечить безопасность.

    3. Предлагаемая стратегия

    3.1 Принципы

    1. Воспроизводимость: Конфигурации, необходимые для запуска проекта, должны быть в Git.
    2. Безопасность: Секреты (API ключи, пароли) никогда не коммитятся. Используются переменные окружения или менеджер секретов.
    3. Актуальность: Конфигурации должны отражать реально используемый стек технологий.
    4. Модульность: Разделение общих конфигураций и локальных настроек.

    3.2 Изменения в .gitignore

    Предлагается обновить .gitignore следующим образом:

    # AI Agent & Tool Configs (PRIVATE - ИСКЛЮЧИТЬ)
# Исключаем только локальные настройки и секреты
.gigaide/
.ai-config/
.ai-config-gui/
agents/local/config/*.local.yaml
agents/gigacode/config/*.local.yaml

# Включаем в Git (убрать из исключений):
# codeassistant/          <- навыки и правила
# .continue/               <- шаблоны конфигураций
# .sourcecraft/            <- конфигурации CI/CD

    3.3 Структура конфигураций

    .config/ai/                    # Общие конфигурации AI (в Git)
├── gigachat.example.yaml      # Пример конфигурации GigaChat
├── yandexgpt.example.yaml     # Пример конфигурации YandexGPT
├── ollama.example.yaml        # Пример конфигурации Ollama
└── continue.example.yaml      # Пример конфигурации Continue.dev

.continue/agents/              # Конфигурации Continue.dev (в Git)
├── new-config.yaml            # Основная конфигурация (без секретов)
└── templates/                 # Шаблоны для разных моделей

codeassistant/                # Навыки Code Assistant (в Git)
├── skills/
│   ├── architect-analize/
│   ├── code/
│   └── ...
└── rules/ignore-patterns.md   # Правила игнорирования

.sourcecraft/                  # Конфигурации SourceCraft (в Git)
├── ci.yaml
└── sites.yaml

# Локальные настройки (в .gitignore)
.gigaide/gigaide.properties    # Локальные настройки IDE
.ai-config/                    # Персональные настройки AI
.ai-config-gui/                # GUI настройки

    3.4 Миграция

    1. Восстановить codeassistant/ и .continue/ в Git: bash git add .codeassistant .continue .sourcecraft git commit -m "feat: add AI agent configurations to Git"

    2. Создать примеры конфигураций для российского стека в config/ai/ (уже создано).

    3. Обновить документацию по настройке AI-агентов.

    4. Настроить pre-commit hook для проверки наличия секретов в конфигурациях.

    3.5 Безопасность

    • Использовать git-secrets или pre-commit hooks для предотвращения коммита секретов.
    • Хранить секреты в .env (в .gitignore) или в менеджере секретов (HashiCorp Vault, Azure Key Vault).
    • В конфигурациях использовать плейсхолдеры ${API_KEY} с подстановкой через переменные окружения.

    5. Заключение

    Предложенная стратегия обеспечивает воспроизводимость конфигураций AI-агентов, соответствие российскому стеку технологий и безопасное управление секретами. Реализация позволит упростить onboarding новых разработчиков и улучшить управляемость проекта.

    Этот документ является частью аудита конфигураций AI-агентов, выполненного в рамках создания MCP-сервера для портфолио системного архитектора.
