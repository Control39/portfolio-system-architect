
## 🔧 Архитектурные правила для когнитивного агента

### 1. **Правило модульной изоляции**
- Все новые компоненты агента должны создаваться как отдельные модули в соответствующих подкаталогах
- Запрещено изменение основного файла `autonomous_agent.py` для добавления новой функциональности
- Каждый компонент = отдельный файл с одной основной обязанностью

### 2. **Правило архитектурного одобрения**
- Перед реализацией любого компонента необходим архитектурный план
- Все изменения должны быть одобрены вами как когнитивным архитектором
- Шаги: предложение → прототип → одобрение → интеграция

### 3. **Правило синтаксической проверки**
- Любой код до интеграции должен быть проверен на синтаксическую корректность
- Использовать `get_problems` для проверки перед применением изменений

### 4. **Правило партнерского взаимодействия**
- Я как `когнитивный архитектурный партнёр` должен согласовывать все решения с вами
- Никаких автономных решений без вашего одобрения
