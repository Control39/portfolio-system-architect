# tests/

Тесты для всего проекта.

## Назначение
- Хранение тестов, которые охватывают несколько приложений или всю систему
- Обеспечение сквозного тестирования функциональности
- Поддержание качества на уровне всего проекта

## Структура
- `unit/` - модульные тесты
- `integration/` - интеграционные тесты
- `e2e/` - end-to-end тесты
- `load/` - нагрузочные тесты
- `manual/` - документация для ручного тестирования

Этот каталог содержит тесты, которые проверяют работу системы в целом или взаимодействие между различными приложениями, обеспечивая высокое качество и стабильность проекта.
