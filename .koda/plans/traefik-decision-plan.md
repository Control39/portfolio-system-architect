# План принятия решения: API Gateway (Traefik vs nginx vs прямые порты)

**Дата создания:** 15 мая 2026 г.
**Статус:** В ожидании решения
**Приоритет:** Средний

---

## Контекст

Текущее состояние:
- 2 сервиса работают через прямые порты (8001, 8002)
- Traefik не работает из-за проблем с Docker socket в Windows
- Нет единой точки входа для всех сервисов

---

## Критерии принятия решения

### 1. Количество сервисов

| Сервисов | Решение | Обоснование |
|----------|---------|-------------|
| 1-3 | Прямые порты | Простота, минимум конфигурации |
| 4-7 | nginx | Баланс между простотой и функциональностью |
| 8+ | Traefik | Auto-discovery, динамическая конфигурация |

**Текущее:** 2 сервиса → **Прямые порты** ✅

---

### 2. Стадия проекта

| Стадия | Решение | Обоснование |
|--------|---------|-------------|
| Разработка | Прямые порты | Быстрая отладка |
| Staging | nginx | Близко к production |
| Production | Traefik/nginx | Единая точка входа, безопасность |

**Текущее:** Разработка → **Прямые порты** ✅

---

### 3. Требования к безопасности

| Требование | Прямые порты | nginx | Traefik |
|------------|--------------|-------|---------|
| TLS/HTTPS | ❌ Сложно | ✅ Средне | ✅ Легко |
| Rate limiting | ❌ Нет | ✅ Да | ✅ Да |
| Authentication | ❌ Нет | ✅ Да | ✅ Да |
| WAF | ❌ Нет | ✅ Да | ✅ Да |

**Текущее:** Нет требований → **Прямые порты** ✅

---

### 4. Окружение разработки

| Окружение | Решение | Обоснование |
|-----------|---------|-------------|
| Windows (Docker Desktop) | nginx/Прямые | Traefik нестабилен |
| Linux (Docker Engine) | Traefik | Полная поддержка |
| WSL2 | nginx/Traefik | Зависит от конфигурации |

**Текущее:** Windows → **nginx или прямые порты** ✅

---

## Варианты действий

### Вариант A: Оставить прямые порты (рекомендуется сейчас)

**Условия:**
- ✅ Сервисов < 5
- ✅ Стадия: разработка
- ✅ Нет требований к безопасности
- ✅ Окружение: Windows

**Действия:**
1. Закоммитить текущие изменения
2. Документировать порты в README
3. Добавить `.env` с портами для консистентности
4. Вернуться к вопросу при появлении 5+ сервисов

**Риски:**
- Конфликты портов при запуске нескольких версий
- Сложность управления при росте числа сервисов

---

### Вариант B: Перейти на nginx (ближайшее будущее)

**Условия:**
- ⚠️ Сервисов 4-7
- ⚠️ Подготовка к staging
- ⚠️ Первые требования к безопасности

**Действия:**
1. Создать `docker/nginx/nginx.conf`
2. Добавить сервис `nginx` в `docker-compose.yml`
3. Настроить маршруты для всех сервисов
4. Протестировать в staging
5. Перейти на production

**Время реализации:** 2-4 часа

---

### Вариант C: Исправить Traefik (для production)

**Условия:**
- ✅ Сервисов 8+
- ✅ Подготовка к production
- ✅ Требуется auto-discovery
- ✅ Окружение: Linux/Kubernetes

**Действия:**
1. Отладить Docker socket в Windows (или перейти на Linux)
2. Настроить Traefik с правильными метками
3. Добавить TLS (Let's Encrypt)
4. Настроить health checks
5. Протестировать load balancing

**Время реализации:** 1-2 дня

---

## Решение (на 15 мая 2026 г.)

**Выбрано:** Вариант A — **Оставить прямые порты**

**Обоснование:**
1. Только 2 сервиса (decision-engine, ml-model-registry)
2. Стадия: активная разработка
3. Окружение: Windows (проблемы с Traefik)
4. Нет требований к безопасности на данный момент
5. Прямой доступ упрощает отладку

**Критерии для перехода:**
- [ ] Появление 5+ сервисов
- [ ] Переход на staging/production
- [ ] Требования к безопасности (TLS, rate limiting)
- [ ] Переход на Linux/Kubernetes
