## ⚖️ Варианты решения

### Вариант A: Полная унификация (Жёсткий)

**Действия:**
1. Все сервисы → FastAPI
2. Единая структура `src/` с роутами/сервисами/утилитами
3. Единый способ импорта (pyproject.toml + pip install -e)
4. Стандартный Dockerfile для всех

**Время:** 7-10 часов

**Плюсы:**
- ✅ Максимальная предсказуемость
- ✅ Проще нанимать (стандартная структура)
- ✅ Легко генерировать новые сервисы

**Минусы:**
- ❌ **Высокий риск сломать работающее** (production!)
- ❌ decision_engine переписать полностью
- ❌ portfolio_organizer (Flask → FastAPI) — 2-3 часа работы
- ❌ cognitive-agent — сложная логика, не подходит для FastAPI

**Последствия:**
- 1-2 дня тестирования после изменений
- Высокий риск регрессии
- Нарушение принципа "не ломать работающее"

---

### Вариант B: Гибридная унификация (Умеренный)

**Действия:**
1. Оставить фреймворки как есть (FastAPI/Flask/кастомный)
2. Стандартизировать **только entry point** (`main.py` или `app.py` в корне)
3. Единый шаблон Dockerfile
4. Единый README-шаблон
5. AI Config Manager как обязательный (заменить decision_engine/loader.py)

**Время:** 3-4 часа

**Плюсы:**
- ✅ Низкий риск (не меняем внутреннюю логику)
- ✅ Быстро (entry point + Dockerfile)
- ✅ Сохраняется гибкость (каждый сервис использует свой фреймворк)
- ✅ Решение проблемы AI Config Manager

**Минусы:**
- ⚠️ Всё ещё разная внутренняя структура
- ⚠️ Разные подходы к путям (но это не критично)

**Последствия:**
- Минимальное тестирование (только entry point)
- Устойчиво к регрессии
- Компромисс между стандартом и гибкостью

---

### Вариант C: Минимальная унификация (Консервативный)

**Действия:**
1. Только **добавить main.py** там, где его нет (5 сервисов)
2. Только **добавить Dockerfile** там, где его нет (4 сервиса)
3. Оставить всё остальное как есть
4. Создать **скрипт генерации** для **новых** сервисов

**Время:** 2 часа

**Плюсы:**
- ✅ Минимальный риск
- ✅ Быстро
- ✅ Новые сервисы будут стандартизированы
- ✅ Старые остаются как есть (не ломать работающее)

**Минусы:**
- ⚠️ Структура остаётся неоднородной
- ⚠️ Новые разработчики будут путаться

**Последствия:**
- Самый безопасный вариант
- Но проблема неоднородности остаётся
