## Критерии качества README

Каждый README должен содержать:

1. ✅ **Название и назначение** — что делает сервис
2. ✅ **Ключевые возможности** — список основных функций
3. ✅ **API endpoints** — если есть HTTP API
4. ✅ **Dependencies** — основные зависимости
5. ✅ **Deployment info** — как запустить (Docker/port)
6. ✅ **Contributing guide** — как вносить изменения
