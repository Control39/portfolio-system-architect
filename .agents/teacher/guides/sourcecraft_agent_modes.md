# Руководство по режимам SourceCraft Agent

## Введение

SourceCraft Agent поддерживает несколько режимов работы, каждый из которых оптимизирован для определенного типа задач. Понимание и правильное использование этих режимов позволяет максимально эффективно взаимодействовать с агентом.

## Обзор режимов

| Режим | Иконка | Описание | Основное применение |
|-------|--------|----------|---------------------|
| **🏗️ Architect** | 🏗️ | Архитектурное проектирование и планирование | Проектирование систем, создание технических спецификаций |
| **💻 Code** | 💻 | Написание и модификация кода | Реализация функций, рефакторинг, исправление ошибок |
| **❓ Ask** | ❓ | Получение объяснений и документации | Поиск информации, анализ кода, обучение |
| **🪲 Debug** | 🪲 | Отладка и диагностика проблем | Поиск ошибок, анализ логов, устранение неполадок |
| **🪃 Orchestrator** | 🪃 | Оркестрация сложных многоэтапных проектов | Управление workflow, координация между режимами |

## 🏗️ Architect Mode (Режим архитектора)

### Когда использовать
- **Проектирование новой системы** или компонента
- **Создание технической документации** и спецификаций
- **Планирование миграции** или рефакторинга
- **Оценка архитектурных решений** и выбор технологий
- **Создание диаграмм** и схем взаимодействия

### Возможности
- **Генерация ADR** (Architecture Decision Records)
- **Создание диаграмм** в формате Mermaid, PlantUML
- **Анализ компромиссов** между различными решениями
- **Планирование этапов реализации**
- **Оценка рисков** и зависимостей

### Примеры команд
```bash
# Спроектировать микросервисную архитектуру для системы управления задачами
"Спроектируй микросервисную архитектуру для системы управления задачами с использованием FastAPI и PostgreSQL"

# Создать ADR для выбора базы данных
"Создай ADR для выбора между PostgreSQL и MongoDB для системы аналитики"

# Спроектировать схему взаимодействия компонентов
"Создай диаграмму взаимодействия компонентов для системы обработки платежей"
```

### Ограничения
- Не может редактировать файлы кода (только `.md` файлы)
- Фокусируется на высокоуровневом дизайне, а не на реализации

## 💻 Code Mode (Режим программиста)

### Когда использовать
- **Реализация новых функций**
- **Рефакторинг существующего кода**
- **Исправление ошибок** и багов
- **Написание тестов**
- **Создание скриптов** и утилит

### Возможности
- **Написание кода** на любом языке программирования
- **Рефакторинг** с сохранением функциональности
- **Оптимизация производительности**
- **Создание тестов** (unit, integration, e2e)
- **Генерация документации** к коду

### Примеры команд
```python
# Реализовать функцию аутентификации
"Реализуй функцию аутентификации с JWT токенами на FastAPI"

# Рефакторинг существующего кода
"Рефактори эту функцию, чтобы уменьшить цикломатическую сложность"

# Создать тесты для модуля
"Напиши unit-тесты для модуля обработки платежей"
```

### Особенности
- Может редактировать любые файлы с кодом
- Поддерживает все популярные языки программирования
- Интегрируется с линтерами и форматтерами

## ❓ Ask Mode (Режим вопросов)

### Когда использовать
- **Поиск информации** о технологиях или библиотеках
- **Объяснение концепций** или паттернов
- **Анализ существующего кода**
- **Получение рекомендаций** по лучшим практикам
- **Обучение** новым технологиям

### Возможности
- **Объяснение сложных концепций** простыми словами
- **Анализ преимуществ и недостатков** технологий
- **Предоставление примеров** использования
- **Сравнение альтернативных решений**
- **Ответы на технические вопросы**

### Примеры команд
```bash
# Объяснить концепцию
"Объясни, что такое Event Sourcing и когда его стоит использовать"

# Сравнить технологии
"Сравни преимущества и недостатки GraphQL vs REST API"

# Проанализировать код
"Проанализируй этот код Python и предложи улучшения"
```

### Особенности
- Не вносит изменения в код
- Фокусируется на объяснениях и рекомендациях
- Использует простой и понятный язык

## 🪲 Debug Mode (Режим отладки)

### Когда использовать
- **Диагностика ошибок** и исключений
- **Анализ логов** и трассировок
- **Поиск утечек памяти** или проблем производительности
- **Воспроизведение багов**
- **Оптимизация проблемного кода**

### Возможности
- **Анализ стектрейсов** и ошибок
- **Предложение исправлений** для багов
- **Диагностика проблем производительности**
- **Анализ логов** и метрик
- **Создание тестов для воспроизведения** проблем

### Примеры команд
```bash
# Проанализировать ошибку
"Проанализируй эту ошибку и предложи исправление: 'Connection refused to database'"

# Найти утечку памяти
"Помоги найти утечку памяти в этом Python скрипте"

# Оптимизировать медленный запрос
"Проанализируй этот SQL запрос и предложи оптимизацию"
```

### Особенности
- Специализируется на решении проблем
- Использует систематический подход к отладке
- Может добавлять логирование и инструментирование

## 🪃 Orchestrator Mode (Режим оркестратора)

### Когда использовать
- **Управление сложными многоэтапными проектами**
- **Координация работы между разными режимами**
- **Планирование долгосрочных задач**
- **Автоматизация workflow**
- **Интеграция различных инструментов и систем**

### Возможности
- **Разбиение сложных задач** на подзадачи
- **Координация выполнения** между режимами
- **Управление зависимостями** между задачами
- **Мониторинг прогресса** выполнения
- **Автоматическое переключение** между режимами по необходимости

### Примеры команд
```bash
# Оркестрировать развертывание приложения
"Оркестрируй развертывание веб-приложения на Azure с CI/CD"

# Управлять миграцией базы данных
"Спланируй и выполни миграцию с SQLite на PostgreSQL"

# Координировать рефакторинг большой кодовой базы
"Оркестрируй рефакторинг монолита на микросервисы"
```

### Особенности
- Высокоуровневое управление задачами
- Автоматическое определение нужных режимов
- Координация между различными специализациями

## Переключение между режимами

### Автоматическое переключение
SourceCraft Agent может автоматически определять нужный режим на основе:
1. **Типа задачи** (проектирование, кодирование, отладка и т.д.)
2. **Контекста разговора** (предыдущие сообщения)
3. **Типов файлов** в проекте
4. **Явных указаний** пользователя

### Ручное переключение
```bash
# Явное указание режима
"В режиме архитектора спроектируй систему..."

# Использование ключевых слов
"Отлади эту ошибку" → автоматически переключается в Debug mode
"Объясни концепцию" → автоматически переключается в Ask mode
"Напиши код для..." → автоматически переключается в Code mode
```

### Команды переключения
- `"Переключись в режим архитектора"`
- `"Используй режим отладки"`
- `"Работай как оркестратор"`

### Пример 3: Миграция технологии
1. **🏗️ Architect**: Планирование миграции
2. **🪃 Orchestrator**: Координация этапов миграции

## Интеграция с инструментами разработки

### VS Code
```json
{
  "sourcecraft.autoModeDetection": true,
  "sourcecraft.defaultMode": "code",
  "sourcecraft.modeSwitchShortcuts": {
    "architect": "ctrl+shift+a",
    "code": "ctrl+shift+c",
    "ask": "ctrl+shift+q",
    "debug": "ctrl+shift+d",
    "orchestrator": "ctrl+shift+o"
  }
}
```

### CLI команды
```bash
# Запуск в определенном режиме
sourcecraft --mode=architect "спроектируй систему"

# Автоматическое определение режима
sourcecraft "помоги с этой ошибкой"

# Переключение режима во время работы
/mode debug
```

## Советы по эффективному использованию

### 1. Начинайте с правильного режима
- Для новых проектов: **🏗️ Architect**
- Для реализации: **💻 Code**
- Для обучения: **❓ Ask**
- Для проблем: **🪲 Debug**
- Для сложных проектов: **🪃 Orchestrator**

### 2. Используйте явные указания
```bash
# Хорошо:
"В режиме архитектора спроектируй..."

# Плохо:
"Спроектируй..." (агент может неправильно определить режим)
```

### 3. Комбинируйте режимы для сложных задач
```bash
# Сначала проектирование, затем реализация
"Спроектируй, а затем реализуй систему аутентификации"
```

### 4. Используйте контекст
```bash
# Предоставляйте контекст для лучшего понимания
"У меня есть FastAPI приложение с проблемой производительности. В режиме отладки помоги найти bottleneck"
```

## Часто задаваемые вопросы

### Q: Какой режим использовать по умолчанию?
**A:** Зависит от вашей основной деятельности:
- Разработчикам: **💻 Code**
- Архитекторам: **🏗️ Architect**
- Техлидам: **🪃 Orchestrator**
- Исследователям: **❓ Ask**

### Q: Можно ли использовать несколько режимов одновременно?
**A:** Нет, агент работает в одном режиме за раз, но может быстро переключаться между ними.

### Q: Как узнать текущий режим?
**A:** В интерфейсе отображается иконка текущего режима. Также можно спросить: "В каком режиме ты сейчас работаешь?"

### Q: Что делать, если агент неправильно определил режим?
**A:** Явно укажите нужный режим: "Переключись в режим архитектора"

## Расширенные возможности

### Пользовательские режимы
Вы можете создавать собственные режимы, расширяя базовые:

```yaml
# .codeassistant/custom_modes.yaml
custom_modes:
  security_auditor:
    base_mode: "debug"
    description: "Режим аудита безопасности"
    capabilities:
      - security_analysis
      - vulnerability_scanning
      - compliance_checking
    file_patterns:
      - "**/*.py"
      - "**/*.js"
      - "**/*.yaml"
```

### Интеграция с CI/CD
Режимы могут быть интегрированы в pipeline:

```yaml
# .github/workflows/code-review.yml
jobs:
  architecture_review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Архитектурный ревью
        run: |
          sourcecraft --mode=architect \
            "проанализируй архитектуру проекта на соответствие best practices"
```

### Статистика использования
```bash
# Просмотр статистики использования режимов
sourcecraft --stats

# Результат:
# 🏗️ Architect: 35% (14 часов)
# 💻 Code: 45% (18 часов)
# ❓ Ask: 10% (4 часа)
# 🪲 Debug: 8% (3 часа)
# 🪃 Orchestrator: 2% (1 час)
```

## Заключение

Правильное использование режимов SourceCraft Agent значительно повышает эффективность работы. Ключевые принципы:

1. **Выбирайте режим осознанно** в зависимости от задачи
2. **Не бойтесь переключаться** между режимами
3. **Используйте явные указания** для точности
4. **Комбинируйте режимы** для сложных задач
5. **Изучайте статистику** для оптимизации workflow

Помните: каждый режим - это специализированный инструмент. Как мастер использует разные инструменты для разных задач, так и вы должны использовать подходящие режимы для максимальной эффективности.
