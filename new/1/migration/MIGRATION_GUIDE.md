# 📖 Руководство по миграции структуры проекта

## Быстрый старт

### Вариант 1: Полная автоматическая миграция (рекомендуется)

```powershell
# 1. Просмотр плана (без изменений)
.\scripts\migration\Migrate-Structure.ps1 -WhatIf

# 2. Выполнение миграции
.\scripts\migration\Migrate-Structure.ps1
# Ответьте 'y' на все вопросы
```

### Вариант 2: Поэтапное выполнение

```powershell
# Этап 0: Подготовка
.\scripts\migration\Migrate-Structure.ps1 -Step 0

# Этап 1: Создание структуры
.\scripts\migration\Migrate-Structure.ps1 -Step 1

# Этап 2: Перемещение файлов
.\scripts\migration\Migrate-Structure.ps1 -Step 2

# Этап 3: Обновление путей
.\scripts\migration\Migrate-Structure.ps1 -Step 3

# Этап 4: Создание конфигов
.\scripts\migration\Migrate-Structure.ps1 -Step 4

# Этап 5: Очистка
.\scripts\migration\Migrate-Structure.ps1 -Step 5

# Этап 6: Проверка
.\scripts\migration\Migrate-Structure.ps1 -Step 6

# Этап 7: Финализация
.\scripts\migration\Migrate-Structure.ps1 -Step 7
```

## Откат изменений

### Откат последнего этапа
```powershell
.\scripts\migration\Migrate-Structure.ps1 -Rollback
```

### Откат через Git
```powershell
# Просмотр истории
git log --oneline

# Откат к checkpoint
git reset --hard <commit-hash>

# Или откат последнего commit
git reset --hard HEAD~1
```

## Проверка после миграции

### 1. Проверка структуры
```powershell
# Проверка ключевых директорий
Test-Path "src/core/utilities"
Test-Path "src/infrastructure/security"
Test-Path "config/templates"
```

### 2. Проверка импортов
```powershell
# Попытка загрузить модуль
Import-Module .\arch-compass.psd1 -Force
```

### 3. Запуск тестов
```powershell
# Если есть Pester тесты
Invoke-Pester tests/
```

## Что происходит на каждом этапе

### Этап 0: Подготовка
- ✅ Проверка чистоты рабочей директории
- ✅ Создание backup ветки
- ✅ Создание checkpoint commit

**Время**: ~2 минуты  
**Откат**: `git reset --hard HEAD~1`

### Этап 1: Создание структуры
- ✅ Создание всех новых директорий
- ✅ Создание .gitkeep файлов для пустых директорий

**Время**: ~5 минут  
**Откат**: Удаление созданных директорий вручную или через git

### Этап 2: Перемещение файлов
- ✅ Перемещение файлов через `git mv` (сохраняет историю)
- ✅ Переименование файлов
- ✅ Реорганизация структуры

**Время**: ~10 минут  
**Откат**: `git reset --hard HEAD~1`

### Этап 3: Обновление путей
- ✅ Обновление Import-Module путей
- ✅ Обновление $PSScriptRoot ссылок
- ✅ Обновление относительных путей

**Время**: ~5 минут  
**Откат**: `git reset --hard HEAD~1`

### Этап 4: Создание конфигов
- ✅ Создание .vscode/settings.json
- ✅ Создание .devcontainer/devcontainer.json
- ✅ Обновление .gitignore

**Время**: ~3 минуты  
**Откат**: Удаление созданных файлов

### Этап 5: Очистка
- ✅ Удаление пустых старых директорий
- ✅ Удаление дубликатов

**Время**: ~2 минуты  
**Откат**: `git reset --hard HEAD~1`

### Этап 6: Проверка
- ✅ Проверка существования ключевых файлов
- ✅ Проверка синтаксиса PowerShell
- ✅ Валидация структуры

**Время**: ~5 минут  
**Откат**: Не требуется (только проверка)

### Этап 7: Финализация
- ✅ Git add всех изменений
- ✅ Создание commit с описанием миграции

**Время**: ~1 минута  
**Откат**: `git reset --hard HEAD~1`

## Часто задаваемые вопросы

### Q: Что делать, если миграция прервалась?
**A**: Используйте `git status` для проверки состояния, затем либо продолжите с нужного этапа (`-Step N`), либо откатитесь через `git reset --hard HEAD~1`.

### Q: Можно ли выполнить миграцию частично?
**A**: Да, используйте параметр `-Step N` для выполнения конкретного этапа.

### Q: Что делать, если после миграции что-то не работает?
**A**: 
1. Проверьте логи ошибок
2. Убедитесь, что все пути обновлены (`Step-3`)
3. Проверьте синтаксис PowerShell файлов
4. При необходимости откатитесь через git

### Q: Сохранится ли история Git после миграции?
**A**: Да, скрипт использует `git mv` для перемещения файлов, что сохраняет историю.

### Q: Нужно ли делать backup вручную?
**A**: Нет, скрипт автоматически создает backup ветку на этапе 0.

## Рекомендации

1. **Перед миграцией**: Убедитесь, что все изменения закоммичены
2. **Во время миграции**: Не прерывайте процесс, дождитесь завершения этапа
3. **После миграции**: Запустите тесты и проверьте работоспособность
4. **Если что-то пошло не так**: Используйте git для отката, не пытайтесь исправить вручную
