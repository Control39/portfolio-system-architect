## Ключевые принципы безопасности

1. **Git как система контроля версий** - каждое изменение отслеживается
2. **Поэтапное выполнение** - можно остановиться на любом этапе
3. **Возможность отката** - каждый этап можно откатить через git
4. **Проверка после каждого этапа** - тестирование работоспособности
5. **Резервные копии** - создание backup перед критическими операциями

## Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Потеря файлов | Низкая | Высокое | Git commit перед началом |
| Сломанные импорты | Средняя | Высокое | Автоматическое обновление путей |
| Потеря истории Git | Низкая | Среднее | Использование git mv |
| Конфликты путей | Средняя | Среднее | Поэтапная миграция |
| Сломанные тесты | Средняя | Среднее | Запуск тестов после каждого этапа |

## Оценка времени

- **Этап 0**: 5 минут (подготовка)
- **Этап 1**: 10 минут (создание структуры)
- **Этап 2**: 15 минут (перемещение файлов)
- **Этап 3**: 20 минут (обновление путей)
- **Этап 4**: 15 минут (новые файлы)
- **Этап 5**: 10 минут (очистка)
- **Этап 6**: 15 минут (проверка)
- **Этап 7**: 5 минут (финализация)

**Итого**: ~1.5 часа с учетом проверок
