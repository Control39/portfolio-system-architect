# Daily Checklist: Операционные проверки

> **Частота:** Ежедневно (утро)
> **Время выполнения:** ~10 минут
> **Автор:** Ekaterina Kudelya
> **Обновлено:** 18 мая 2026

---

## 📋 Утренняя проверка

### 1. Статус всех сервисов (2 мин)

```bash
.\navigate.ps1 -Status
```

**Ожидаемый результат:** Все 15 сервисов в статусе ✅

---

### 2. Проверка критических endpoints (2 мин)

```bash
# Auth Service
curl http://localhost:8100/health

# API Gateway
curl http://localhost:8080/health

# PostgreSQL
docker exec postgresql psql -U postgres -c "SELECT 1"

# Redis
redis-cli ping
```

**Ожидаемый результат:** Все возвращают `OK` / `healthy`

---

### 3. Мониторинг (3 мин)

Открыть Grafana: http://localhost:3000

**Проверить:**
- CPU usage <80%
- Memory usage <85%
- Disk usage <70%
- Нет critical alerts

---

### 4. Логи ошибок (2 мин)

```bash
# Проверить последние ошибки
docker-compose logs --tail=100 | grep -i "error\|exception\|fatal" | tail -20
```

**Ожидаемый результат:** Нет повторяющихся ошибок

---

### 5. Бэкапы (1 мин)

```bash
# Проверить наличие последнего бэказа
ls -lh /backups/latest.*
```

**Ожидаемый результат:** Бэказ за последние 24 часа

---

## 📊 Еженедельные проверки (понедельник)

### 1. Покрытие тестами

```bash
pytest --cov=apps --cov-report=term-missing
```

**Цель:** >=80% для всех сервисов

---

### 2. Security скан

```bash
trivy fs .
bandit -r apps/ src/
```

**Цель:** 0 критических уязвимостей

---

### 3. Обновление зависимостей

```bash
pip list --outdated
```

**Действие:** Проверить важные обновления

---

## 📈 Ежемесячные проверки

### 1. Анализ инцидентов

- Сколько было P0/P1/P2/P3
- Среднее время восстановления (MTTR)
- Что можно улучшить

---

### 2. Оптимизация затрат

- Проверить использование ресурсов
- Удалить неиспользуемые контейнеры
- Очистить старые логи

---

## 🚨 Когда тревожиться

| Метрика | Норма | Тревога | Критично |
|---------|-------|---------|----------|
| CPU usage | <60% | 60-80% | >80% |
| Memory | <70% | 70-85% | >85% |
| Disk | <60% | 60-75% | >75% |
| Error rate | 0% | 0.1-1% | >1% |
| Response time | <200ms | 200-500ms | >500ms |

---

## 📞 Кому сообщать

| Проблема | Кто | Когда |
|----------|-----|-------|
| Единичная ошибка | Self | Сразу исправить |
| Повторяющиеся ошибки | Tech Lead | При 3+ случаях |
| Падение сервиса | On-call | Немедленно |
| Несколько сервисов | CTO | P0 инцидент |
