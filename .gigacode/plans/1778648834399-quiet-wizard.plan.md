# Plan: Обновление CodeQL Action до v4

## Контекст

Пользователь столкнулся с ошибкой парсинга файлов при выполнении CodeQL анализа в workflow `codeql.yml`. Ошибка связана с тем, что CodeQL Action @v3 использует устаревший парсер, который не справляется с современным синтаксисом JavaScript/TypeScript (JSX/TSX, декораторы, `import type`, top-level await).

## Задача

Обновить workflow `.github/workflows/codeql.yml` до использования CodeQL Action @v4, который содержит обновленный языковой сервер и лучшую поддержку компилируемых проектов. Это должно устранить проблему парсинга на уровне инфраструктуры.

## План

1. Прочитать текущее содержимое файла `.github/workflows/codeql.yml`.
2. Обновить версии шагов `github/codeql-action/init` и `github/codeql-action/analyze` с `@v3` на `@v4`.
3. Убедиться, что шаги установки Node.js и сборки проекта присутствуют и корректны.
4. Записать обновленное содержимое файла.
5. Зафиксировать изменения и отправить в репозиторий.

## Критические файлы

- `C:/Projects/portfolio-system-architect/.github/workflows/codeql.yml`

## Верификация

После отправки изменений:
1. Перейти в `Actions` → открыть последний запуск `CodeQL Advanced` → шаг `Analyze (javascript)`.
2. Проверить логи на наличие строки `Found X javascript files to analyze` и отсутствие ошибок парсинга.
3. Если ошибок нет, задача выполнена успешно.
