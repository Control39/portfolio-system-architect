# Рефакторинг проекта: Реализация рекомендаций

## Обзор выполненных изменений

1.  **Настройка PYTHONPATH и сборки в pyproject.toml:**
    *   Обновлены секции `[tool.hatch.build.targets.wheel]` и `[tool.hatch.build.targets.sdist]` в корневом `pyproject.toml`, чтобы включить в пакет как `src`, так и `apps`.
    *   Удалено устаревшее поле `pythonpath` из конфигурации `pytest`, так как правильная настройка `hatch` и `PYTHONPATH` делает его избыточным.

2.  **Унификация структуры приложений и удаление дублирования:**
    *   Подтверждено, что код в приложении `it-compass` и других сервисах уже находится в правильной папке `apps/<service>/src/`, что соответствует целевой структуре.
    *   Обнаружены и отмечены для удаления файлы `requirements.txt` во многих приложениях (`apps/*/requirements.txt`), которые дублируют зависимости из корневого `pyproject.toml`.

3.  **Управление зависимостями:**
    *   Подтверждено, что `hatch` является основным инструментом управления зависимостями, как и было рекомендовано.

## Текущее состояние и следующие шаги

**Важно:** Изменения в `pyproject.toml` были успешно применены. Однако, удаление файлов `requirements.txt` из каталогов приложений не удалось из-за технических ограничений на выполнение командных строковых операций.

Для завершения рефакторинга необходимо вручную выполнить следующие команды в корневом каталоге проекта:

```bash
# Удалить все файлы requirements.txt из папок приложений
git rm apps/*/requirements.txt

# Зафиксировать изменения
git add pyproject.toml
git commit -m "refactor(architecture): унифицировать структуру, удалить дублирующие requirements.txt, настроить PYTHONPATH через hatch"
```

После этого проект будет соответствовать рекомендованной архитектуре с централизованным управлением зависимостями и корректными путями импорта.
