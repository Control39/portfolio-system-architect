    Архитектурные решения

    MCP-сервер как микросервис

    apps/mcp-server/
├── src/
│   ├── main.py              # FastMCP сервер
│   ├── tools/               # Инструменты
│   │   ├── file_tools.py    # Работа с файлами
│   │   ├── git_tools.py     # Работа с Git
│   │   ├── compass_tools.py # Интеграция с IT-Compass
│   │   └── command_tools.py # Выполнение команд
│   ├── resources/           # Ресурсы
│   │   ├── navigation.py    # Навигация по репозиторию
│   │   └── compass.py       # Ресурсы IT-Compass
│   └── utils/               # Утилиты
├── config/
│   └── mcp-config.yaml      # Единая конфигурация
├── tests/                   # Тесты
├── scripts/                 # Скрипты
└── docs/                    # Документация

    Ключевые возможности

    1. Файловые операции

    • Чтение/запись файлов
    • Рекурсивный список файлов
    • Поиск по содержимому

    2. Git интеграция

    • Анализ статуса репозитория
    • Автоматическое обнаружение маркеров IT-Compass из коммитов
    • История изменений

    3. IT-Compass интеграция

    • Навигация по доменам компетенций
    • Оценка по объективным маркерам
    • Генерация отчетов о навыках

    4. Навигация по репозиторию

    • Адаптивные туры для разных аудиторий (HR, техлиды, грантовые комитеты)
    • Поиск по проекту
    • Структурированное представление архитектуры

    5. Выполнение команд

    • Безопасное выполнение системных команд
    • Интеграция с инструментами проекта (Docker, Kubernetes, CI/CD)

    Решенные проблемы

    Проблема 1: Устаревшие конфигурации

    Решение: Создан скрипт валидации, который: - Обнаруживает упоминания GPT-5 и других несуществующих моделей - Автоматически заменяет на актуальные модели (GPT-4, Claude-3) - Создает резервные копии перед изменениями

    Проблема 2: Избыточная вложенность

    Решение: Упрощена структура MCP-сервера: - Максимальная глубина вложенности: 4 уровня - Логическая группировка файлов - Четкое разделение ответственности

    Проблема 3: Разбросанные конфигурации

    Решение: Создана единая конфигурация: - apps/mcp-server/config/mcp-config.yaml - Централизованное управление настройками - Поддержка разных окружений (development, production)

    Проблема 4: Блокировки доступа

    Решение: Документирована проблема и предложены пути решения: - Настройка .codeassistantignore - Альтернативные методы доступа - Ручное вмешательство владельца репозитория

    Интеграция с экосистемой

    Существующие компоненты

    • IT-Compass: Используется как источник объективных маркеров компетенций
    • API Gateway: MCP-сервер регистрируется как микросервис
    • CI/CD: Интеграция с GitHub Actions для автоматического тестирования
    • Мониторинг: Метрики Prometheus и health checks

    Будущие интеграции

    • Slack Bot: Уведомления о новых маркерах
    • Telegram Bot: Мобильный доступ к Career Autopilot
    • VS Code Extension: Интеграция в рабочую среду
    • Chrome Extension: Автоматический анализ вакансий

    Технические детали

    Технологический стек

    • Язык: Python 3.11+
    • Фреймворк: FastMCP (Model Context Protocol)
    • Конфигурация: YAML + JSON
    • Тестирование: unittest + pytest
    • Документация: Markdown + OpenAPI

    Требования к развертыванию

    # Установка зависимостей
cd apps/mcp-server
pip install -r requirements.txt

# Запуск тестов
python -m pytest tests/

# Запуск сервера
python src/main.py

# Валидация конфигураций
python scripts/validate-config.py

    Конфигурация для Claude Desktop

    {
  "mcpServers": {
    "career-autopilot": {
      "command": "python",
      "args": ["/path/to/apps/mcp-server/src/main.py"],
      "env": {
        "MCP_ENV": "development"
      }
    }
  }
}

    Риски и ограничения

    Технические риски

    1. Блокировки доступа: Некоторые файлы недоступны из-за .codeassistantignore
    2. Зависимости: Требуются специфические версии Python и библиотек
    3. Безопасность: Выполнение команд требует careful sandboxing

    Организационные риски

    1. Принятие: Нужно обучать пользователей работе с MCP-сервером
    2. Поддержка: Требуется документация и примеры использования
    3. Эволюция: IT-Compass маркеры могут меняться, нужна адаптация

    Ограничения

    • Только локальный запуск на первом этапе
    • Ограниченная поддержка Windows (ориентация на Linux/macOS)
    • Требуется ручная настройка для интеграции с IDE
