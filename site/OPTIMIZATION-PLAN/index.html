    План очистки и оптимизации репозитория

    Дата: 22 апреля 2026 г. Цель: Улучшить читаемость, упростить навигацию, убрать загромождение Принцип: Минимальные изменения с максимальным эффектом

    📋 Резюме

    Категория Проблема Решение Приоритет
    Корневые файлы 10+ временных/отладочных файлов в корне Переместить в tools/ или удалить 🔴 Высокий
    Дубликаты 2 README, 2 ARCHITECTURE.md, 10+ отчётов Объединить или переместить 🟡 Средний
    AI-артефакты 4 папки AI-инструментов в корне Сгруппировать в tools/ai-agents/ 🟡 Средний
    Кэши и артефакты .pytest_cache/, cache/, site/, plans/ Добавить в .gitignore 🟢 Низкий
    Документация 30+ подпапок, разбросанные ADR Консолидировать и индексировать 🟡 Средний

    🔴 Приоритет 1: Очистка корневого каталога (2 часа)

    1.1 Переместить временные/отладочные файлы

    Файл Действие Аргументация
    server.py Переместить в tools/dev-tools/server.py Временный HTTP-сервер, не часть продакшена
    test.py Удалить Неименованный тестовый файл, дублирует tests/
    test_rag_api.py Переместить в tests/manual/ Ручной тест, не входит в CI
    test_rag_simple.py Переместить в tests/manual/ Ручной тест, не входит в CI
    index.html Переместить в docs/archive/ Статичный файл без контекста
    refactor.sh Удалить или переместить в scripts/ Скрипт без документации
    KODA-OPTIMIZATION.md Удалить (зафиксировано в коммите) Временный отчёт, цель достигнута

    Команды:

    # Создать папки
mkdir -p tools/dev-tools tests/manual docs/archive scripts

# Переместить файлы
mv server.py tools/dev-tools/
mv test_rag_api.py test_rag_simple.py tests/manual/
mv index.html docs/archive/
mv refactor.sh scripts/  # или rm refactor.sh

# Удалить временные
rm -f test.py KODA-OPTIMIZATION.md

    Ожидаемый результат: - Корень уменьшится на 7 файлов - Улучшится читаемость ls -la - Исчезнут «заброшенные» артефакты

    1.2 Переместить AI-артефакты (опционально)

    Папка Действие Аргументация
    .gigacode/ Переместить в tools/ai-agents/gigacode/ Инструмент, не часть проекта
    .gigaide/ Удалить или переместить Пустая папка (по структуре)
    .continue/ Переместить в tools/ai-agents/continue/ Настройка IDE, не код
    .sourcecraft/ Переместить в tools/ai-agents/sourcecraft/ Конфигурация инструмента
    cache/ Добавить в .gitignore Кэш, генерируется автоматически
    plans/ Переместить в tools/ai-agents/plans/ Временные планы ИИ
    reports/ Переместить в docs/reports/ Генерируемые отчёты
    site/ Добавить в .gitignore Сгенерированный сайт

    Команды:

    # Создать папку
mkdir -p tools/ai-agents

# Переместить
mv .gigacode .continue .sourcecraft plans tools/ai-agents/
mv reports docs/reports/
mv site/ docs/archive/

# Добавить в .gitignore
echo "cache/" >> .gitignore
echo "site/" >> .gitignore

    Ожидаемый результат: - Корень уменьшится на 6 папок - Все AI-инструменты в одном месте - Чёткое разделение: «код проекта» vs «инструменты разработки»

    🟡 Приоритет 2: Консолидация документации (3-4 часа)

    2.1 Устранить дубликаты README/ARCHITECTURE

    Файл Действие Аргументация
    README.ru.md Объединить с README.md (i18n секции) Поддержка одного файла проще
    docs/ARCHITECTURE.md Удалить, оставить ARCHITECTURE.md в корне Корневой файл — точка входа
    quality-gates-report-20260416-1747.md Переместить в docs/reports/quality-gates/ Динамический отчёт, не документация

    Команды:

    # Объединить README (ручная работа)
# 1. Создать шаблон с markdown-переключателями
# 2. Перенести контент из README.ru.md в секции {lang="ru"}
# 3. Удалить README.ru.md

# Переместить отчёт
mkdir -p docs/reports/quality-gates
mv quality-gates-report-20260416-1747.md docs/reports/quality-gates/

    Ожидаемый результат: - 1 README вместо 2 - 1 ARCHITECTURE.md вместо 2 - Отчёты в docs/reports/

    2.2 Консолидировать отчёты

    Отчёт Действие Аргументация
    REFACTORING_SUMMARY.md Переместить в docs/reports/refactoring/ Исторический отчёт
    PROFESSIONAL_ENVIRONMENT_SETUP_REPORT.md Переместить в docs/reports/setup/ 一次性 отчёт
    SECURITY_VULNERABILITY_ANALYSIS_REPORT.md Переместить в docs/security/reports/ Security-отчёт
    QUALITY-GATES.md Оставить в корне (важно для CI) Критичный документ

    Команды:

    mkdir -p docs/reports/refactoring docs/reports/setup docs/security/reports
mv REFACTORING_SUMMARY.md docs/reports/refactoring/
mv PROFESSIONAL_ENVIRONMENT_SETUP_REPORT.md docs/reports/setup/
mv SECURITY_VULNERABILITY_ANALYSIS_REPORT.md docs/security/reports/

    Ожидаемый результат: - Корень очистится на 3 файла - Все отчёты в одном месте - Упростится навигация

    2.3 Структурировать ADR (Architectural Decision Records)

    Проблема Решение Аргументация
    ADR разбросаны по docs/architecture/decisions/ Упорядочить по номерам ADR-001.md, ADR-002.md Стандарт для ADR
    Нет индекса ADR Создать docs/architecture/decisions/INDEX.md Быстрая навигация
    Устаревшие ADR без статуса Добавить Status: DEPRECATED в заголовки Ясность актуальности

    Команды:

    # Проверить текущую структуру
ls docs/architecture/decisions/

# Создать индекс
cat > docs/architecture/decisions/INDEX.md << 'EOF'
# Architectural Decision Records (ADR)

| Номер | Название | Статус | Дата |
|-------|----------|--------|------|
| ADR-001 | [Разделение на реализацию и анализ](ADR-001.md) | ✅ Active | 2026-01 |
| ADR-002 | [Skill caa-audit как мост](ADR-002.md) | ✅ Active | 2026-01 |
EOF

    Ожидаемый результат: - Чёткая структура ADR - Индекс для быстрой навигации - Ясность по актуальности решений

    🟡 Приоритет 3: Оптимизация .gitignore (30 минут)

    3.1 Добавить генерируемые файлы

    Паттерн Что скрывает Аргументация
    cache/ Кэш-папка Генерируется автоматически
    site/ Сгенерированный сайт Перестраивается по мере необходимости
    plans/ Временные планы ИИ Артефакты, не код
    *.pyc Скомпилированный Python Автоматически генерируется
    __pycache__/ Кэш импортов Стандарт Python
    *.log Логи Добавлено, но проверить
    *.tmp Временные файлы Очистка мусора

    Команды:

    cat >> .gitignore << 'EOF'

# ==========================================
# Дополнительно для оптимизации (2026-04)
# ==========================================
cache/
site/
plans/
*.pyc
__pycache__/
*.log
*.tmp
EOF

    Ожидаемый результат: - Меньше мусора в git status - Чистый дифф при коммитах

    🟢 Приоритет 4: Упрощение структуры docs/ (2-3 часа)

    4.1 Консолидировать подпапки

    Текущая папка Действие Аргументация
    docs/archive/ Объединить с docs/legacy/ Переименовать для ясности
    docs/internal/ Удалить (дублирует docs/methodology/) Избыточность
    docs/external-ecosystem/ Удалить или переименовать в docs/ecosystem/ Упрощение имени
    docs/reports/ Создать (уже планируется выше) Консолидация отчётов

    Команды:

    # Переименовать
mv docs/archive docs/legacy
mv docs/external-ecosystem docs/ecosystem
rm -rf docs/internal  # если пусто или дублирует

# Создать для отчётов
mkdir -p docs/reports/{quality-gates,refactoring,setup}

    Ожидаемый результат: - 30+ → 20+ подпапок - Упрощённая навигация

    ⚠️ Риски и смягчение

    Риск Вероятность Влияние Митигация
    Потеря контекста при перемещении Низкая Среднее Добавить ссылки в старый путь (redirect)
    Сломанные ссылки в документации Средняя Высокое Прогнать mkdocs build для проверки
    Потеря исторических отчётов Низкая Низкое Сохранить в docs/legacy/
    Конфликты при merge Средняя Среднее Сделать feature branch, тестировать в PR

    ✅ Критерии успеха

    1. Корень проекта: < 30 файлов и папок
    2. Документация: Единый индекс, нет дубликатов
    3. .gitignore: Все генерируемые файлы исключены
    4. Навигация: Новый разработчик находит нужное за < 5 минут
    5. CI/CD: Все тесты проходят после изменений
