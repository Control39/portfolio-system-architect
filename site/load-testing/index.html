    План нагрузочного тестирования

    Цель

    Проверить производительность и стабильность системы при высокой нагрузке, имитируя реальное использование несколькими пользователями одновременно.

    Область тестирования

    Тестирование охватывает основные компоненты системы:

    • Infra-Orchestrator: анализ архитектурных решений
    • Decision Engine: генерация рассуждений с помощью YandexGPT
    • IT-Compass: отслеживание навыков и профилактика выгорания
    • ML-Model-Registry: регистрация и управление ML-моделями
    • Portfolio-Organizer: организация портфолио
    • Career-Development: планирование карьеры

    Методология

    Используется инструмент Locust для создания нагрузки. Тесты написаны на Python и находятся в tests/load/test_load.py.

    Профиль пользователя

    Симулируются реалистичные сценарии использования:

    Действие Частота Описание
    Получение анализа архитектуры Высокая Частое использование основной функции
    Получение рассуждений в облаке Высокая Частое использование AI-функций
    Получение оценки навыков Средняя Регулярная проверка компетенций
    Получение моделей ML Средняя Доступ к реестру моделей
    Создание проекта портфолио Низкая Редкие операции создания
    Получение карьерного плана Низкая Периодическое планирование

    Параметры нагрузки

    Параметр Значение Примечание
    Количество пользователей 100 Максимальное ожидаемое количество одновременных пользователей
    Скорость появления 10 пользователей/сек Плавное нарастание нагрузки
    Продолжительность теста 1 час Достаточно для стабилизации системы
    Целевой хост http://localhost:8000 API Gateway

    Метрики производительности

    Будут измеряться следующие метрики:

    Основные метрики

    Метрика Цель Критерий успеха
    Среднее время ответа < 1000 мс Большинство запросов должны отвечать быстро
    95-й перцентиль времени ответа < 2000 мс Даже при пиковой нагрузке задержки должны быть приемлемыми
    Частота ошибок < 1% Система должна быть стабильной
    Пропускная способность > 50 запросов/сек Должна выдерживать ожидаемую нагрузку

    Дополнительные метрики

    • Использование CPU и памяти сервисами
    • Использование сети
    • Задержки базы данных
    • Использование кэша

    Сценарии тестирования

    1. Нормальная рабочая нагрузка

    • 50 пользователей
    • Продолжительность: 30 минут
    • Цель: проверить стабильность при типичной нагрузке

    2. Пиковая нагрузка

    • 100 пользователей
    • Продолжительность: 1 час
    • Цель: проверить пределы производительности

    3. Стресс-тест

    • 150 пользователей (на 50% больше максимального)
    • Продолжительность: 15 минут
    • Цель: определить точку отказа системы

    Подготовка к тестированию

    1. Подготовка окружения

    # Запуск всех сервисов
docker compose up -d

# Ожидание готовности сервисов
sleep 30

# Проверка состояния
curl -f http://localhost:8000/health

    2. Подготовка данных

    # Создание тестовых данных
python scripts/generate_test_data.py --users 1000 --projects 500

    Запуск тестов

    Запуск через CLI (без интерфейса)

    locust -f tests/load/test_load.py \
  --host=http://localhost:8000 \
  -u 100 \
  -r 10 \
  --run-time=1h \
  --headless \
  --csv=results/load_test

    Запуск с веб-интерфейсом

    locust -f tests/load/test_load.py --host=http://localhost:8000

    Затем открыть http://localhost:8089 в браузере.

    Анализ результатов

    Результаты будут доступны в нескольких форматах:

    1. CSV-файлы

    Будут сгенерированы следующие файлы:

    • load_test_stats.csv - статистика по всем запросам
    • load_test_failures.csv - информация об ошибках
    • load_test_history.csv - история метрик во времени

    2. Веб-интерфейс Locust

    В реальном времени доступны:

    • Графики RPS (запросов в секунду)
    • Графики среднего времени ответа
    • Графики 95-го перцентиля
    • Количество пользователей
    • Частота ошибок

    3. Отчет

    После завершения теста будет сгенерирован отчет с выводами:

    ## Отчет о нагрузочном тестировании

**Дата**: $(date)
**Версия системы**: $(git rev-parse --short HEAD)

### Сводка

- **Статус**: Успешно/Частично успешно/Неуспешно
- **Продолжительность**: X минут
- **Пиковые пользователи**: X
- **Общее количество запросов**: X
- **Частота ошибок**: X%

### Ключевые метрики

| Метрика | Результат | Статус |
|--------|---------|--------|
| Среднее время ответа | X мс | ✅/❌ |
| 95-й перцентиль | X мс | ✅/❌ |
| Частота ошибок | X% | ✅/❌ |
| Пропускная способность | X запросов/сек | ✅/❌ |

    Интеграция с CI/CD

    Нагрузочное тестирование можно интегрировать в CI/CD процесс:

    name: Load Testing

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  load-test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:14
        env:
          POSTGRES_DB: portfolio_test
          POSTGRES_PASSWORD: postgres
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
    - name: Checkout
      uses: actions/checkout@v4

    - name: Setup Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.11'

    - name: Install dependencies
      run: |
        pip install -r requirements-dev.txt
        pip install locust

    - name: Start application
      run: |
        docker compose up -d
        sleep 30

    - name: Run load test
      run: |
        locust -f tests/load/test_load.py \
          --host=http://localhost:8000 \
          -u 50 \
          -r 5 \
          --run-time=10m \
          --headless \
          --csv=results/load_test

    - name: Upload results
      uses: actions/upload-artifact@v4
      with:
        name: load-test-results
        path: |
          results/load_test_*.csv

    - name: Analyze results
      run: |
        # Проверка ключевых метрик
        python scripts/analyze_load_results.py results/load_test_stats.csv

    Мониторинг во время тестирования

    Рекомендуется запускать мониторинг во время тестирования:

    # Запуск Prometheus и Grafana
docker compose -f docker-compose.yml -f docker-compose.monitoring.yml up -d prometheus grafana

    Доступные дашборды:

    • Grafana: http://localhost:3000 (admin/admin)
    • System Performance
    • Database Metrics
    • API Gateway Metrics
    • Prometheus: http://localhost:9090

    Рекомендации по оптимизации

    Если тесты покажут проблемы с производительностью, рекомендуется:

    1. Оптимизация базы данных:
    2. Добавить индексы для часто используемых запросов
    3. Настроить кэширование запросов

    4. Рассмотреть горизонтальное масштабирование

    5. Оптимизация API:

    6. Реализовать кэширование ответов
    7. Оптимизировать сериализацию данных

    8. Рассмотреть пагинацию для больших ответов

    9. Оптимизация AI-сервисов:

    10. Реализовать кэширование ответов YandexGPT
    11. Настроить таймауты и повторные попытки

    12. Рассмотреть батчинг запросов

    13. Инфраструктурные улучшения:

    14. Настроить балансировку нагрузки
    15. Увеличить ресурсы для критических сервисов
    16. Рассмотреть использование CDN
