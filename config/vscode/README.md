# Конфигурация расширений VS Code

## 📋 Обзор

Эта папка содержит конфигурационные файлы для управления расширениями VS Code в проекте `portfolio-system-architect`. Конфигурация основана на анализе технологического стека проекта и обеспечивает единообразие среды разработки.

## 📁 Файлы

| Файл | Назначение |
|------|------------|
| `vscode-extensions.json` | Основная конфигурация расширений с категориями |
| `vscode-recommended-extensions.json` | Рекомендуемые расширения для VS Code (автоматически генерируется) |
| `README.md` | Эта документация |

## 🎯 Категории расширений

### 1. Обязательные (required)
**Влияние на оценку:** Высокое (70% оценки)

Расширения, необходимые для работы с проектом:
- Python разработка
- Контейнеризация и DevOps
- Git инструменты
- Качество кода

### 2. Рекомендуемые (recommended)
**Влияние на оценку:** Среднее (отображается в отчете)

Расширения, улучшающие разработку:
- FastAPI специфика
- Kubernetes и инфраструктура
- Веб и API инструменты
- Документация

### 3. Опциональные (optional)
**Влияние на оценку:** Нет

Расширения по выбору разработчика:
- AI ассистенты
- Дополнительные инструменты
- Экспериментальные расширения

### 4. Исключенные (excluded)
**Влияние на оценку:** Отрицательное (-10% за каждое)

Расширения, которые не должны использоваться в проекте:
- Конфликтующие расширения
- Устаревшие инструменты
- Несовместимые плагины

```json
  ]
}
```

## 📝 Как добавить новое расширение

### 1. Определите категорию

**Вопросы для определения категории:**
- Расширение необходимо для работы с проектом? → **required**
- Расширение улучшает разработку, но не обязательно? → **recommended**
- Расширение по выбору разработчика? → **optional**
- Расширение конфликтует или не нужно? → **excluded**

### 2. Найдите ID расширения

1. Откройте VS Code
2. Перейдите в Marketplace (Ctrl+Shift+X)
3. Найдите расширение
4. Скопируйте ID из:
   - URL: `https://marketplace.visualstudio.com/items?itemName=издатель.название`
   - Описания: Обычно в формате `издатель.название`

### 3. Добавьте в конфигурацию

```json
{
  "required": [
    ...,
    "новый-издатель.новое-название"
  ]
}
```

### 4. Проверьте изменения

```bash
# Перейдите в корень проекта
cd /path/to/portfolio-system-architect

# Проверьте конфигурацию
make check

# При необходимости синхронизируйте
make sync
```

## 🔄 Обновление конфигурации

### Автоматическое обновление

Система автоматически проверяет конфигурацию при:
- Изменении файла `vscode-extensions.json`
- Запуске CI/CD workflow
- Еженедельном расписании

### Ручное обновление

1. Отредактируйте `vscode-extensions.json`
2. Запустите проверку: `make check`
3. Если есть проблемы, запустите синхронизацию: `make sync`
4. Создайте pull request с изменениями

## 🤝 Совместная работа

### Workflow для команды

1. **Разработчик** предлагает новое расширение
2. **Ревьюер** проверяет уместность категории
3. **Менеджер** утверждает изменение
4. **CI/CD** автоматически проверяет конфигурацию
5. **Все разработчики** синхронизируют расширения

### Разрешение конфликтов

Если возникают конфликты расширений:

1. Проверьте логи VS Code
2. Временно переместите расширение в `optional`
3. Протестируйте в изолированной среде
4. Примите решение о категории
5. Обновите конфигурацию

## 📈 Метрики и отчеты

### Оценка соответствия

Система рассчитывает оценку по формуле:
```
Оценка = (Установленные обязательные / Все обязательные) × 70 - (Исключенные × 10)
```

### Генерация отчетов

```bash
# Консольный отчет
make check

# Markdown отчет
make report

# Отчет для CI/CD
python scripts/vscode-extensions-manager.py --output-score
```

## 🐛 Устранение неполадок

### Ошибка "Invalid JSON"

```bash
# Проверьте синтаксис
python -m json.tool config/vscode/vscode-extensions.json

# Ищите проблемные строки
grep -n "//" config/vscode/vscode-extensions.json
```

### Ошибка "Extension not found"

1. Проверьте ID расширения
2. Убедитесь, что расширение существует в Marketplace
3. Проверьте регистр и пунктуацию

### Конфликт расширений

1. Временно отключите одно из расширений
2. Проверьте работу проекта
3. Обновите конфигурацию соответствующим образом
